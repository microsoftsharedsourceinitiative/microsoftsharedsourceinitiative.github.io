Providing DEVMODE Structure Additions

Your user interface plug-in can add its own private members to the DEVMODE structure, as illustrated in the following figure.

Public and Private DEVMODE Sections

A UI plug-in can use these private DEVMODE members for storing values associated with customized printer options. The plug-in makes these options available to the user by modifying a driver-supplied property sheet page or by adding new property sheet pages.

If your UI plug-in adds private DEVMODE members, the added members must be prefixed by the OEM_DMEXTRAHEADER structure.

You are not required to add members to the DEVMODE structure, but if you do, your plug-in must implement the IPrintOemUI::DevMode method. This method's purpose, depending on input arguments, is to return the size of, initialize, convert, or validate the additional DEVMODE members.
