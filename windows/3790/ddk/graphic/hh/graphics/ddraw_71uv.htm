Turning On a Display Card and Changing Modes on Windows 98/Me

The following functions are implemented in a Windows 98/Me GDI driver to turn on the display card, and also to change the mode of the display card. A GDI driver that runs on Windows 98/Me is the 16-bit portion of an overall display driver. For information on initializing a GDI driver, see Initialization in the Graphics Device Interface section of the Windows 98/Me section of the Windows DDK documentation. For information on how the GDI driver is used to initialize a DirectDraw driver--32-bit portion of the display driver, see Windows 98/Me Driver Initialization.

Enable
Initializes software and hardware for the driver. Enable can either turn the display card on, or it can change the display mode depending on the parameters sent to it when it is called. Enable and other graphics driver functions are documented in the Graphics Device Interface section of the Windows 98/Me section of the Windows DDK documentation.
ReEnable
Reinitializes the driver object. ReEnable is called every time the display mode changes. In turn, ReEnable calls Enable with parameters to reinitialize the driver object. When Enable is called through ReEnable, the mode is changed, but no other object structure is created. The GDI driver-supplied DDCreateDriverObject function is called in Enable, but the bReset parameter is set to one, which just resets the mode.
