vddIsOurIRQ

The vddIsOurIRQ callback function indicates whether the display device generated the interrupt request (IRQ) and, if so, the interrupt events that the IRQ represents.

DWORD
  vddIsOurIRQ(
    DWORD  dwDevHandle
    );

Parameters

dwDevHandle
Handle to the miniport driver's device extension.

Return Value

vddIsOurIRQ returns zero if the IRQ was not caused by the display device; otherwise, it returns one or more of the following flags that represent the interrupt events that generated the IRQ:

Flag Meaning
DDIRQ_BUSMASTER The device generated an interrupt because a transfer request completed.
DDIRQ_DISPLAY_VSYNC The device generated an interrupt based on the display V-sync.
DDIRQ_VPORT0_LINE The device generated an interrupt based on a programmable line for hardware video port number 0.
DDIRQ_VPORT0_VSYNC The device generated a V-sync interrupt for hardware video port number 0.
DDIRQ_VPORT1_LINE The device generated an interrupt based on a programmable line for hardware video port number 1.
DDIRQ_VPORT1_VSYNC The device generated a V-sync interrupt for hardware video port number 1.
DDIRQ_VPORT2_LINE The device generated an interrupt based on a programmable line for hardware video port number 2.
DDIRQ_VPORT2_VSYNC The device generated a V-sync interrupt for hardware video port number 2.

Headers

Declared in ddkmmini.h. Include ddkmmini.h.

Comments

On Windows 98/Me platforms, an IRQ can be managed either by the display driver or by the DirectDraw runtime on the driver's behalf. The runtime calls the driver's vddIsOurIRQ function only when the driver does not manage the IRQ. When the runtime manages IRQs for the driver and an IRQ occurs, the runtime calls the driver's vddIsOurIRQ function to determine whether the VGA chip caused the IRQ and, if so, what interrupt events caused the IRQ.
