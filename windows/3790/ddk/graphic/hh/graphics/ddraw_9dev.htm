Testing and Registering DirectX Device Drivers

This topic applies only to Windows 98/Me.

The Microsoft® Windows® Hardware Quality Labs (WHQL) ensures the compatibility of hardware with Microsoft Windows operating systems. WHQL provides test kits to qualify devices along with their drivers for the "Designed for Windows" logos and for inclusion on the Windows Hardware Compatibility List (HCL). Device drivers should be tested using these test kits. In effect, these device drivers then become registered.

By default, DirectX Setup replaces registered drivers only with other registered drivers that have higher version numbers. Unregistered drivers can be replaced during application installation at the discretion of the end user or application. For more information about WHQL, see Testing Drivers and Distributing a Driver.

To become a registered DirectX driver, the device driver must first pass the standard WHQL tests for Windows 98/Me, and then pass an additional set of DirectX-specific tests. If the driver passes all of these tests, WHQL links the code into the new device driver that allows DirectX to determine whether the driver is registered, and a registered version number is assigned. The registered binary version of the driver is then returned to the vendor.
