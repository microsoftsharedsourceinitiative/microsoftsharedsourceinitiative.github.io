CPixel Support Methods for Light-Weight Mipmaps

This section describes the methods defined for the CPixel class. Method prototypes are defined in the pixel.hpp file. This file along with pixel.cpp and pixlib.cpp are in the DDK and are used to build the PixLib.lib support library.

Methods of the CPixel class are used to recover the layout of light-weight system memory mipmap textures. For example, only information about the top level of a light-weight mipmap texture is stored. However, the runtime uses CPixel class methods to calculate the locations of sublevels for this light-weight mipmap texture. Consequently, if a driver must copy a light-weight system memory mipmap texture to video memory, the driver can use the CPixel class methods to calculate the source texture's size and the offset to the source texture's sublevels. Alternatively, driver writers can implement their own code that calculates this size and sublevel offset. To ensure that the runtime and drivers recover the memory layout of light-weight system memory mipmap textures in the same manner, this code must follow the same rules as the CPixel class.

In order for your driver to use CPixel class methods, you must include the pixel.hpp file in your code and link to PixLib.lib when you build your driver.
