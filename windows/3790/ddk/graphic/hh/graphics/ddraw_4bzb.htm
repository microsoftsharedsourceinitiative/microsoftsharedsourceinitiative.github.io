Kernel-mode DxApi Interface Support For Windows 98/Me

Note  DxApi interface support is only available for Windows 98/Me in kernel mode because it requires changes to the Main VDD.

Kernel-mode DxApi interface support is useful for the following operations:

  • Autoflipping using an IRQ for devices that do not support hardware autoflipping or that have limitations that make it undependable. This allows DirectDraw to always revert to software autoflipping when hardware autoflipping is unavailable.
  • Field skipping using an IRQ to support MPEG drivers that can undo the 3:2 pulldown of MPEG data originally sampled from film.
  • Bus mastering, so devices can continuously transfer data without having to call Lock / Unlock for every frame. This is especially useful in kernel mode because the drivers for these devices exist in kernel mode in WDM.
  • Capturing video and VBI. In kernel mode, it is easy to capture video that is based on a video port IRQ or graphics IRQ.

Support for the DxApi interface is added into the miniVDD.
