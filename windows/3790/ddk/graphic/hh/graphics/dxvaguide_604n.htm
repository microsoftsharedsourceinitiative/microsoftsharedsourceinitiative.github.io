Deinterlace Container Device

The DirectX VA deinterlace container device does not use typed parameters. Thus, the sample functions described in Sample Functions for Deinterlacing can be used for many different purposes. However, these functions can only be used in the context of a DirectX VA device, so it is necessary to first define and create a deinterlace container device.

If a driver supports accelerated decoding of compressed video, the VMR calls the driver to create two DirectX VA devices: one to perform the actual video decoding work and one to be used strictly for deinterlacing.

It should be emphasized that the deinterlace container device is a software construct only, and does not represent any functional hardware contained on a device.
