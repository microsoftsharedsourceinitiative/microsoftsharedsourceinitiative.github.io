32-bit Callback Functions

Callback functions should be implemented in the 32-bit portion of the HAL. If, for some reason, a particular callback cannot be implemented in 32-bit, it can be written in 16-bit instead. If it is written in 16-bit, do not set the bit in the callback table that specifies it is a 32-bit callback function. This way, the callback is called from the 16-bit side by default (it still must be listed in the DDHALINFO structure). When that is done, ddraw16.dll calls the callback function rather than ddraw.dll, and the callback function is run on the 16-bit side.

The following sample 32-bit HAL callbacks are discussed in this section:

Blt32

DdGetHeapAlignment

DdUpdateNonLocalHeap

Flip32

GetBltStatus32

GetFlipStatus32

GetScanLine32

Lock32

WaitForVerticalBlank32

Except for DdGetHeapAlignment and DdUpdateNonLocalHeap, each of the preceding callbacks have NT-based equivalent callback functions. The NT-based equivalent callbacks are specified in the Comments section of their respective Windows 98/Me callbacks.
