Calling the HAL Function

This topic applies only to Windows 98/Me.

When a HAL function is invoked, DirectDraw tests to determine whether the function is 16- or 32-bit. If the HAL function is 32-bit, ddraw.dll calls the 32-bit DirectDraw driver DLL directly. If the HAL function is 16-bit, the pointers are aliased, ddraw16.dll is thunked to, and ddraw16.dll calls the 16-bit function. After that occurs, the pointer aliases are freed.
