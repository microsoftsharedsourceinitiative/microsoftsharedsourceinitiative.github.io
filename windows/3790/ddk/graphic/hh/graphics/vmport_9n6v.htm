Accessing System Memory through AGP

The miniport driver should call the following video port driver-supplied functions to enable its device to access main memory:

  1. Call AgpReservePhysical to reserve a range of physical addresses on the system bus to which the AGP controller can respond.
  2. Call AgpCommitPhysical to lock physical memory and have it mapped to a portion of the AGP-decodable physical address range returned by AgpReservePhysical. The miniport driver can call AgpCommitPhysical several times to do many smaller commitments rather than one large one.

Then, for an application to be able to see and use this committed AGP-controlled memory, the video miniport driver must call the following functions:

  1. Call AgpReserveVirtual to reserve a range of virtual addresses in the application's address space.
  2. Call AgpCommitVirtual to lock virtual memory and have the virtual addresses mapped to the AGP-decodable physical address range returned by AgpReservePhysical.

An application can access committed virtual memory as if it were contiguous.
