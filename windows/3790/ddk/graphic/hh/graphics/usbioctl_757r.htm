IOCTL_USBPRINT_VENDOR_SET_COMMAND

Operation

The IOCTL_USBPRINT_VENDOR_SET_COMMAND request allows upper-layer software (such as a language monitor) to issue a vendor-specific SET command to the target device.

Input

The Irp->Parameters.DeviceIoControl.OutputBufferLength member specifies the size, in bytes, of the buffer passed in the Irp->AssociatedIrp.SystemBuffer member, containing the vendor-specific SET command.

The Irp->AssociatedIrp.SystemBuffer member contains parameters packed together as an array. Interpreted as an array of bytes, the buffer contains the following:

Array Element Contents
SystemBuffer[0] Vendor Request code
SystemBuffer[1] Vendor Request value - Most Significant Byte
SystemBuffer[2] Vendor Request value - Least Significant Byte
SystemBuffer[3...] Request-specific data to be sent to the device

Output

None

Status Block

The Information member is set to zero.

The Status member is set to STATUS_SUCCESS if the request is successful. Otherwise, it is set to an appropriate error condition, such as STATUS_INVALID_PARAMETER or STATUS_NO_MEMORY.

Headers

Declared in ioctl.h. Include ioctl.h.
