Initialization Procedure

This topic applies only to Windows 98/Me.

This section describes the code required for your driver to properly initialize for DirectDraw support. The functions are described in the order they participate in the initialization process. Consequently, the discussion begins with the defines and the GDI driver Control function. The Control function calls your 32-bit HAL entry point in your DirectDraw driver DLL (DriverInit in the sample), which in turn calls the functions that fill in the DDHALINFO structure with the information about your driver (buildDDHALInfo32, DDCreateDriverObject, and buildDDHALInfo16 in the sample). For information on the Control function, see the Graphics Device Interface section of the Windows 98/Me section of the Windows DDK documentation.

Note  For clarity, the sample initialization functions in this section are written in C++. For an example of an optimized version of the initialization procedure, see the sample code included in the DirectX DDK (p3samp).
