Mapping Gamma Ramp Values to RAMDACs

This topic applies only to Windows Me.

The RAMDAC uses the buffer (or gamma ramp) as a lookup table before mapping gamma ramp data into the most significant bits of the words. The gamma ramp always expects values in the range 0x0000 to 0xFFFF.

ICM 3.0: The Long-term Solution

As a long-term solution for gamma ramp support, Microsoft plans to add the functionality of the DirectDraw call into the next version of ICM, version 3.0. DirectDraw will continue to support this feature, but when the next version of ICM is complete, ICM will be used for this function instead of the current method of loading the gamma calibrator. Applications will not be affected, but gamma calibrators will.

ICM will be able to control the gamma ramp process without allowing other applications to intercept the gamma ramp adjustment; this will make it easier to display accurate gamma ramps.
