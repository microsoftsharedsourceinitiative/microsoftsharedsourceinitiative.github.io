Display and Print Devices: Windows DDK

Overview

This topic applies only to Windows 98/Me.

The following procedure summarizes what the vendor must do to ensure that a DirectX driver is registered and works correctly with DirectX DLLs.

To ensure that a DirectX driver works correctly with DirectX DLLs

  1. Ensure that all file names provided with the device driver are unique to the vendor, as described in Naming Windows 98/Me Device Drivers.
  2. Ensure that the device driver version number is correct for the Microsoft Windows® version and DirectX version, as described in the section Ensuring Correct Version Numbers.
  3. Ensure that your DirectX driver is registered by completing Windows Hardware Quality Labs (WHQL) testing and acceptance, as described in Testing and Registering DirectX Device Drivers.

DirectX Setup does not recognize an installed driver as newer unless it is registered and versioned as a DirectX driver.

Note  A "registered" driver is one for which the driver has been tested and accepted by WHQL with additional testing for DirectX compatibility. An "unregistered" driver is one that has not been tested and accepted by WHQL for DirectX compatibility. A registered DirectX driver is one that is provided by Microsoft or that has successfully passed WHQL testing, including validation that the driver correctly provides a version number as described in this topic.
