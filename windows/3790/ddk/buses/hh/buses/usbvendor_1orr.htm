Vendor-Supplied USB Client Drivers

This section discusses guidelines for the development and installation of vendor-supplied universal serial bus (USB) device drivers. In this section, the term "USB client driver" refers to any driver, or driver stack, that communicates with the USB stack by means of USB request blocks (URBs). Typically, a USB client driver sits on top of the USB stack and manages a USB device. For example, Windows provides a human interface device (HID) class driver, hidclass.sys, that manages input devices. The HID class driver communicates with the USB stack by means of a miniclass driver, hidusb.sys. The HID class driver and miniclass driver are, together, an example of a USB client driver. For more information about the system input architecture, see Driver Stacks for HIDClass Devices.

Also provided in this section is a description of the I/O request interface that client drivers must use to communicate with the USB stack, and information about the sample USB client driver that is included with the Windows DDK.
