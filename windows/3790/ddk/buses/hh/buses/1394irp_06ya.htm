IOCTL_IEEE1394_API_REQUEST

Operation

An IEEE 1394 driver uses the IRP_MJ_DEVICE_CONTROL IRP, with IoControlCode IOCTL_IEEE1394_API_REQUEST, to dynamically add or remove a virtual 1394 device. This I/O request is only available in Microsoft® Windows® XP and later operating systems.

Input

AssocicatedIrp.SystemBuffer->SystemBuffer points to an IEEE1394_API_REQUEST structure. The RequestNumber member of the IEEE1394_API_REQUEST structure specifies the type of request. The u member of the IEEE1394_API_REQUEST structure is a union that specifies the request-type-specific parameters of the request. The parameters and their meaning are documented with each request.

Parameters.DeviceIoControl.InputBufferLength provides the length of the input buffer.

Legal values for the RequestNumber member of IEEE1394_API_REQUEST are as follows:

IEEE1394_API_ADD_VIRTUAL_DEVICE

IEEE1394_API_REMOVE_VIRTUAL_DEVICE

Output

AssocicatedIrp.SystemBuffer->SystemBuffer points to the IEEE1394_API_REQUEST structure passed as input. As part of completing the request, the bus driver fills in certain members of the u member with information for the driver. The returned information is documented with each request.

I/O Status Block

The information the bus driver returns in the I/O Status Block is documented with each request.

Headers

Defined in 1394.h. Include 1394.h.
