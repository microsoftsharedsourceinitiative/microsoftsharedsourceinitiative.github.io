System Support for Buses: Windows DDK

_URB_FRAME_LENGTH_CONTROL

This structure is obsolete in Windows 2000 and later operating systems and is no longer supported by Microsoft.

If you use a URB_FUNCTION_TAKE_FRAME_LENGTH_CONTROL request or a URB_FUNCTION_RELEASE_FRAME_LENGTH_CONTROL request, the request will fail and the system will report an error.

In the past, USB client drivers used this structure to take or release control of the frame length on the bus.

struct _URB_FRAME_LENGTH_CONTROL {
  struct _URB_HEADER  Hdr;
} ;

Members

Hdr
Pointer to a _URB_HEADER structure that specifies the URB header information. Hdr.Function must be either URB_FUNCTION_TAKE_FRAME_LENGTH_CONTROL or URB_FUNCTION_RELEASE_FRAME_LENGTH_CONTROL, and Hdr.Length must be sizeof(_URB_FRAME_LENGTH_CONTROL).

Headers

Declared in usb.h. Include usb.h.

Comments

Only one client can have control of frame length at any time. To take control of the frame length, a driver sets Hdr.Function to URB_FUNCTION_TAKE_FRAME_LENGTH_CONTROL. When a client that has taken control of the frame length is ready to release control, it must release control by sending this URB with Hdr.Function set to URB_FUNCTION_RELEASE_FRAME_LENGTH_CONTROL.
