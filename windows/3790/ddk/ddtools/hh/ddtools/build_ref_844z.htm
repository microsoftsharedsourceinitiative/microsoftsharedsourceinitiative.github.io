O

The O constant specifies the subdirectory into which build objects will be placed.

The benefit of using $(O) is that any files that you have built and placed in the objects subdirectory will be deleted on the next clean build. This guarantees that no collisions will occur between two builds running on the same machine at the same time. They will never override each other's files if you follow the convention that everything you build goes in $(O).
