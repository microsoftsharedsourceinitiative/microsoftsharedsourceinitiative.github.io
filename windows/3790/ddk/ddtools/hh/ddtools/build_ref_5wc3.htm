NTDEBUG

Use the NTDEBUG environment variable to specify what type of symbolic information you want the Build utility to place in the symbol files it creates.

This environment variable is rarely used in the sources file. Instead, it is set in the environment by the Windows DDK.

To indicate a checked (or debug) build, specify:

NTDEBUG=ntsd

To indicate a free (or retail) build, specify:

NTDEBUG= ntsdnodbg

The only difference between the two of them is that specifying "ntsd" causes the Build utility to define DBG=1, while specifying "ntsdnodbg" causes it to define DBG=0.

The DBG variable is tested by a number of debugging-related driver routines; for more information, see Kernel-Mode Debugging Routines. You can also use #if DBG statements in your source code to conditionally compile free and checked versions of your driver.
