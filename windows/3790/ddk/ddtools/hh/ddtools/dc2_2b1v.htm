Query and Set File Information Test

Command-line parameter: /q

During the Query and Set File Information Test, Device Path Exerciser issue calls to retrieve and change the object, file, and volume information of devices opened by the Basic Open Operations and other open operations, including the Relative-Open Test.

Device Path Exerciser issues each query or set call at least 1024 times with a valid buffer and a variety of buffer lengths and file information classes. One request of each type is also sent with an invalid buffer pointer and a zero buffer length.

This test is optional. To add the Query and Set File Information Test to a Device Path Exerciser test session, use the /q parameter.

If you use the /pr parameter, which sets the protection option, Device Path Exerciser varies the security setting on the buffer in each query and set call. If you use the /s parameter, Device Path Exerciser performs the test on devices opened during the Relative Open Test.

This test uses the ZwQueryInformationFile, and ZwSetInformationFile functions, which are documented in the Windows DDK, and the ZwQueryVolumeInformationFile, ZwSetVolumeInformationFile, ZwQueryObject functions, which are documented in the Installable File System Kit (IFS Kit).
