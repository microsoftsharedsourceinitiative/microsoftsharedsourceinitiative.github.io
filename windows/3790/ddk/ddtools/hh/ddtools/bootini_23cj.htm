Driver Development Tools: Windows DDK

/nopae

Forces Ntldr to load the non-Physical Address Extension (PAE) version of the Windows kernel, even when the processor supports PAE and the computer has more than 4 GB of physical memory.

[ /nopae ]

Comments

PAE is an Intel-supported memory address extension that allows 32-bit x86-based systems to use 36-bit addresses to address up to 64 GB of physical memory.

Consider using this parameter on systems where a device does not operate properly when PAE is enabled.

This parameter is valid only on x86-based systems.

Example

multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="Windows Server 2003, Enterprise" /fastdetect /nopae

Bootcfg Command

bootcfg /raw "/nopae" /A /ID 1
