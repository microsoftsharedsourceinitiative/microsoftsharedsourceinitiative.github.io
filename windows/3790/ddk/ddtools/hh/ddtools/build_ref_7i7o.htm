COMPILER_WARNINGS

The COMPILER_WARNINGS macro specifies the name of the file that contains warning information. By default, this is $(SDK_INC_PATH)\warning.h. If a different file is specified in this macro, the name of this file will be passed to the compiler with the /FI switch.

The file contains a list of compiler pragmas used to disable, enable, or promote warnings for the entire build.
