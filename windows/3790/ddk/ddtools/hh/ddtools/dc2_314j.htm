Streams Test

Command-line parameter: /s /j

During the Streams Test, Device Path Exerciser tries to open a variety of named data streams on the device. The tests use a series of arbitrary stream names with content and characters that might be valid for other uses on some devices. This test determines whether the driver can properly handle data stream requests, especially if the driver exports a device that does not support or anticipate data streams.

A named data stream is an attribute of a file object. You specify a named data stream by writing the name of the file, a colon, and the name of the data stream, for example, File01.txt:AccessDate where AccessDate is a named data stream, that is, an attribute of the File01.txt file.

Device Path Exerciser records the stream names used in the test in the Dc2.log file.

The Streams Test is an optional extension of the Relative Open Test. To select this test, use the /j parameter (Streams test) with the /s parameter (Relative Open Test).
