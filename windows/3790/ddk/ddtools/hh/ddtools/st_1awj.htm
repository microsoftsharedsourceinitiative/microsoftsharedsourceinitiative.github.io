Summary Message File

By default, the Tracefmt utility creates a summary message file after processing all the messages in a trace log. A summary message file is a text file that contains the following information about a software trace:

  • The number of logger buffers processed
  • The number of messages processed and lost
  • The elapsed time, in microseconds, of the tracing session
  • For each message, the event name (as specified by its GUID's friendly name), the event type (as specified by "filename_extNNN"), and the GUID

The summary message file is used primarily to debug software tracing and its format is subject to change.
