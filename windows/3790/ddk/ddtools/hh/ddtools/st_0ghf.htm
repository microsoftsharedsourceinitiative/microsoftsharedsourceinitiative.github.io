Tracepdb Utility

The Tracepdb command-line utility creates a trace message format file that the Tracefmt utility uses to convert logged messages to a human-readable form.

Usage

The following is the minimum required to create a trace message format file in user-specified location:

tracepdb -f DriverName.pdb -p MyPathName

The following are the commonly used options.

Option Description
-f DriverName.pdb Specifies a driver's PDB file.
-p MyPathName Specifies the path where the Tracepdb utility saves the trace message format file. If this option is not specified, the default location is the current directory. The file name is DriverName.tmf.
-v The utility prints a list of options to standard out.

Location

The Tracepdb utility is located in the tools\tracing directory of the Windows DDK.

Comments

A trace consumer must use the Tracepdb utility to create a trace message format file.

To specify the path that the Tracefmt utility uses to locate trace message format files, a trace consumer can set a TRACE_MESSAGE_SEARCH_PATH environment variable in the following way:

SET TRACE_MESSAGE_SEARCH_PATH=MyPathName

Note  In addition to a trace message format file, the Tracepdb utility also generates a file with a .tmc extension. This file is currently not used, but might be used as input to an enhanced logging capability in a future release of WPP software tracing.
