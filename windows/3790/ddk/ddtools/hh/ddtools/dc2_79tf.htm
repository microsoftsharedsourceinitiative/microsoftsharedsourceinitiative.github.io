Direct Device Open Operations

Command-line parameter: /dd

During the Direct Device Open Operations, Device Path Exerciser opens the device directly, as a device, not as a file in a file system. Direct Device Open Operations are always synchronous. If the call is successful, Device Path Exerciser uses the handle provided to perform other selected tests.

Direct Device Open Operations are an optional extension of the Synchronous Open Operations (/k). To add Direct Device Open Operations to a Device Path Exerciser test session, use the /dd parameter. If you specify the /dd parameter without the /k parameter, the /dd parameter is ignored.
