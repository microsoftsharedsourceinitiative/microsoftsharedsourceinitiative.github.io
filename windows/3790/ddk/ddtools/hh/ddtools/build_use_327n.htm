Setting Up the Build Environment

Before running the Build utility, you must install the DDK to a directory whose path does not contain spaces.

When the DDK is installed, it defines the Start menu shortcuts for several different build environments. Select a build environment by choosing one of the environment shortcuts located in Start | Programs | Development Kits | Windows DDK version | Build Environments.

Note  When a build environment title contains the term "64 bit," this refers to the Intel Itanium architecture. When a build environment title contains the term "AMD64," this refers to the AMD x86-64 architecture.

When you select one of the build environment shortcuts, a special Command Prompt window appears. A batch file named setenv.bat executes in this window, setting the values of numerous environment variables. For more information about environment variables, see Using Macros and Environment Variables.

The Build utility should always be launched from one of these build environment windows.
