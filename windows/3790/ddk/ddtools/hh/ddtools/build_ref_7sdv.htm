Build Utility Command-Line Options

The following is a complete list of command-line options for the Build utility. These switches should all be preceded with a hyphen (-):

Option Effect
-# Forces _objects.mac to be regenerated.
-0 Pass-zero generation only; do not compile or link.
-2 Same as old -Z (only do a two-pass build — no pass zero).
-3 Same as -Z.
-a Allows synchronized blocks and drains during link pass.
-b Displays full error message text without truncating anything.
-c Deletes all object files.
-C Deletes all .lib files only.
-clean Equivalent to -nmake clean.
-D Check dependencies before building.
-e Generates the log, error, and warning files. (For more information, see Using Log Files.)
-E Always keep the log files.
-f Forces rescan of all source and include files.
-F When displaying errors/warnings to stdout, print the full path.
-G Enables target-specific dirs files if and only if there is just one target.
-i Ignores extraneous compiler warning messages.
-I Do not display thread index if multiprocessor build.
-j filename Use filename as the name for log files. The j switch changes the first field, so it produces filename.log, filename.wrn, and filename.err. This can be used to set up a single machine with a shared source drive, which is connected with an x86 machine and with an Itanium machine. On each of these machines, you can set the j switch so you can build all those machines at the same time across the net and produce the log, warning, and error files for each platform's build.
-jpath pathname Use pathname as the path for log files instead of ".".
-k Keep out-of-date targets (do not delete them).
-l Link only, no compiles.
-L Compile only, no link phase.
-M [threads] Multiprocessor build. Without threads, the -M switch causes Build to launch one thread for each processor. If threads is present, it specifies the exact number of threads to spawn. When -M is used without threads on a single-processor computer; it has no effect.
-m Background mode; run the Build utility in the idle priority class.
-nmake arg Argument to pass to NMAKE.
-o Display out-of-date files.
-O Generate obj\_objects.mac file for current directory.
-p Pause before compile and link phases.
-P Print elapsed time after every directory.
-q Query only; do not run NMAKE.
-r dirpath Restarts clean build at specified path.
-s Display status line at top of display.
-S Display status line with include file line counts.
-t Display the first level of the dependency tree.
-T Display the complete dependency tree.
-u Display unused BUILD_OPTIONS.
-$ Display the complete dependency tree hierarchically.
-v Enable include file version checking.
-w Show warnings on screen.
-why List reasons for building targets.
-x filename Exclude include file from dependency checks.
-y Show files scanned.
-z No dependency checking or scanning of source files; one pass compile and link.
-Z No dependency checking or scanning of source files; three passes.
-386 Build targets for i386.
-x86 Same as -386.
-ia64 Build targets for Itanium.
-? Display help text with these command-line options.

Parameters not preceded with a hyphen will be understood as additional source directories to be built. Parameters preceded with a tilde (~) specify directories that are not to be built. These directories must be listed in the dirs file.

An asterisk (*) as a parameter means that the Build utility should build all optional source directories.
