Using Padded Responses and Stripped Responses

Padded Responses entries are modem INF Responses entries that have leading and trailing <cr><lf> sequences — for example:

HKR, Responses, "<cr><lf>CONNECT 300/V42<cr><lf>",   1, 02, 02, 2C,01,00,00, 00,00,00,00

In contrast, the following example shows a stripped response that lacks a leading or trailing <cr><lf> and that requires a separate <cr> and <lf> response:

HKR, Responses, "CONNECT 300/V42",   1, 02, 02, 2C,01,00,00, 00,00,00,00
HKR, Responses, "<cr>",    1, 01, 00, 00,00,00,00, 00,00,00,00
HKR, Responses, "<lf>",    1, 01, 00, 00,00,00,00, 00,00,00,00

Note  Use padded responses rather than stripped responses in modem INF Responses entries. Unimodem can process padded responses faster and more reliably than stripped responses. Also, the additional entries required for stripped responses make the INF file larger.

Use ProCalc to create Responses entries, as described in Creating Data Modem INF File Entries with ProCalc.
