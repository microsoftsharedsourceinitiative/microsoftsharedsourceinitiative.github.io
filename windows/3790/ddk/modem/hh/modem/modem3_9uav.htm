Disabling Dial Suffix

The dial suffix should cause the modem to return to command state after dialing. The DialSuffix entry is a string of characters (generally “;”) that is appended to any dial string before the terminator character string (generally "<cr>"). The following example shows how to set the DialSuffix value:

HKR, Settings, DialSuffix,, ";"

Use a null DialSuffix if a dial suffix is not applicable. Use a null suffix for devices that inadvertently listen for dial tone at each dial command. The following example shows how to set the null DialSuffix:

HKR, Settings, DialSuffix,, ""

During staged dialing, several initialization strings are issued as Unimodem dial-out proceeds (for example, a number is dialed to get an outside line then another is dialed to connect, and so forth). Unimodem uses the dial suffix during its staged dialing process.

Some modem firmware contains a bug that causes a dial suffix to break all or part of any initialization that follows the dial suffix. Modem firmware that contains a dial suffix bug of this type should be fixed, even though it is possible to disable dial suffix as a workaround within an INF file.

To disable dial suffix, do not include the DialSuffix entry in the INF file. If the dial suffix is disabled, the modem might not be able to dial long numbers or calling card numbers.

Some voice modems support staged dialing in data mode but not in voice mode. Set the flag in the VoiceProfile entry if the dial string must always end with DialSuffix in voice mode. For more information about the VoiceProfile entry, see Adding Voice Support.
