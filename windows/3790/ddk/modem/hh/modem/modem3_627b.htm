Disabling SwitchType Entries

If you are writing an INF file for CAPI cards and want to configure SwitchType, MSN, DN, EAZ or SPID using your CAPI subsystem, do not include any of the registry entries of type ISDN\SwitchType. The corresponding Unimodem B channel protocol selection entries will still function.

If you bundle setup applications with your devices that support automatic SPID detection and verification, do not include any of the entries of type ISDN\SwitchType. The affected Unimodem D channel protocol selection keys will still function.
