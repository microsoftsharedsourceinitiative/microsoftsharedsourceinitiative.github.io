Modem Testing Using a Terminal Application

You can use HyperTerminal or any other terminal communications application for simple initialization, AT command testing, and debugging.

To use HyperTerminal

  1. Close NDIS Test if it is running.
  2. Start a HyperTerminal session to access the modem directly.
  3. Issue AT commands until you find one or more commands that do not produce an OK response from the modem.
  4. Modify these commands in the INF file, or remove them entirely.
  5. Uninstall the modem from Control Panel and reinstall it again so that the newly modified INF file is in use.
  6. Rerun NDIS Test.

Note  To speed testing, cut and paste initialization strings from the modem INF file into HyperTerminal.
