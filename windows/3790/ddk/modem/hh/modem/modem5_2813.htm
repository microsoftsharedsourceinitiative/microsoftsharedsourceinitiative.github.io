Adding Support for Voice to Data Answering Switch

The VoiceToDataAnswer entries are a sequence of commands that Unimodem sends to the modem to switch it to data answering mode after it has been in voice answering mode. The following example demonstrates a sequence of commands for switching voice to data mode:

HKR, VoiceToDataAnswer,    1,, "at#cls=0<cr>"    (select data)
HKR, VoiceToDataAnswer,    2,, "ata<cr>"         (answer)

Note  Unless the voice features are disabled, Unimodem answers all incoming calls on voice modems as voice calls.
