220 - Implicit cast between semantically different integer types.

Additional Information: Comparing HRESULT to -1.

Recommended Fix: Consider using SUCCEEDED or FAILED macro instead.

Note   For this warning, the SCODE type is equivalent to HRESULT.

This warning indicates that an HRESULT is being compared against an explicit, non-HRESULT value of -1, which is not a well-formed HRESULT.

It is generally best to use the SUCCEEDED or FAILED macro to test the value of an HRESULT.

In most cases, this warning is caused by the code mistakenly expecting that a function that actually returns an HRESULT instead returns an integer (using –1 as a failure value). If the actual intent is to check specifically for the E_FAIL HRESULT (for example, ignoring other possible failure values), then an explicit comparison to E_FAIL is appropriate. Otherwise, use the SUCCEEDED or FAILED macros.

Example

Defective Source

if (SomeFunction() == -1) {
    return E_FAIL;
}

return S_OK;

Corrected Source

if (FAILED(SomeFunction())) {
    return E_FAIL;
}

return S_OK;
