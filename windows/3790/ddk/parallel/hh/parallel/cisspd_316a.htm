Client Interfaces to System-Supplied Parallel Drivers

This section describes the client interface that the system-supplied parallel drivers provide for operating parallel ports and devices attached to parallel ports. In particular, vendor-supplied parallel drivers use this interface to operate parallel devices attached to parallel ports.

The system-supplied function driver for parallel ports supports an interface consisting of the following components:

I/O Requests for Parallel Ports

Internal Device Control Requests for Parallel Ports

Parallel Port Callback Routines

Parallel Port Data Types

The system-supplied bus driver for parallel ports supports an interface consisting of the following components:

I/O Requests for Parallel Devices

Device Control Requests for Parallel Devices

Internal Device Control Requests for Parallel Devices

Parallel Device Callback Routines

Parallel Device Data Types

For more information about Windows support for parallel ports and devices attached to parallel ports, see:

Introduction to Parallel Ports and Devices

System-Supplied Parallel Drivers

Vendor-Supplied Parallel Drivers

Sample Parallel Drivers

Note  This documentation does not apply to Microsoft® Windows® Me. Windows Me uses the Windows 95 Virtual Communication Driver model to support parallel ports. For more information about Windows Me support for parallel ports, see information about virtual communications drivers in the Windows 95 documentation section of the Windows DDK.
