IOCTL_DOT4_DESTROY_SOCKET

Operation

The IOCTL_DOT4_DESTROY_SOCKET request deletes a specified IEEE 1284.4 primary socket. See Dot4 Socket for more information about Dot4 sockets.

Input

Irp->AssociatedIrp.SystemBuffer is a pointer to a client-allocated DOT4_DRIVER_CMD structure, whose members must be set to zero.

The Parameters.DeviceIoControl.OutputBufferLength member of the IRP's current I/O stack location is set to a value greater than or equal to the size, in bytes, of a DOT4_DC_DESTROY_DATA structure.

Irp->MdlAddress is a pointer to a client-allocated MDL buffer that holds a DOT4_DC_DESTROY_SOCKET structure that inputs the following information:

bHsid
Specifies the socket ID of the IEEE 1284.4 primary socket to delete.

Output

None.

I/O Status Block

The Information member is set to the size, in bytes, of a DOT4_DC_DESTROY_DATA structure.

The Status member is set to one of the generic status values for Dot4 IOCTLS or to one of the following request-specific values:

STATUS_BUFFER_TOO_SMALL
The Parameters.DeviceIoControl.OutputBufferLength member of the IRP's current I/O stack location is less than the size, in bytes, of a DOT4_DC_DESTROY_DATA structure.
STATUS_NO_MEMORY
The system did not successfully map the client-allocated MDL buffer for the DOT4_DC_DESTROY_DATA structure that inputs information.
STATUS_UNSUCCESSFUL
A Dot4 channel is open on the specified primary socket, or the socket ID is invalid.

Headers

Defined in d4drvif.h. Include d4drvif.h.
