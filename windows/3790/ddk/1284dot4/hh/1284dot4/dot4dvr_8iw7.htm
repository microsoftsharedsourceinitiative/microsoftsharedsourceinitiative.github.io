Dot4 Driver Operation

This section includes the following topics about the operation of the Dot4 driver:

  • Dot4 Session

    To operate a Dot4 device, a client starts a Dot4 session (or joins a running session), and then uses a sequence of device control requests that specify Dot4 Function I/O control codes.

  • Dot4 Function

    A Dot4 function is a generic name for an independent function supported by a Dot4 device.

  • Dot4 Service

    The functionality of Dot4 device is exported as a set of Dot4 services.

  • Dot4 Socket

    A Dot4 socket represents a communication endpoint of a Dot4 channel.

  • Dot4 Channel

    A Dot4 channel is a bidirectional communication link that a client can use to access a Dot4 service.

  • Dot4 Data Link

    The Dot4 driver establishes an IEEE 1284.4 data link with a Dot4 device.

  • Reading and Writing Over a Dot4 Channel

    Clients use IOCTL_DOT4_READ and IOCTL_DOT4_WRITE I/O control codes to read and write data over a Dot4 channel.

  • Dot4 Activity Broadcasting

    A client can use Dot4 activity broadcasting to be notified after certain events, or activities, occur on a Dot4 channel.

  • Dot4 Flow Control Policy

    Dot4 supports a credit-based flow control policy, which a client can set, that partly regulates the flow of data over Dot4 channels during a Dot4 session.

  • Dot4 Timeouts

    Dot4 supports timeout values, which a client can set, that partly control the operation of a Dot4 session.
