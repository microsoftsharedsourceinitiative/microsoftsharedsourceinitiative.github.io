Reading and Writing Over a Dot4 Channel

A client uses IOCTL_DOT4_READ and IOCTL_DOT4_WRITE requests to read and write data over a Dot4 channel. The Dot4 driver does not support IRP_MJ_READ and IRP_MJ_WRITE requests because these requests do not provide a way to specify a channel.

The Dot4 driver supports two types of read requests:

  • Read-bytes request

    Copies data from a channel to a client-allocated read buffer. The Dot4 driver uses direct I/O to copy the data from the channel to the read buffer.

  • Query-read-bytes request

    Determines the number of data bytes that are available on a channel.

The Dot4 driver supports two types of write requests:

  • Write-bytes request

    Copies data to a channel from a client-allocated write buffer. The Dot4 driver uses direct I/O to copy the data from the write buffer to the channel.

  • Query-write-bytes request

    Determines the number of data bytes that a channel is ready to accept.

A client specifies read and write information using a DOT4_DRIVER_CMD structure.
