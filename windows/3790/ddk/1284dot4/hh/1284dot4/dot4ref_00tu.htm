IEEE 1284.4 Devices: Windows DDK

IOCTL_DOT4_OPEN_CHANNEL

Operation

The IOCTL_DOT4_OPEN_CHANNEL request opens a Dot4 channel for a specified Dot4 service.

The client can specify that the Dot4 driver add Dot4 activity broadcasting immediately after the channel is opened.

Input

Irp->AssociatedIrp.SystemBuffer is a pointer to a client-allocated DOT4_DRIVER_CMD structure, whose members must be set to zero.

The Parameters.DeviceIoControl.InputBufferLength member of the IRP's current I/O stack location is set to the size, in bytes, of a DOT4_DRIVER_CMD structure.

Irp->MdlAddress is a pointer to a client-allocated MDL buffer for a DOT4_DC_OPEN_DATA structure that inputs and outputs information.

The client inputs the following information:

bHsid
Specifies an IEEE 1284.4 primary socket ID, which is obtained by using an IOCTL_DOT4_CREATE_SOCKET request.
fAddActivity
Specifies whether to add activity broadcasting on a Dot4 channel immediately after the channel is opened. If fAddActivity is set to TRUE (nonzero), the Dot4 driver adds activity broadcasting; otherwise, activity broadcasting is not added.

The Parameters.DeviceIoControl.OutputBufferLength member of the IRP's current I/O stack location is set to the size, in bytes, of a DOT4_DC_OPEN_DATA structure.

Output

Irp->MdlAddress is a pointer to the client-allocated MDL buffer for the DOT4_DC_OPEN_DATA structure that outputs the following information:

hChannelHandle
Specifies a channel handle.

I/O Status Block

The Information member is set to the size, in bytes, of a DOT4_DC_OPEN_DATA structure.

The Status member is set to one of the generic status values for Dot4 IOCTLS or to one of the following values:

STATUS_BUFFER_TOO_SMALL
The Parameters.DeviceIoControl.OutputBufferLength member of the IRP's current I/O stack location is less than the size, in bytes, of a DOT4_DC_OPEN_DATA structure.
STATUS_NO_MEMORY
The system did not successfully map the client-allocated MDL buffer for the DOT4_DC_OPEN_DATA structure that inputs and outputs information.
STATUS_UNSUCCESSFUL
One of the following errors occurred:
  • The specified primary socket ID or associated channel is invalid.
  • The channel is already open.

Headers

Defined in d4drvif.h. Include d4drvif.h.
