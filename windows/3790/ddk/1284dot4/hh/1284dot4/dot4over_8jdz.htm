Installing a Dot4 Scanner Function

A vendor only has to supply an INF file for the Dot4 scanner function. The sample INF file dot4oems.inf describes the sections that install a Dot4 scanner function.

The following table summarizes the device setup classes and example hardware IDs for a Dot4 scanner function, as specified in dot4oem.inf and dot4oems.inf:

Device Setup Class Enumerator Hardware ID
Image Dot4.sys DOT4\OEM_Company_1284.4_M9C9C&SCAN
Dot4 Parallel.sys LPTENUM\OEM_Company_1284.4_M9C9C

A client accesses an Image device by using the services of the WIA drivers. A client opens a device by using its symbolic link name, which has the format "\DosDevices\Parscan<n>", where n is an integer greater than or equal to zero.

The dot4oems.inf INF file includes the following AddReg directive that specifies default service configuration information for a Dot4 scanner function:

[Dot4Scan_AddReg]
;
; Add keys under Enum\enumeratorID\device-instance-id
;
;       These entries in the registry control
HKR,,"ReadSocketId",0x00010001,0x0004
HKR,,"WriteSocketId",0x00010001,0x0004
HKR,,"ReadBufferSize",0x00010001,0x7FFF
HKR,,"WriteBufferSize",0x00010001,0x7FFF

The meaning of these settings is the same as that described in Service Configuration Settings for a Dot4Print Device. Note, however, that the default values for a Dot4 scanner function and a Dot4Print device are different.
