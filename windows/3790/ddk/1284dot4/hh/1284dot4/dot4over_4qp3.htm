IEEE 1284.4 Devices: Windows DDK

Dot4 Device Setup Class

Microsoft defines the Dot4 device setup class for the parent device that controls the operation of a multifunction IEEE 1284.4 peripheral device. Note that in the DDK, Dot4 refers to the Dot4 device setup class, not to the IEEE specification IEEE P1284.4, Standard for Data Delivery and Logical Channels for IEEE Std. 1284 Interfaces.

An IEEE 1284.4-compliant device can:

  • Export its functionality as a set of services, which a client accesses over channels.
  • Communicate by using a protocol that supports:
    • Multiple, concurrent, independent exchanges of data and control information between peers over an IEEE 1284.4 data link.
    • A packet-based, non-blocking, flow control system.

See the IEEE 1284.4 specification for more information about IEEE 1284.4 compliant devices.

Microsoft provides the Dot4 driver, the system-supplied function and bus driver for Dot4 devices, in Windows 2000 and later. The Dot4 driver abstracts the IEEE 1284.4 protocol and operates a Dot4 device in the following way:

  • A Dot4 device is a Plug and Play parent device that supports one or more Plug and Play child devices, or Dot4 functions.
  • A Dot4 function supports one or more subfunctions, or Dot4 services.
  • To access a Dot4 service, a client opens a Dot4 function, creates a Dot4 socket for the service, and opens a Dot4 channel on the socket.

To install a Dot4 device, a vendor must:

  • Publish the name and IEEE 1284.4 secondary socket ID of each service.
  • Supply an INF file for the parent Dot4 device and the INF files that are required to install the child Dot4 functions supported by the Dot4 device — see Installing a Dot4 Device.

An example of a Dot4 device is a multifunction device that supports one or more of the following functional capabilities: printing, scanning, and faxing. The Dot4 driver represents each functional capability as an independent Dot4 function. Examples of Dot4 services include generic operations such as: data write, data read, configuration download, configuration upload, fax send, and fax receive.
