DOT4_ACTIVITY

A DOT4_ACTIVITY structure represents an activity, or event, that occurs on a Dot4 channel.

typedef struct _DOT4_ACTIVITY {
  ULONG  ulMessage;
  ULONG  ulByteCount;
  CHANNEL_HANDLE  hChannel;
} DOT4_ACTIVITY, *PDOT4_ACTIVITY;

Members

ulMessage
Specifies one of the following activity types:
Activity Type Meaning
DOT4_CHANNEL_CLOSED The channel is closed.
DOT4_DISCONNECT The Dot4 data link is not connected.
DOT4_MESSAGE_RECEIVED A message was received on a channel.
DOT4_STREAM_CREDITS Credits are available to send data over a stream-type channel.
DOT4_STREAM_RECEIVED Stream data was successfully received on a channel.
ulByteCount
Specifies one of the following message-specific values:
Activity Type Value
DOT4_DISCONNECT and DOT4_CHANNEL_CLOSED The value is set to zero.
DOT4_MESSAGE_RECEIVED and DOT4_STREAM_RECEIVED The number of bytes received.
DOT4_STREAM_CREDITS The number of credits issued by a Dot4 device.
hChannel
Specifies the handle of the channel on which the activity took place. A DOT4_DISCONNECT message returns a channel handle of zero.

Headers

Declared in d4drvif.h. Include d4drvif.h.

Comments

A client uses an IOCTL_DOT4_ADD_ACTIVITY_BROADCAST request to start Dot4 activity broadcasting. If activity broadcasting is activated, the client uses an IOCTL_DOT4_WAIT_ACTIVITY_BROADCAST request to be notified after an activity occurs on the channel. The Dot4 driver returns a DOT4_ACTIVITY structure that describes the activity.
