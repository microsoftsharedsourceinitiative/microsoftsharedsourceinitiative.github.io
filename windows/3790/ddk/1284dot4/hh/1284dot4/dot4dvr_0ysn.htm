Dot4 Policy Registry Key

The following Policy key is located under a Dot4 device's Plug and Play hardware device key:

Policy key
Contains the following flow control entry values:
CreditOnZeroLength entry value (REG_DWORD)
Boolean flag that specifies whether the Dot4 driver can issue credits to a peripheral by sending only a header packet. The default value is FALSE (zero).
PiggyBackCredit entry value (REG_DWORD)
Boolean flag that specifies whether the Dot4 driver can piggyback credits when it sends a header packet to a peripheral over a stream channel. The default value is FALSE (zero).
AskCreditAfterWrite entry value (REG_DWORD)
Boolean flag that specifies whether the Dot4 driver requests host-to-peripheral credits if Dot4 is out of credits after writing a data packet. The default value is TRUE (nonzero).
CommandPacketTwoParts entry value (REG_DWORD)
Boolean flag that specifies whether the Dot4 driver sends command packets over the Transaction Channel in two parts (that is, it uses separate, consecutive writes of the header and data). The default value is TRUE (nonzero).

The Dot4 driver creates a Policy key if one does not exist. If an entry value does not exist under the Policy key, the Dot4 driver creates the entry value and sets it to the specified default value.
