Dot4 Channel

A Dot4 channel provides a logical conduit for bidirectional communication between a client and a Dot4 device. The client uses a Dot4 channel to access a Dot4 service. Dot4 channels operate concurrently and independently of one another, and the transmission of data is flow-controlled. A Dot4 channel can be a packet-type channel or a stream-type channel, which is determined by the type of IEEE 1284.4 primary socket that was created for the channel.

A client opens a Dot4 channel on a Dot4 socket by using an IOCTL_DOT4_OPEN_CHANNEL request. A client uses a DOT4_DC_OPEN_DATA structure to input and output information.

A client can wait for a Dot4 device to open a Dot4 channel by using an IOCTL_DOT4_WAIT_FOR_CHANNEL request.

A client closes a Dot4 channel by using an IOCTL_DOT4_CLOSE_CHANNEL request. A close channel request is normally followed by an IOCTL_DOT4_DESTROY_SOCKET request.
