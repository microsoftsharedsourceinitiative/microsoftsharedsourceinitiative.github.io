IEEE 1284.4 Devices: Windows DDK

Dot4 Service

The functionality of a Dot4 device is exported as a set of Dot4 services. A vendor must document the service names and associated IEEE 1284.4 secondary socket IDs that a Dot4 device supports.

To access a service, a client uses the following I/O requests:

  • IRP_MJ_CREATE

    An IRP_MJ_CREATE request, which specifies write access, opens a Dot4 function that supports a service.

  • IOCTL_DOT4_CREATE_SOCKET

    An IOCTL_DOT4_CREATE_SOCKET request obtains an IEEE 1284.4 primary socket ID for the service. A client uses a vendor-documented service name or IEEE 1284.4 secondary socket ID to specify the service. If the client specifies the service name, the secondary socket ID is not used. A client uses the primary socket ID to open a Dot4 channel for the service.

  • IOCTL_DOT4_OPEN_CHANNEL

    An IOCTL_DOT4_OPEN_CHANNEL request opens a Dot4 channel for the service.

  • Dot4 function I/O control codes

    Dot4 function I/O control codes control the operation of the Dot4 channel.

To disconnect from the service, a client uses an IOCTL DOT4_CLOSE_CHANNEL request. This request is normally followed by an IOCTL_DOT4_DESTROY_SOCKET request.

A client can discover what services are defined for a device by enumerating the service names under the Services key of the device's Plug and Play hardware device key. See Dot4 Services Registry Key and Installing a Dot4 Device for more information.
