DOT4_DC_CREATE_DATA

The DOT4_DC_CREATE_DATA structure specifies information that Dot4 driver uses to create an IEEE 1284.4 primary socket.

typedef struct _DOT4_DC_CREATE_DATA
{
  unsigned char  bPsid;
  CHAR  pServiceName[MAX_SERVICE_LENGTH + 1];
  unsigned char  bType;
  ULONG  ulBufferSize;
  USHORT  usMaxHtoPPacketSize;
  USHORT  usMaxPtoHPacketSize;
  unsigned char bHsid;
} DOT4_DC_CREATE_DATA, *PDOT4_DC_CREATE_DATA;

Members

bPsid
Specifies a well-known IEEE 1284.4 secondary socket ID (that is, a socket ID that a device vendor documents). If a Dot4 service name is specified, the socket ID is not used. A socket ID of zero is associated with an IEEE 1284.4 Transaction Channel, and is invalid if the socket ID is used.
pServiceName
Specifies an array that contains the service name (a NULL-terminated ANSI string) of a secondary socket. If a service name is specified, the socket ID is not used.
bType
Specifies the type of socket, which is either STREAM_TYPE_CHANNEL or PACKET_TYPE_CHANNEL.
ulBufferSize
Specifies the size, in bytes, of the socket's internal read buffer. The Dot4 driver sets the internal buffer size to the larger of ulBufferSize and the peripheral-to-host packet size that the Dot4 driver sets for the socket. The ulBufferSize member should be set to zero for a write-only socket.
usMaxHtoPPacketSize
Specifies the maximum size, in bytes, of a packet that can be sent from the host to the peripheral. The Dot4 driver negotiates the largest packet size that the peripheral supports, bounded by the maximum requested by the client.
usMaxPtoHPacketSize
Specifies the maximum size, in bytes, of a packet that can be sent from the peripheral to the host. The Dot4 driver negotiates the largest packet size that the peripheral supports, bounded by the maximum requested by the client.
bHsid
Specifies the IEEE 1284.4 primary socket ID that a client uses to open a Dot4 channel. The Dot4 driver sets this value.

Headers

Declared in d4drvif.h. Include d4drvif.h.

Comments

A client uses a DOT4_DC_CREATE_DATA structure to input and output information with an IOCTL_DOT4_CREATE_SOCKET request. A client sets all the members except the bHsid member, and the Dot4 driver sets bHsid.
