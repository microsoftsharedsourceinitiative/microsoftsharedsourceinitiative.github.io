IOCTL_MOUNTMGR_VOLUME_MOUNT_POINT_DELETED

Operation

Mount Manager clients use this IOCTL to alert the Mount Manager that a volume mount point has been created so that the Mount Manager can replicate the database entry for the given mount point.

The Win32® routine DeleteVolumeMountPoint sends this IOCTL to the Mount Manager, in order to inform the Mount Manager that a directory junction is no longer pointing to a volume name. The Mount Manager responds by deleting the volume name formerly contained in the directory junction along with its unique ID from the volume hosting the directory junction.

Input

Parameters.DeviceIoControl.InputBufferLength in the I/O stack location of the IRP indicates the size, in bytes, of the input buffer, which must be greater than or equal to sizeof(MOUNTMGR_VOLUME_MOUNT_POINT).

The Mount Manager client initializes the MOUNTMGR_VOLUME_MOUNT_POINT structure, defined in mountmgr.h, at the beginning of the buffer at Irp->AssociatedIrp.SystemBuffer.

Output

None

I/O Status Block

If the operation is successful, the Status field is set to STATUS_SUCCESS.

If InputBufferLength is less than sizeof(MOUNTMGR_VOLUME_MOUNT_POINT), the Status field is set to STATUS_INVALID_PARAMETER.

Headers

Defined in mountmgr.h. Include mountmgr.h.
