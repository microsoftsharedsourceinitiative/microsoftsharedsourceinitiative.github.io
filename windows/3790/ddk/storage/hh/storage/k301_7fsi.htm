StorPortAllocateRegistryBuffer

The StorPortAllocateRegistryBuffer routine is called by the miniport to allocate a buffer that can be used to read and write registry data. 

PUCHAR
StorPortAllocateRegistryBuffer (
  IN PVOID  HwDeviceExtension,
  IN PULONG  Length
  );

Parameters

HwDeviceExtension
Pointer to the hardware device extension. This is a per-HBA storage area that the port driver allocates and initializes on behalf of the miniport driver. Miniport drivers usually store HBA-specific information in this extension, such as the state of the HBA and the HBA's mapped access ranges. This area is available to the miniport driver in the DeviceExtension->HwDeviceExtension member of the HBA's device object immediately after the miniport driver calls StorPortInitialize. The port driver frees this memory when it removes the device.
Length
Pointer to a long that indicates, on input, the length in bytes of the registry buffer that the caller requires. On output, Length is updated to reflect the actual size in bytes of the buffer that was allocated.

Return Value

Pointer to the registry buffer that the caller requested, or NULL if some error prevents the memory from being allocated.

Headers

Declared in storport.h. Include storport.h.

Comments

Each instantiation of a miniport can only have one registry buffer allocated at a time. If a miniport attempts to allocate more than one registry buffer, the allocation will fail and StorPortAllocateRegistryBuffer will return NULL.
