IOCTL_CHANGER_SET_POSITION

Operation

Sets the changer's robotic transport mechanism to the specified element address, typically to optimize moving or exchanging media by positioning the transport beforehand.

Input

Parameters.DeviceIoControl.InputBufferLength in the I/O stack location indicates the size in bytes of the parameter buffer, which must be >= sizeof(CHANGER_SET_POSITION). The buffer at Irp->AssociatedIrp.SystemBuffer contains a CHANGER_SET_POSITION structure that specifies the transport to move and the destination. If the Flip member is TRUE and the device supports two-sided media, the media currently carried by the transport should be flipped.

Output

None

I/O Status Block

The Information field is set to sizeof(CHANGER_SET_POSITION). The Status field is set to STATUS_SUCCESS, or possibly to STATUS_INFO_LENGTH_MISMATCH, STATUS_INSUFFICIENT_RESOURCES, STATUS_INVALID_DEVICE_REQUEST, or STATUS_INVALID_PARAMETER.

Headers

Defined in ntddchgr.h. Include ntddchgr.h.
