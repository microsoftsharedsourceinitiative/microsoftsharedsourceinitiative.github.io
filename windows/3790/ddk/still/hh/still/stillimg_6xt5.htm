Still Image Devices: Windows DDK

Still Image Drivers

The Windows DDK update ships a new architecture called Windows Image Acquisition (WIA). WIA is built on the foundation of the Microsoft® Still Image architecture (Microsoft STI), thus drivers built on STI easily migrate to WIA.

This section is provided for legacy drivers developed on Microsoft STI. It describes the COM interfaces that are defined by Microsoft STI and available to vendors of such still image hardware as flatbed scanners and digital still image cameras. The COM interfaces described in this section are called by, or defined by, the following vendor-supplied software:

  • Device-specific components of image acquisition API software, such as the TWAIN data sources that are required so the TWAIN API can support specific still image devices.
  • User-mode still image minidrivers, which provide a communications path from image acquisition software to lower-level device and bus drivers.
