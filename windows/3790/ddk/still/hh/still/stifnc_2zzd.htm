USB Still Image Structures

This section describes the structures associated with I/O control codes that are recognized by the kernel-mode still image driver for USB buses. The structures are defined in usbscan.h.

If you are developing still image drivers for Windows 98, Windows 98 SE, or Windows Me, be sure to specify that the structures defined in usbscan.h are packed on 8-byte boundaries. This alignment is required for Windows 98/Me, and is the default for DDK builds. You can change the packing alignment using the /Zp compiler switch or with a "#pragma pack" preprocessor directive. (See the Platform SDK documentation for more information about compiler switches and #pragma directives.)
