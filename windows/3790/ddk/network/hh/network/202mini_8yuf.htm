Indicating Receives with NdisM(Co)IndicateReceivePacket

When a connectionless miniport driver calls NdisMIndicateReceivePacket or a connection-oriented miniport driver calls NdisMCoIndicateReceivePacket, the miniport driver passes a pointer to an array of packet descriptors, each of which describes a packet. Each packet that is indicated up must be completed separately by the miniport driver.

Connection-oriented miniport drivers identify the virtual connection (VC) on which the packet(s) have been received. At the request of the connection-oriented call manager or integrated miniport call manager (MCM) that received the incoming call from the network, the miniport driver previously set up and activated the VC in its MiniportCoCreateVc and MiniportCoActivateVc functions, respectively. The miniport driver passes an NDIS-supplied handle that identifies the VC in its call to NdisMCoIndicateReceivePacket.

For a detailed description of indicating up receive data with NdisM(Co)IndicateReceivePacket , see Indicating Receive Packets with NdisM(Co)IndicateReceivePacket
