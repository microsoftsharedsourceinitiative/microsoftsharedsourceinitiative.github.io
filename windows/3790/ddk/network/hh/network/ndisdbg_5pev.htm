Values For NDIS Debugging Variables in Windows NT 4.0

You can enable different levels of debug tracing in various NDIS components by using the following values for NDIS debugging variables:

ndisDebugLevel
Selects the level or amount of debug tracing in NDIS components that are selected with the ndisDebugSystems value. The following values specify levels that you can select:
Level Description Value
DBG_LEVEL_INFO All available debug information. This is the highest level of trace. 0x00000000
DBG_LEVEL_WARN Warnings. 0x00001000
DBG_LEVEL_ERR Errors. 0x00002000
DBG_LEVEL_FATAL Fatal errors, which can cause the operating system to crash. This is the lowest level of trace. 0x00003000

ndisDebugSystems
Selects various NDIS components to debug trace. The following values specify the NDIS components that you can select:
Component Description Value
DBG_COMP_INIT Handles adapter initialization. 0x00000001
DBG_COMP_CONFIG Handles adapter configuration. 0x00000002
DBG_COMP_SEND Handles sending data over the network. 0x00000004
DBG_COMP_RECV Handles receiving data from the network. 0x00000008
DBG_COMP_MEMORY Handles memory management. 0x00000010
DBG_COMP_FILTER Handles filter operations. 0x00000020
DBG_COMP_PROTOCOL Handles protocol operations. 0x00000040
DBG_COMP_REQUEST Handles requests. 0x00000080
DBG_COMP_UNLOAD Handles unloading operations. 0x00000100
DBG_COMP_WORK_ITEM Handles work-item operations. 0x00000200
DBG_COMP_OPEN Handles operations that open reference objects. 0x00000400
DBG_COMP_LOCKS Handles locking operations. 0x00000800
DBG_COMP_ALL Handles all NDIS components. 0xFFFFFFFF

You can select more than one NDIS component. If more than one component is selected, OR the data values together.
