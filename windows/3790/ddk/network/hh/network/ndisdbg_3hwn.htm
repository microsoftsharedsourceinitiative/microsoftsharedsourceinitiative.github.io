Loading NDIS Debugger Extensions

Before you can use NDIS Debugger extensions, the debugger you are using must load the NDIS debugger DLL. The debugger can either load the NDIS debugger DLL explicitly with the debugger's own load command, or implicitly when the debugger first runs one of the NDIS debugger DLL's extension commands. The following procedures show both methods of loading NDIS Debugger extensions:

To explicitly load the NDIS Debugger DLL

  1. At the debugger's command prompt, enter the following command: 
    !load ndiskd.dll
  2. The debugger loads Ndiskd.dll but does not run an extension.

To implicitly load the NDIS Debugger DLL

  1. At the debugger's command prompt, enter the following command: 
    !ndiskd.extension_name [arguments]
  2. The debugger loads Ndiskd.dll, calls a DLL entry point to run a specific extension command, and passes arguments to that extension.

Once the debugger loads the NDIS debugger DLL, you can specify any of the NDIS debugger DLL's extensions at the debugger's command prompt to run that extension command as shown in the following example:

!extension_name

Note  When you run an extension without also specifying Ndiskd.dll, the debugger searches for the extension in the extension DLL that the debugger most recently loaded. If the debugger does not find the extension, the debugger searches through extension DLLs in the order that they were loaded and runs the first instance of the extension that the debugger finds.
