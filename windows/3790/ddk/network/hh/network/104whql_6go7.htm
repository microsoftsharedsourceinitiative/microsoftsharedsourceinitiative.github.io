Prohibited Kernel-Mode Routines for NDIS-WDM Drivers

NDIS-WDM drivers should not call some kernel-mode support routines, but not because of WHQL restrictions. NDIS-WDM drivers should not call the following kernel-mode support routines because calling these routines directly interferes with NDIS support for Plug and Play:

  • IoCreateDevice
  • IoCreateSymbolicLink

Rather than call these kernel-mode support routines, NDIS-WDM drivers should call the NdisMRegisterDevice function instead.
