Handling Microsoft Extensions to Windows Sockets

The Windows Sockets switch handles all Microsoft-specific Windows Sockets extension functions internally. The Windows Sockets documentation in the Platform SDK defines an extension as a mechanism that exposes advanced transport functionality to application programs. These extension functions are: TransmitFile, AcceptEx, and GetAcceptExSockAddrs. The switch converts these calls, as necessary, and forwards them to the appropriate SAN service provider function: WSPSend, WSPAccept, WSPRdmaWrite, or WSPRdmaRead.
