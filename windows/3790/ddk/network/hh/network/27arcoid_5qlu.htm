ARCNET Objects

This section describes the OIDs for ARCNET objects, both operational and statistical types.

The ARCNET OIDs are defined in ntddndis.h. When building a driver that supports these OIDs, include ntddndis.h.

These OIDs are described in the following topics:

ARCNET Operational Characteristics

ARCNET Statistics

ARCNET Operational Characteristics

The following chart summarizes the OIDs used to get operational characteristics for NDIS drivers and NICs on an ARCNET network.

Length Q S Name
2 M   OID_ARCNET_PERMANENT_ADDRESS
Permanent station address
2 M   OID_ARCNET_CURRENT_ADDRESS
Current station address

The following describes each of the OID_XXX listed in the preceding chart more fully.

OID_ARCNET_PERMANENT_ADDRESS

The address of the NIC encoded in the hardware.

OID_ARCNET_CURRENT_ADDRESS

The address the NIC is currently using.

ARCNET Statistics

The following chart summarizes the OID used to get ARCNET NIC statistics.

Length Q S Name
4 O   OID_ARCNET_RECONFIGURATIONS
Number of reconfigurations

The following describes the OID_XXX listed in the preceding chart more fully.

OID_ARCNET_RECONFIGURATIONS

The OID_ARCNET_RECONFIGURATIONS OID returns the number of network-reconfiguration interrupts received by the NIC driver for an ARCNET card since the system was started.

A reconfiguration takes place every time an ARCNET card is connected to or disconnected from the network. A reconfiguration also takes place every time an ARCNET card is reset. Therefore, each time an ARCNET card is reset on, added to, or removed from the network, every other NIC on the ARCNET network will augment its reconfiguration count.
