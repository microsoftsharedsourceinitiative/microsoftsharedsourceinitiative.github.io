Registering the Miniport Driver

After the DriverEntry function has called NdisMInitializeWrapper to create an association with the NDIS Library, it must initialize a structure of type NDIS_MINIPORT_CHARACTERISTICS. The NDIS_MINIPORT_CHARACTERISTICS structure specifies the version of NDIS that is compatible with the miniport driver (see Specifying the NDIS Version Number) and the entry points of the required and optional upper-edge (MiniportXxx) functions that the miniport driver exports (see Registering MiniportXxx Functions). DriverEntry then calls NdisMRegisterMiniport with a pointer to the NDIS_MINIPORT_CHARACTERISTICS structure.
