Receiving Data in an Intermediate Driver with a Connection-Oriented Lower Edge

An underlying connection-oriented miniport driver indicates packets by calling NdisMCoIndicateReceivePacket, passing a pointer to an array of pointers to one or more packet descriptors. If an intermediate driver is layered above the miniport driver, NDIS then calls the intermediate driver's ProtocolCoReceivePacket function.

For more information about receiving data in an intermediate driver with a connection-oriented lower edge, see Receiving Data in a Connection-Oriented Protocol Driver.
