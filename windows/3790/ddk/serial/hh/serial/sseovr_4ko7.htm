Read and Write Timeouts for a COM port

A client can use an IOCTL_SERIAL_SET_TIMEOUTS request to set time-out values that Serial uses for read and write requests. Serial continues to transfer bytes until the requested number of bytes are transferred or a time-out event occurs.

Serial's time-out operation is compliant with the user-mode operation of COM ports that the Win32 Communications API provides.

Note that the time-out operation is not applied to a pending request while it is queued. The time-out operation is applied to a request after the request becomes current (that is, Serial starts to process the request).

For more information on the read and write time-outs, see the following:

  • SetCommTimeouts function and the COMMTIMEOUTS structure in the Platform SDK documentation
  • SERIAL_TIMEOUTS structure in the include file %install directory%\inc\ntddser.h in the Windows 2000 DDK
