Windows 95 Multilingual IME

Windows 98 Multilingual IME Release Notes

Updated files may be found on the Windows® 98 DDK in the \src\IME folder. This includes a current sample project (fakeime) and additional notes for both user-level (SDK) and kernel-level (DDK) implementations.

In Windows® 95, IMEs are provided as dynamic link libraries (DLLs), in contrast to IMEs of Windows® 3.1 Asian Edition, and each IME runs as one of the multilingual keyboard layouts. Compared to the Windows 3.1 IME, the Windows 95 multilingual IMEs provide the following advantages:

  • Run as a component of the multilingual environment
  • Provide multiple input contexts for each application task
  • Provide one active IME for each application thread
  • Give information to an application through its message loop (message order is not changed)
  • Provide strong support for both IME-unaware and IME-aware applications

To take full advantage of IME, an application needs to support the Windows 95 IME application I/F.

This documentation describes the application I/F of the Windows 95 IME architecture.
