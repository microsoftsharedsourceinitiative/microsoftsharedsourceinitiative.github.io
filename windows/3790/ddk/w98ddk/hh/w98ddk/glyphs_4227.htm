CMColorInfo

The CMColorInfo function converts a logical color (a ColorRef value) to a physical color (a physical color value), or converts a physical color to a logical color, depending on the value of the lpPColor parameter.

COLORREF WINAPI
  CMColorInfo(
    LPPDEVICE  lpDestDev,
    DWORD  dwColorin,
    LPPCOLOR  lpPColor,
    HCMTRANSFORM  hcmTransform
    );

Parameters

lpDestDev
Points to a PDEVICE or PBITMAP structure specifying the destination device or bitmap.
dwColorin
Specifies either a logical or physical color, depending on the value of the lpPColor parameter.
lpPColor
Points to a PCOLOR structure or is NULL. If lpPColor points to a PCOLOR structure, the function assumes that dwColorin specifies a logical color (an RGB value) and copies the physical color value that most closely matches the logical color to this structure. If lpPColor is NULL, the function assumes that dwColorin specifies a physical color and returns the logical color that matches the physical color.
hcmTransform
Handle of color transform to be used in transforming dwColorin.

Return Value

The return value is the logical color that most closely matches the physical color either specified by dwColorin, or copied to the PCOLOR structure pointed to by lpPColor.

Comments

The export ordinal for the CMColorInfo function is 33.

The CMColorInfo function is identical to ColorInfo. The added parameter refers to the transform to apply to the logical color in order to obtain the physical color.

When converting a logical color, CMColorInfo chooses the best possible physical color to match the logical color and returns the ColorQuad value that corresponds to this physical color.

GDI uses the physical colors returned by CMColorInfo to set text colors, background colors, and pixel colors using the Pixel function.

If dwColorin is a logical color, it will never specify color index.
