Multiple Outputs

For both single-instance and multi-instance drivers, the miniVDD handles GET_NUM_UNITS and creates multiple pDevices. The miniVDD uses VDD_GET_DISPLAY_CONFIG to get the unit number (diUnitNumber). One, both, or none of the devices can be active at once. If the specific hardware has restrictions on modes when both units are active, then ValidateModeEx needs to reflect this in the miniVDD.

Note  In this case, the user might have to detach a device to change modes.
