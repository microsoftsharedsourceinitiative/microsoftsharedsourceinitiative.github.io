PSET_WMI_DATAITEM

The PSET_WMI_DATAITEM routine is a callback into the driver to set the contents of a single data item within a data block. When the driver has finished filling the data block, it must call IoWMICompleteRequest to complete the IRP. The driver can return STATUS_PENDING if the IRP cannot be completed immediately.

typedef NTSTATUS (*PSET_WMI_DATAITEM) (
    IN PDEVICE_OBJECT DeviceObject,
    IN PIRP Irp,
    IN ULONG GuidIndex,
    IN ULONG InstanceIndex,
    IN ULONG DataItemId,
    IN ULONG BufferSize,
    IN PUCHAR Buffer
    );

Parameters

DeviceObject
Device whose data block is being set.
Irp
IRP that makes this request.
GuidIndex
Index into the list of GUIDs provided when the device registered.
InstanceIndex
Index that denotes which instance of the data block is being set.
DataItemId
Has the ID of the data item being set.
BufferSize
Size of the data item passed.
Buffer
New values for the data item.

Return Value

Returns a status code.
