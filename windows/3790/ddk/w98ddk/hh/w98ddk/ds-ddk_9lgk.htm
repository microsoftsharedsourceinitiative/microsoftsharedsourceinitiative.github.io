IDsDriver::QueryInterface

The IDsDriver::QueryInterface method determines whether the object supports a particular COM interface. If it does, the system increases the object's reference count by 1, and the application can use that interface immediately. 

HRESULT QueryInterface( 
  REFIID riid, 
  LPVOID FAR* ppvObj
 );

Parameters

riid
Reference identifier of the interface being requested.
ppvObj
Address of a pointer that will be filled with the returned interface pointer if the query succeeds.

Return Values

Returns DS_OK if successful, or one of the following error values otherwise:

DSERR_INVALIDPARAM

DSERR_NOINTERFACE

Remarks

When the application no longer needs to use the interface retrieved by a call to this method, it calls the Release method for that interface to free it. The QueryInterface method allows Microsoft and third parties to extend objects without interfering with existing or future functionality.

This method is part of the IUnknown interface inherited by the object.

This object must support IUnknown and IDsDriver. Both interfaces should be accessible through this method. Note, however, that IDsDriverBuffer should not be accessible through this method. Sound buffer objects are separate (although related) objects, so IDsDriverBuffer should only be accessible by calling IDsDriver::CreateSoundBuffer.
