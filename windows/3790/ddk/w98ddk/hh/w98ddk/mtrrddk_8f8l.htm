MTRRSetPhysicalCacheTypeRange

The MTRRSetPhysicalCacheTypeRange function sets a physical range to a particular cache type and requires the caller to be at nonhardware interrupt time. If the system does not support setting cache policies based on physical ranges, no action is taken.

Parameters

PhysicalAddress
Starting address of the range being set.
NumberOfBytes
Length, in bytes, of the range being set.
CacheType
Caching type for which the physical range is to be set to. The following values are provided.
Value Description
NonCached Setting ranges to be NonCached are done for bookkeeping reasons. A return of SUCCESS when setting a range as NonCached does not mean it has been physically set to NonCached. The caller must use a cache-disabled virtual pointer for any NonCached range.
Cached Successful return indicates that the physical range has been set to cache.
FrameBuffer Successful return indicates that the physical range has been set to be frame buffer cached.

Return Values

STATUS_SUCCESS
If successful, the cache attributes of the physical range have been set.
STATUS_NO_SUCH_DEVICE
FrameBuffer type has been requested but is not supported.
STATUS_UNSUCCESSFUL STATUS_INTERNAL_ERROR
Requested physical range is below 1 MB, or another error has occurred.
