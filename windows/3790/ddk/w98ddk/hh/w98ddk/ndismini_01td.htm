Architecture

The architecture of the NDIS WAN support in Windows® 95 is slightly different from Windows NT®/Windows® 2000 though all the differences are internal and should not affect a properly written driver. In Windows 95, the Connection Wrapper and the NDISWAN protocol have been combined into one VxD while the Dial Up Networking Adapter (PPPMAC.VXD) is used as the WAN Wrapper. This differs from Windows NT/Windows 2000 where the NDISWAN protocol is combined with the NDISWAN adapter (which serves as the WAN Wrapper), while the Connection Wrapper (NDISTAPI.SYS) functions as a separate driver.
