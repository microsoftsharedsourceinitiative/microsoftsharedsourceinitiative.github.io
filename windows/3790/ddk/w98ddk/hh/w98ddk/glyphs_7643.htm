USB and Parallel Port Printing Architecture on Windows Operating Systems

The following figure shows the system-supplied and vendor-supplied components that can be involved in printing over USB and parallel ports on Windows operating systems.

Windows USB and Parallel Port Printing Architecture

The only required vendor-supplied component is the high-level printer driver, which is in the layer above the spooler. A vendor can optionally supply a language monitor that is in the layer between the spooler and usbmon.dll (for USB) or localmon.dll (for parallel port).

The preceding figure shows a vendor-supplied kernel-mode USB print driver (labeled IHV-supplied USB Print Driver) that is in the layer above the USB driver stack, and which consists of usbhub.sys, usbd.sys, and some layers underneath. In the past, a vendor-supplied USB print driver was needed for USB printing on Windows 98 operating systems, which did not include an in-box kernel-mode USB print driver.

Printer vendors can now redistribute the Microsoft-supplied USB print driver and port monitor, usbprint.sys and usbmon.dll, for installation on Windows 98 systems. Microsoft is encouraging vendors to redistribute usbprint.sys and usbmon.dll rather than writing their own USB print drivers and port monitors.
