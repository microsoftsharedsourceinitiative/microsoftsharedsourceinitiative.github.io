Enabling Wake Up

Applications can request that devices wake the machine by calling the RequestDeviceWakeup Win32 function, which calls the configuration manager Enable_DevNode_Wakeup service, as follows

  1. The component that wants to enable wake up on a device calls Enable_DevNode_Wakeup.
  2. The configuration manager sets a flag in the device node to indicate that the device associated with the device node is enabled for wake up.
  3. The configuration manager then sends a CONFIG_ENABLE_WAKEUP message to the device driver.
  4. Upon receiving the CONFIG_ENABLE_WAKEUP message, the device driver can prepare any internal status it needs in order to enable wake up.

The process for disabling wake up is similar to the preceding. A system component calls Disable_DevNode_Wakeup, which results in a CONFIG_DISABLE_WAKEUP message being sent to the device driver.

In summary, components that want to enable device wake up should call Enable_DevNode_Wakeup. Device drivers that support wake up must listen for and handle CONFIG_ENABLE_WAKEUP and CONFIG_DISABLE_WAKEUP messages.
