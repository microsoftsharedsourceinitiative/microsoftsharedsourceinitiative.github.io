Set_DevNode_PowerCapabilities

Device drivers call the Set_DevNode_PowerCapabilities service to set the capabilities mask for their device after combining the capabilities of the device with that of the hardware (originally found by calling Get_DevNode_PowerCapabilities). 

CONFIGRET
  CONFIGMG_Set_DevNode_PowerCapabilities(
    DEVNODE  dnDevNode, 
    ULONG  ulPowerCapabilities, 
    ULONG  ulFlags
    );

Parameters

dnDevNode
Handle to the device node from which to get the capabilities.
ulPowerCapabilities
Pointer to a ULONG that receives the device's capabilities. Pointer ust be one of the following values as defined in configmg.h.
Flag Meaning
CM_POWERSTATE_D0 Device supports the D0 state.
CM_POWERSTATE_D1 Device supports the D1 state.
CM_POWERSTATE_D2 Device supports the D2 state.
CM_POWERSTATE_D3 Device supports the D3 state.

ulFlags
Flags indicating the power capabilities for the device. Must be one of the following.
Flag Meaning
CM_CAPABILITIES_NORMAL Capabilities usually evoked (default).
CM_CAPABILITIES_FOR_WAKEUP Capabilities evoked on device power wake up.

Note  New power capabilities are only accepted for a device the first time CONFIGMG_Set_DevNode_PowerCapabilities is called for a given devnode. Subsequent calls result in a logical AND of ulPowerCapabilities with the device's prior power capabilities. This means subsequent calls can only remove power capabilities.
