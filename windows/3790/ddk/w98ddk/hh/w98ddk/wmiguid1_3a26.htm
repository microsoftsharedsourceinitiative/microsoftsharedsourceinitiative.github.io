IRP_MN_DISABLE_COLLECTION

The driver receives an IRP_MN_DISABLE_COLLECTION to inform it that it should cease collection of data for the data block specified by DataPath. Only those devices that register data blocks with the WMIREG_FLAG_EXPENSIVE flag will receive this IRP. DataPath points at a GUID for the data block and the driver should disable collection for all instances of the data block. The driver should verify that the DataPath is valid and, if so, disable collection for data blocks and continue.

WMI will manage any reference counting issues in the case that multiple data consumers have requested that collection for a data block be enabled. The driver should not expect a second disable without having received an intervening enable.

Members

ProviderId
Contains the WMI provider ID of the device that should respond to this request.
DataPath
Points to the GUID of the data block to stop collecting.

Return Value

Returns STATUS_SUCCESS, STATUS_WMI_GUID_NOT_FOUND, or STATUS_INVALID_DEVICE_REQUEST.
