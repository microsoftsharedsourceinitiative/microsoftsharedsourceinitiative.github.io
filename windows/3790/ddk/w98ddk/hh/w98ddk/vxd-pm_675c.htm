CONFIG_WAKEUP

The configuration manager sends CONFIG_WAKEUP to a driver to indicate that its device is signaling wake up. Upon receiving this message, the device driver can begin handling the wake up event. This may require turning the device on by calling Set_DevNode_PowerState. If the device driver is a bus driver, it should check to see whether its children are signaling wake up and, if so, call Device_Wakeup accordingly.
