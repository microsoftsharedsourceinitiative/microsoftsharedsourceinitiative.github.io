MIDM_CLOSE

Sent to a MIDI input device driver to deallocate a specified device.

  • The return value is an error code, or zero (MMSYSERR_NOERROR) if the operation is successful. Possible error codes are:
MMSYSERR_NOTENABLED The driver failed to load or initialize.
MIDIERR_STILLPLAYING There are still buffers in the queue.

DWORD dwParam1
Unused.
DWORD dwParam2
Unused.

If there are input buffers that have been received with the MIDM_ADDBUFFER message and have not been returned to the application, the driver should fail the close operation by returning MIDIERR_STILLPLAYING. If the close operation is successful, the driver should use the DriverCallback function to send the client a MIM_CLOSE message.
