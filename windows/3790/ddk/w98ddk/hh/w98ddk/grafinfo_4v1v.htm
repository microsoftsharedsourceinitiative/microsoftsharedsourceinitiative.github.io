Minimum Number of Blanks

The sMinBlankSkip box contains a value that specifies the minimum number of bytes of null data that must occur before the Universal Printer Driver begins removing embedded blanks from the block of bitmap data. If you use the Universal Printer Driver to remove most of the null raster data sent to the printer to reduce data transmission, printing and performance time may increase if this value is too small due to print-head movement when switching from graphics mode to cursor movement commands. It is suggested that dot-matrix minidrivers use values no smaller than the number of bytes of raster data required for 2 inches of output.

Diconix 150 Plus

For the Diconix 150 Plus printer, the sMinBlankSip box contains the value zero.

HP LaserJet IIP

For the HP LaserJet IIP printer, the sMinBlankSkip box contains the value 32; this means that the Universal Printer Driver will not strip blank data from a bitmap unless there are at least 32 consecutive NULL bytes.
