WM_IME_CHAR

The WM_IME_CHAR message is sent to an application when the IME gets a character of the conversion result. The form of the value to be sent is similar to the English Windows WM_CHAR message. The difference is that wParam can specify a double-byte character.

There is no return value.

wParam
Double-byte FE character.
lParam
Similar to the English Windows version of the WM_CHAR message.
Bit Value
0 - 15 Repeat count: Since the first byte and second byte are continuous, this is always 1.
16 - 23 Scan Code for an FE character.
24 - 28 Not used.
29 Context code.
31 Conversion state.

DefWindowProc generates two WM_CHAR messages, each including one byte of a DBCS character, for a double-byte FE character. If the message includes a SBCS character, DefWindowProc generates one WM_CHAR message.
