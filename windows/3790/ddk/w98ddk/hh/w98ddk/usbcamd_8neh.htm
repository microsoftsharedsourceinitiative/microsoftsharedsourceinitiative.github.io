USBCAMD_DriverEntry

USBCAMD_DriverEntry is called from the cam minidriver DriverEntry routine to register the minidriver with USBCAMD. This effectively binds the two drivers together.

ULONG
  USBCAMD_DriverEntry(
    PVOID Context1,
    PVOID  Context2,
    ULONG  DeviceContextSize,
    ULONG  FrameContextSize,
    PADAPTER_RECEIVE_PACKET_ROUTINE  ReceivePacket
    );

Parameters

Context1
Any value or structure appropriate to the minidriver implementation. This input parameter is not used directly in this function but is passed along to other calls within the minidriver.
Context2
Any value or structure appropriate to the minidriver implementation. This input parameter is not used directly in this function but is passed along to other calls within the minidriver.
DeviceContextSize
Size, in bytes, of the minidriver’s device-specific context.
FrameContextSize
Size, in bytes, of a video frame structure. Use NULL if not needed.
ReceivePacket
Entry point for receiving adapter-based SRB requests.

Return Values

Returns an NT_STATUS value indicating the result of the registration attempt. If a value other than STATUS_SUCCESS is returned, the minidriver will be unloaded.

Comments

FrameContextSize is optional. A non-NULL value should be provided only with calls to CamNewVideoFrame or CamProcessRawVideoFrame.
