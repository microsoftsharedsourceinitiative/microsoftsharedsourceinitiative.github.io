_IOS_IsDriveLocked

Summary

This service checks the specified drive to see if it is locked for exclusive access.

Entry/Exit

The service expects the drive number as the parameter passed in (TOS+4) and will return, in EAX, either a zero or a non-zero. Zero signifies not locked, and non-zero indicates there is a lock.

Note  Drive numbering starts with zero for drive letter A: and counts up from there. If you are checking drive F: for instance, you should pass in a five.

Usage

This service function is used to check a drive whose lock state locked is unknown, and will detect if a drive has been locked using _IOS_LockDrive. The return value indicates locked or unlocked with non-zero and zero respectively.

This service uses (writes to) EAX, ECX, EDX, and Flags.
