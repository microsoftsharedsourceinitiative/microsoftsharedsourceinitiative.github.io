IRP_MN_ENABLE_COLLECTION

The driver receives an IRP_MN_ENABLE_COLLECTION to inform it that it should begin data collection of data for the data block specified by DataPath. Only those devices that register data blocks with the WMIREG_FLAG_EXPENSIVE flag will receive this IRP. DataPath points at a GUID for the data block being collected and the driver should enable collection for all instances of the data block. The driver should verify that the DataPath is valid and, if so, enable collection for data blocks and continue.

WMI will manage any reference counting issues in the case that multiple data consumers have requested that collection for a data block be enabled. The driver should not expect a second enable without having received an intervening disable, with the exception that an enable for all instances of a data block may be received after receiving an enable for a single instance of a data block.

Members

ProviderId
Contains the WMI provider ID of the device that should respond to this request.
IrpStack->Parameters.WMI.DataPath
Points to the GUID of the data block to begin collecting.

Return Value

Returns STATUS_SUCCESS, STATUS_WMI_GUID_NOT_FOUND, or STATUS_INVALID_DEVICE_REQUEST.
