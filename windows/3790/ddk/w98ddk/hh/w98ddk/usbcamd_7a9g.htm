CamProcessUSBPacket

CamProcessUSBPacket, called at the DISPATCH_LEVEL, allows the minidriver to determine whether this packet is part of the current video frame or a new video frame. This function should complete as quickly as possible. Any image processing should be deferred to the CamProcessRawVideoFrame routine.

ULONG
  CamProcessUSBPacket(
    PDEVICE_OBJECT  BusDeviceObject,
    PVOID  DeviceContext,
    PVOID  CurrentFrameContext,
    PUSBD_ISO_PACKET_DESCRIPTOR  SyncPacket,
    PVOID  SyncBuffer,
    PUSBD_ISO_PACKET_DESCRIPTOR  DataPacket,
    PVOID  DataBuffer,
    PBOOLEAN  FrameComplete,
    PBOOLEAN  NextFrameIsStill
    );

Parameters

BusDeviceObject
Device object created by the hub. URBs can be submitted to the device through this device object.
DeviceContext
Minidriver device context.
CurrentFrameContext
Context for this particular frame.
SyncPacket
ISO packet descriptor from sync pipe. Set to NULL if the interface has only one pipe.
SyncBuffer
Pointer to data for the SyncPacket
DataPacket
ISO packet descriptor from data pipe.
DataBuffer
Pointer to DataPacket.
FrameComplete
Indicates to USBCAMD that this is the first data packet for a new video frame.
NextFrameIsStill
Indicates FALSE if the video stream is providing a live image, but TRUE if the next frame is a still image or an image capture stream.

Return Values

The return value is the number of bytes that should be copied into RawFrameBuffer or FrameBuffer.
