USBCAMD_ControlVendorCommand

USBCAMD_ControlVendorCommand is available to send vendor-specific commands to the control pipe. This function may be called at IRQL >= PASSIVE_LEVEL. If the function is called at IRQL > PASSIVE_LEVEL, the command will be deferred. After completion, the CommandComplete parameter is called with a pointer to the CommandContext.

NTSTATUS
  USBCAMD_ControlVendorCommand(
    IN PVOID  DeviceContext,
    IN UCHAR  Request,
    IN USHORT  Value,
    IN USHORT  Index,
    IN PVOID  Buffer,
    IN OUT PULONG  BufferLength,
    IN BOOLEAN  GetData,
    IN PCOMMAND_COMPLETE_FUNCTION  CommandComplete,
    IN PVOID  CommandContext
    );

Parameters

DeviceContext
Device-specific context.
Request
Value of the bRequest field for the vendor command.
Value
Value of the wValue field for the vendor command.
Index
Value of the wIndex field for the vendor command.
Buffer
Data buffer if the command has data. May be NULL.
BufferLength
Pointer length, in bytes, of buffer. May be NULL, if Buffer is NULL.
GetData
Flag indicating data direction. GetData indicates data was sent from device to host.
CommandComplete
Function called when command is completed.
CommandContext
Context passed to CommandComplete function.

Return Values

Returns NTSTATUS code from command of STATUS_PENDING if command is deferred.
