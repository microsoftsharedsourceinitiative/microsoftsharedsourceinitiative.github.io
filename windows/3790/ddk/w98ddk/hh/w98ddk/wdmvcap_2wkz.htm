KSPROPERTY_VIDEODECODER_CAPS

The KSPROPERTY_VIDEODECODER_CAPS read-only property identifies the hardware capabilities of the analog video decoder. In response to this property request, the driver returns the following structure:

typedef struct {
  KSPROPERTY  Property;
  ULONG  StandardsSupported;
  ULONG  Capabilities;
  ULONG  SettlingTime;
  LONG   HSyncPerVSync;
 } KSPROPERTY_VIDEODECODER_CAPS_S, *PKSPROPERTY_VIDEODECODER_CAPS_S;

Members

Property
Specifies the property used.
StandardsSupported
Specifies the video standards supported by the device. Support for multiple standards by bitwise OR of KS_AnalogVideo_*r.
Capabilities
Specifies a flag based on KS_VIDEODECODER_FLAGS. The following flags are provided.
Flag Description
KS_VIDEODECODER_FLAGS_
CAN_DISABLE_OUTPUT		 Specifies that the decoder can tri-state its output lines. This flag is only important for video decoders which use a video port.
KS_VIDEODECODER_FLAGS_
CAN_USE_VCR_LOCKING		 Specifies that the video decoder can alter its PLL timings to lock onto noisy signals. This is typically used when the video source is a VCR that introduces variability in the vertical and horizontal sync timing.
KS_VIDEODECODER_FLAGS_
CAN_INDICATE_LOCKED		 Specifies that the video decoder can indicate whether it has locked to an incoming analog video signal.g.

KSPROPERTY_VIDEODECODER_CAPS_S.Capabilities uses the following flags: 

typedef enum {
    KS_VIDEODECODER_FLAGS_CAN_DISABLE_OUTPUT  = 0X0001, 
    KS_VIDEODECODER_FLAGS_CAN_USE_VCR_LOCKING = 0X0002, 
    KS_VIDEODECODER_FLAGS_CAN_INDICATE_LOCKED = 0X0004, 
}KS_VIDEODECODER_FLAGS;
SettlingTime
Specifies the settling time when changing input video sources expressed in milliseconds.
HSyncPerVSync
Specifies the number of horizontal sync pulses the video decoder produces during the vertical sync period. Decoders that follow the ITU-656 should set this value to six.
