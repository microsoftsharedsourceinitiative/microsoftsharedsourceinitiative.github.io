The following sample was taken from the S3 ViRGE INF file. It is a partial listing of the AddReg section for the S3 driver. This sample defines custom refresh rates for 640x480, 800x600, 1024x768, 1152x864, and 1280x1024.

Because 1600x1200 has no custom refresh rates specified, only "Adapter Default" is available to the user at that resolution. Modes at 16 bpp and higher color depths use the same custom refresh rates specified for their 8 bpp counterparts.

HKR,"MODES\8\640,480",,,"60,72,75,85"
HKR,"MODES\8\800,600",,,"56,60,72,75,85"
HKR,"MODES\8\1024,768",,,"43,60,70,75,85"
HKR,"MODES\8\1152,864",,,"43,60,70,75,85"
HKR,"MODES\8\1280,1024",,,"45,60,72,75"
HKR,"MODES\8\1600,1200"
HKR,"MODES\16\640,480"
HKR,"MODES\16\800,600"
HKR,"MODES\16\1024,768"
HKR,"MODES\16\1152,864"
HKR,"MODES\16\1280,1024"
HKR,"MODES\16\1600,1200"

HKR,"MODES\24\800,600"

As an example, if this part does not support 75 Hz at 1024x768x16bpp, the line HKR,"MODES\16\1024,768 is replaced by HKR,"MODES\16\1024,768",,,"43,60,70,85"

This new set of custom refresh rates overrides the 8-bpp rates for this mode only.
