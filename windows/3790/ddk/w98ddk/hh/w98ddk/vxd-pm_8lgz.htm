Get_DevNode_PowerCapabilities

Device drivers call the Get_DevNode_PowerCapabilities service to determine the capabilities mask for their device. Initially, this mask is set to the capabilities as reported by the hardware using standard mechanisms. That is, the mechanisms used by the device driver's parent drivers.

CONFIGRET
  CONFIGMG_Get_DevNode_PowerCapabilities(
    DEVNODE  dnDevNode, 
    PFARULONG  pulPowerCapabilities, 
    ULONG  ulFlags
    );

Parameters

dnDevNode
Handle to the device node from which to get the capabilities.
pulPowerCapabilities
Pointer to a ULONG that receives the device's capabilities. Pointer must be one of the following values as defined in configmg.h.
Flag Meaning
CM_POWERSTATE_D0 Device supports the D0 state.
CM_POWERSTATE_D1 Device supports the D1 state.
CM_POWERSTATE_D2 Device supports the D2 state.
CM_POWERSTATE_D3 Device supports the D3 state.

ulFlags
Flags indicating the power capabilities for the device. Must be one of the following:
Flag Meaning
CM_CAPABILITIES_NORMAL Capabilities evoked normally (default)
CM_CAPABILITIES_FOR_WAKEUP Capabilities evoked on device power wake up
