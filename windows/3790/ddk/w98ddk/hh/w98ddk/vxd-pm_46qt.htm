Sleep Phase

During the sleep phase, the configuration manager tells each loaded device driver to put its child devices into low power states.

The configuration manager calls each driver (in sequence from children toward the root) with a CONFIG_SET_POWERSTATE_Dx message corresponding to the CONFIG_TEST_SET_POWERSTATE_Dx message which the driver responded to successfully.

Upon receiving a CONFIG_SET_POWERSTATE_Dx message, each driver saves hardware context, as described in Switching to Low Power State.

In summary, device drivers must listen for and respond to CONFIG_TEST_SET_POWERSTATE_Dx messages. Device drivers must listen for CONFIG_SET_POWERSTATE_Dx messages and call the configuration manager Set_DevNode_PowerState service to put their devices into the correct power state. (For more information, see Switching to Low Power State.)
