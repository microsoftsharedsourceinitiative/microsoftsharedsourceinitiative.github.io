PCIAllocatePages

The PCIAllocatePages function allocates pages from system memory.

PVOID
  PCIAllocatePages(
    ULONG  ulNumPages, 
    ULONG  ulAlignment, 
    ULONG  ulFlags
    );

Parameters

ulNumPages
Number of pages to allocate.
ulAlignment
Specifies an alignment mask for the acceptable starting pages for the physical address. The following values are provided.
Value Meaning
0x00000000 Address is a multiple of 4K.
0x00000001 Address is a multiple of 8K.
0x00000003 Address is a multiple of 16K.
0x00000007 Address is a multiple of 32K.
0x0000000F Address is a multiple of 64K.
0x0000001F Address is a multiple of 128K.

ulFlags
Specifies the allocation flags. Must be zero or one of the following values.
Value Meaning
PCIPAGEZEROINIT Fills the pages with zeros. If this value is not given, the initial content of the memory block is undefined.
PCIPAGECONTIG Allocates contiguous physical pages to create the memory block.
PCIPAGEWRITETHROUGH Allocates using “write-through” memory.
PCIPAGECACHEDISABLE Allocates using “uncached” memory.

All other values are reserved. The default is to allocate memory that is not zero initialized, not contiguous, and cached. It is invalid to specify both PCIPAGEWRITETHROUGH and PCIPAGECACHEDISABLE.

Return Values

This service returns NULL or the linear address of the allocated pages.
