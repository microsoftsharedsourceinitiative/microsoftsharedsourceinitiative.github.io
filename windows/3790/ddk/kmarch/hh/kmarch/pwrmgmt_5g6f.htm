Handling a System Set-Power IRP in a Device Power Policy Owner

The device power policy owner in each device stack is responsible for putting its device stack into an appropriate device power state for the system power state specified in each system set-power IRP.

When a device power policy owner receives an IRP_MN_SET_POWER for a system power state, it responds by passing down the IRP and, in an IoCompletion routine, sending an IRP_MN_SET_POWER for a corresponding device power state. When all drivers in the stack have completed the device IRP, the device power policy owner completes the system IRP.
