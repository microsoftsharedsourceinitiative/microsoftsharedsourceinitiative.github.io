Kernel-Mode Driver Architecture: Windows DDK

Synchronization

This section describes the kernel-mode support routines that drivers can call to:

  • Synchronize the execution of their own standard driver routines in Driver Routines and I/O Objects
  • Temporarily change the current IRQL for a call to a support routine or that return the current IRQL in IRQL
  • Synchronize access to resources with spin locks or to perform interlocked operations without spinlocks in Spin Locks and Interlocks
  • Manage time-outs or determine system time in Timers
  • Use system threads or to manage synchronization within a nonarbitrary thread context in Driver Threads, Dispatcher Objects, and Resources
