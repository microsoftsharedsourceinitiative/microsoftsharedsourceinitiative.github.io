Smart Card Callback Parameters

With all IOCTL requests, except for IOCTL_SMARTCARD_IS_ABSENT and IOCTL_SMARTCARD_IS_PRESENT, SmartcardDeviceControl will initialize the IoRequest member of the SMARTCARD_EXTENSION structure before calling a callback function. The following table indicates the sort of initialization that SmartcardDeviceControl performs.

Member of IoRequst Initialization Performed by SmartcardDeviceControl
IoRequest.RequestBuffer Stores the user data to be sent to the card in the buffer pointed to by this member.
IoRequest.RequestBufferLength Stores the length of the user buffer in this member.
IoRequest.ReplyBuffer Stores the data returned by the smart card in the buffer pointed to by this member.
IoRequest.ReplyBufferLength Stores the size of the reply buffer in this member.
IoRequest.Information Stores the number of bytes actually received from the card in the ULONG pointed to by this member.
MajorIoControlCode Stores the major I/O control code of the IOCTL request in this member.
MinorIoControlCode Stores the minor I/O control code of the IOCTL request (if any) in this member.

In addition, with the WDM version of the driver library, the structure SmartcardExtension->OsData will be set up as follows.

Member Description
CurrentIrp Receives a pointer to the requesting IRP for every control request, except IOCTL_SMARTCARD_IS_ABSENT and IOCTL_SMARTCARD_IS_PRESENT.
NotificationIrp Receives a pointer to the requesting IRP for the control request IOCTL_SMARTCARD_IS_ABSENT or IOCTL_SMARTCARD_IS_PRESENT.
