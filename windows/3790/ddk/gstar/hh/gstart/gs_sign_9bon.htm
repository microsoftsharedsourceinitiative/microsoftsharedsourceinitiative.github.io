WHQL Digital Signatures

Driver packages that pass Windows Hardware Quality Labs (WHQL) testing can be digitally signed by WHQL. If your driver package is digitally signed by WHQL, it can be distributed through the Windows Update program or other Microsoft-supported distribution mechanisms.

You can apply for a WHQL digital signature through the Windows Haredware Quality Labs Web site. A WHQL digital signature consists of a digitally signed catalog file that WHQL provides. The digital signature does not change the driver binary files or the INF file that you submit for testing.

To receive a WHQL digital signature, your driver package must meet the following criteria:

  • A WHQL test program must be available for the driver, and the driver must have passed the test. For more information, see "Designed for Windows" Logo Testing.
  • The driver must be installed by an INF file.
  • The INF file must not contain errors that are identified by the ChkINF or INFCatReady (InfCatR.exe) programs.
  • The driver package must not include replacements for, or duplicates of, any system files that are supplied by Microsoft.

After your driver package has been signed, you should test it in every supported installation path on every target operating system to make sure that it installs without any errors or warnings.

WHQL digital signatures are valid for 20 years.

For more information about WHQL digital signatures, see Using WHQL Digital Signatures To Authenticate Drivers.
