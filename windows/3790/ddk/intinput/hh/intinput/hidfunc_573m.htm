HidD_SetNumInputBuffers

The HidD_SetNumInputBuffers routine sets the maximum number of input reports that the HID class driver ring buffer can hold for a specified top-level collection.

BOOLEAN
  HidD_SetNumInputBuffers(
    IN HANDLE  HidDeviceObject,
    OUT ULONG  NumberBuffers
    );

Parameters

HidDeviceObject
Specifies an open handle to a top-level collection.
NumberBuffers
Specifies the maximum number of buffers that the HID class driver maintains for input reports from the collection specified by HidDeviceObject.

Return Value

HidD_SetNumInputBuffers returns TRUE if it succeeds; otherwise, it returns FALSE.

Headers

Declared in hidsdi.h. Include hidsdi.h.

Comments

Only user-mode applications can call HidD_SetNumInputBuffers. Kernel-mode drivers can use an IOCTL_SET_NUM_DEVICE_INPUT_BUFFERS request.

If HidD_SetNumInputBuffers returns FALSE and the Microsoft® Win32® function GetLastError indicates that an invalid parameter was supplied, the number of input buffers specified was below the minimum number allowed by the HID class driver.

For more information, see HID Concepts, HID collections, and Operating HID Collections.
