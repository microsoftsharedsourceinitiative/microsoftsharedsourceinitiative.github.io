General Requirements on HIDClass Drivers

You can supply three basic types of drivers to support a HIDClass device:

  1. If the system-supplied HID minidrivers do not support the bus or port that the device is attached to, you must supply a HID minidriver for an input device.

    The system supplies a HID minidriver for USB HID devices and, in Microsoft® Windows® 2000 and later versions, a minidriver for game port devices.

    For information about the operation of a HID minidriver, see Operation of the HID Class Driver and HID Minidrivers.

  2. An optional WDM function and filter driver for a HID collection.
  3. An optional WDM driver that does not attach a device object to the device stack of a collection. Instead, the driver accesses the collection by opening a file on the collection.

Vendor drivers for HID collections are optional because a user-mode application can directly access and control HID collections by using HID support routines. However, you can provide a driver to add custom functionality, or to map the HID class driver interface to another driver interface. For example, a vendor driver might provide a value-added service to all joystick collections. Also, the system-supplied HID mapper drivers map the i8042 port interface that is used by the system-supplied non-HIDClass keyboard and mouse class drivers, to the interface that is used by the HID class driver for HID collections.

If a vendor supplies a driver, the driver:

  • must comply with the minimum requirements on a WDM function driver, as described in Windows Driver Model. This includes driver entry points, dispatch routines, Plug and Play, power management, and Windows Management Instrumentation (WMI). This basic functionality provides the generic operation that Windows requires of the driver.
  • typically supports a vendor-defined device interface — see Device Interface Classes. Upper-level drivers or user-mode applications use the custom interface to access the devices that the vendor driver operates. The custom interface might add functionality or, perhaps, simplify the interface to the HID class driver.

If the driver is not a function driver or filter driver, it can use Plug and Play notification to find HID collections. After finding a collection, the driver opens the collection and operates it in the same manner as a function or filter driver.

For information about how a driver operates a HID collection, see HID Concepts and Operating HID Collections.
