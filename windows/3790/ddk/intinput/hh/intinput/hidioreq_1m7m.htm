IOCTL_SET_NUM_DEVICE_INPUT_BUFFERS

Operation

The IOCTL_SET_NUM_DEVICE_INPUT_BUFFERS request sets the size of the input report queue for a top-level collection.

The input report queue is implemented as a ring buffer. If a collection transmits data to HID class driver faster than it is read, device data can be lost. The size of the report input queue can be tuned using IOCTL_SET_NUM_DEVICE_INPUT_BUFFERS.

For general information about HIDClass devices, see HID Concepts, HID collections, and Operating HID Collections.

Input

Parameters.DeviceIoControl.InputBufferLength in the I/O stack location of the IRP indicates the size, in bytes, of the input buffer, which must be >= sizeof(ULONG).

Irp->AssociatedIrp.SystemBuffer points to a buffer that will receive the new size of the report input queue. The size of the buffer is sizeof(ULONG).

Output

None.

I/O Status Block

The HID class driver sets the following fields of Irp.IoStatus:

  • Status is set to STATUS_SUCCESS if the transfer completed without error. Otherwise, it is set to an appropriate NTSTATUS error code.

Headers

Defined in hidclass.h. Include hidclass.h.
