Device Stacks for Game Port Devices

This following figure shows the system-supplied driver stacks that support game port (I/O port 201) devices.

Driver Stacks for HIDClass Game Port Devices

The following device objects are created by the system-supplied drivers:

  • The game port bus driver gameenum.sys creates physical device objects (PDO for a hardware device attached to a game port (required).
  • The HID class driver creates functional device objects (FDO) for each PDO created by the game port bus driver (required).
  • The HID class driver creates a PDO for each HID collection that the hardware device supports (required).

    The HID class driver accesses the hardware device through the hidgame.sys minidriver.

An optional vendor driver creates an FDO or a filter DO for a HID collection. The vendor driver accesses the HID collection and provides a device-specific upper-level interface to other kernel-mode drivers or user-mode applications.

Note  Vendor drivers are not required to operate a HIDClass game port device. Joysticks are shared devices. User-mode system components directly access a joystick by opening its PDO and using the HID support routines. User-mode vendor applications can also operate game port collections in the same manner.
