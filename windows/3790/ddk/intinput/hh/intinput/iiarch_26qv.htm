Driver Stacks for HIDClass Game Port Devices

In Windows 2000 and later versions, game port devices (I/O port 201) are HIDClass devices.

The following figure illustrates the system-supplied driver stack for game port devices in Windows 2000 and later versions.

Driver Stack for Game Port Devices in Windows 2000 and Later Versions

As shown in the figure, the HIDClass components (hidgame.sys, the HID class driver, hidparse.sys and hid.dll) and DirectInput fully support game port devices in Windows 2000 and later versions.

Vendor drivers for game port devices are not required.

User-mode applications can access game port devices indirectly through DirectInput. A user-mode application can also directly access a joystick by using the HIDClass support routines. The application calls CreateFile to obtain a file handle to the joystick collection and uses the HID support routines to communicate with the collection.

For information about driver support for game ports in Windows Me, see Driver Stacks for Non-HIDClass Game Port Devices.
