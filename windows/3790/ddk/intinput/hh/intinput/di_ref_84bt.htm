Interactive Input Devices: Windows DDK

IDirectInputJoyConfig8

The IDirectInputJoyConfig8 interface contains methods that allow hardware developers who are writing property sheets to write and read information to and from the registry.

Note  If you need to open registry keys, you should use the IDirectInputJoyConfig8::OpenConfigKey and IDirectInputJoyConfig8::OpenTypeKey methods rather than opening registry keys directly. Using either of these methods ensures that the correct registry branch is opened. In addition, the IDirectInputJoyConfig8 interface will be supported in future versions of DirectInput when the underlying registry data may be structured differently.

The IDirectInputJoyConfig8 interface consists of the following methods:

Acquire

AddNewHardware

DeleteConfig

DeleteType

EnumTypes

GetConfig

GetTypeInfo

GetUserValues

OpenAppStatusKey

OpenConfigKey

OpenTypeKey

SendNotify

SetConfig

SetCooperativeLevel

SetTypeInfo

SetUserValues

Unacquire

In addition, all COM interfaces inherit the following IUnknown interface methods:

AddRef

QueryInterface

Release

These three methods allow additional interfaces to be added to the DirectInputJoyConfig object without affecting the functionality of the original interface.

Also please note the following difference between Windows Millennium Edition and Windows XP:

On Windows XP machines. Using the IDirectInputJoyConfig8 interface, use the instructions in the order described following to change the joystick ID:

    hres = GetConfig(uSrcID);
    if (SUCCEEDED(hres)) {
        SetConfig(uDstID);
        DeleteConfig(uSrcID);
    }

However, this order of instructions does not work on Windows 95/98/Me. For these operating systems, use code written as follows:

    hres = GetConfig(uSrcID);
    if (SUCCEEDED(hres)) {
        DeleteConfig(uSrcID);
        SetConfig(uDstID);
    }
