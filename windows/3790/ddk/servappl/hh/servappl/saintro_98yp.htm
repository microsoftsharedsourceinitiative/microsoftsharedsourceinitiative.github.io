Initializing the Local Display Device

The SA port driver calls the local display miniport driver's SaHwInitialize function to initialize the hardware resources that are specified in the miniport driver's device extension structure. The miniport driver can perform the following initialization tasks:

  • Assign the device extension's pointer to a DEVICE_OBJECT to the local display device's object pointer that is passed in the SaHwInitialize call.
  • Initialize the device extension's synchronization object. If the device extension contains, for example, a mutex (KMUTEX object), SaHwInitialize calls KeInitializeMutex to initialize the mutex.
  • Set the size of the device extension's internal-buffer-size variable. For example, this variable can be set to SA_DISPLAY_MAX_BITMAP_SIZE, which is the maximum possible size.
  • Call the SaPortAllocatePool function to allocate nonpageable system memory for the device extension's internal buffer. This buffer holds the display information that is transferred from the requesting application to the local display device.
  • Call the RtlZeroMemory function to fill the internal buffer's memory with zeros.
