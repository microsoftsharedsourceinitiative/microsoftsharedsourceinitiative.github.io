Initializing the Keypad Miniport Driver

During startup, the operating system initializes the keypad miniport driver by calling the driver's DriverEntry function. DriverEntry allocates an SAPORT_INITIALIZATION_DATA structure and calls the RtlZeroMemory function to fill the structure with zeros. DriverEntry then specifies the following values in members of this structure:

  • SA_DEVICE_KEYPAD device type in the DeviceType member.
  • the entry points for the keypad miniport driver's callback functions in their respective members.
  • the size of the keypad device's device extension structure in the DeviceExtensionSize member.

DriverEntry then passes this filled SAPORT_INITIALIZATION_DATA in a call to the SaPortInitialize function. The SA port driver performs all the necessary initialization tasks for the miniport driver.
