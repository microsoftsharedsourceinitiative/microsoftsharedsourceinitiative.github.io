SaRead

The SaRead function is a required function if the miniport driver must transfer data from its device to the system. 

NTSTATUS
  SaRead(
    IN PVOID DeviceExtension,
    IN PIRP Irp,
    IN PVOID FsContext,
    IN LONGLONG StartingOffset,
    OUT PVOID DataBuffer,
    IN ULONG DataBufferLength
    );

Parameters

DeviceExtension
Pointer to a driver-allocated area in which the miniport driver maintains the state of the device. The SaHwInitialize function sets up this area.
Irp
Specifies the IRP that contains the IRP_MJ_READ request being handled.
FsContext
Pointer to a file-system context area for a handle to the file object that represents the SA device object.
StartingOffset
Specifies the offset, in bytes, for this transfer operation.
DataBuffer
Pointer to a buffer that receives the data that is transferred from the device.
DataBufferLength
Specifies the number of bytes to transfer.

Return Value

SaRead returns STATUS_SUCCESS on successfully completing an operation synchronously; it returns STATUS_PENDING on successfully queuing an operation for asynchronous operation in another thread; otherwise, it returns one of the error status values defined in ntstatus.h.

Headers

The prototype for SaRead is declared in saport.h as (*PSA_IO_ROUTINE). Include saport.h.

Comments

If the miniport driver queues the read operation for asynchronous operation in another thread, it returns STATUS_PENDING. After the read operation completes, the miniport driver calls the SaPortCompleteRequest function to complete the original IRP_MJ_READ request.

The SaRead function runs at IRQL <= DISPATCH_LEVEL.
