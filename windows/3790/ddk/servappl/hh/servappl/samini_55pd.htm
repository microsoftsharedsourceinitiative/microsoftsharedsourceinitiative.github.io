SaHwInitialize

The SaHwInitialize function is a required function that sets up an SA device for I/O operations.

NTSTATUS 
  SaHwInitialize(
    IN PDEVICE_OBJECT  DeviceObject,
    IN PIRP  Irp,
    IN PVOID  DeviceExtension,
    IN PCM_PARTIAL_RESOURCE_DESCRIPTOR  PartialResources,
    IN ULONG  PartialResourceCount
    );

Parameters

DeviceObject
Pointer to a DEVICE_OBJECT structure. This is the device object for the device being initialized.
Irp
Specifies the IRP that contains the IRP_MN_START_DEVICE request being handled.
DeviceExtension
Pointer to a driver-allocated area in which the miniport driver maintains the state of the device. SaHwInitialize sets up this area.
PartialResources
Pointer to a CM_PARTIAL_RESOURCE_DESCRIPTOR structure that specifies one or more system hardware resources for the SA device.
PartialResourceCount
Specifies the number of system hardware resources identified at PartialResources.

Return Value

SaHwInitialize returns STATUS_SUCCESS on success; otherwise, it returns one of the error status values defined in ntstatus.h.

Headers

The prototype for SaHwInitialize is declared in saport.h as (*PSA_HW_INITIALIZE). Include saport.h.

Comments

The SA port driver submits no requests to a miniport driver until the miniport driver's initialization completes.

The SA port driver calls SaHwInitialize in the context of SaPortInitialize.

If SaHwInitialize fails the initialization, it must release all resources it has already allocated before it returns control.

SaHwInitialize runs at IRQL = PASSIVE_LEVEL.
