Understanding Server Availability Driver Samples

The server availability driver samples in the DDK work with virtual devices rather than actual devices. These virtual devices consist of a keypad and a local display and are implemented as the sagui.exe application. Because these virtual devices cannot generate interrupts for their sample drivers, the drivers do not implement SaInterruptServiceRoutine and SaIsrForDpcRoutine functions.

The DDK contains the following server availability driver samples:

  • mskeypad

    This sample keypad miniport driver reads key presses that are submitted from a virtual keypad device. Because this virtual device cannot interrupt the driver, the virtual device submits an IOCTL_VDRIVER_INIT code to the driver when there is key-press information to read.

  • msnvram

    This sample nonvolatile random access memory (NVRAM) miniport driver reads and writes information to the registry to simulate an NVRAM device.

  • msdisp

    This sample local display miniport driver writes display messages to a virtual local display device. This sample driver creates a named event on which the virtual device waits. When the driver sets the event to the signaled state, the virtual device is notified to begin to receive display messages from the driver. Because this virtual device cannot interrupt the driver, the virtual device submits an IOCTL_VDRIVER_INIT code to the driver to indicate that the virtual device is ready to receive display messages.

For details on these samples, see the source code for each sample in the DDK.
