SAPORT_INITIALIZATION_DATA

The SAPORT_INITIALIZATION_DATA structure contains driver-specific information and entry pointers to the callback functions that a miniport driver supports.

typedef struct _SAPORT_INITIALIZATION_DATA {
  ULONG  StructSize;
  ULONG  DeviceType;
  ULONG  DeviceExtensionSize;
  ULONG  FileContextSize;
  PSA_HW_INITIALIZE  HwInitialize;
  PSA_CREATECLOSE  CreateRoutine;
  PSA_CREATECLOSE  CloseRoutine;
  PSA_DEVICE_IOCTL  DeviceIoctl;
  PSA_IO_ROUTINE  Write;
  PSA_IO_ROUTINE  Read;
  PSA_CANCEL  CancelRoutine;
  PKSERVICE_ROUTINE  InterruptServiceRoutine;
  PIO_DPC_ROUTINE  IsrForDpcRoutine;
} SAPORT_INITIALIZATION_DATA, *PSAPORT_INITIALIZATION_DATA;

Members

StructSize
Specifies the size in bytes of this structure.
DeviceType
Specifies the device type that the miniport driver supports. This type must be set to one of the constants in the following table.
Device type Meaning
SA_DEVICE_DISPLAY The SA device is a local display.
SA_DEVICE_KEYPAD The SA device is a keypad.
SA_DEVICE_NVRAM The SA device is nonvolatile RAM.
SA_DEVICE_UNKNOWN The SA device is unknown.

DeviceExtensionSize
Specifies the size in bytes of the miniport driver's device-extension structure. The miniport driver uses this structure to maintain the state of its device.
FileContextSize
Specifies the size in bytes of the miniport driver's file-system context area.
HwInitialize
Specifies the driver-supplied SaHwInitialize callback.
CreateRoutine
Specifies the driver-supplied SaCreateRoutine callback.
CloseRoutine
Specifies the driver-supplied SaCloseRoutine callback.
DeviceIoctl
Specifies the driver-supplied SaDeviceIoctl callback.
Write
Specifies the driver-supplied SaWrite callback.
Read
Specifies the driver-supplied SaRead callback.
CancelRoutine
Specifies the driver-supplied SaCancelRoutine callback.
InterruptServiceRoutine
Specifies the driver-supplied SaInterruptServiceRoutine callback.
IsrForDpcRoutine
Specifies the driver-supplied SaIsrForDpcRoutine callback.

Headers

Defined in saport.h. Include saport.h.

Comments

If the miniport driver does not implement a particular callback, the member for that callback must be set to NULL. The miniport driver's DriverEntry function allocates the memory for SAPORT_INITIALIZATION_DATA and calls the SA port driver's SaPortInitialize function to provide the port driver with information about the miniport driver.
