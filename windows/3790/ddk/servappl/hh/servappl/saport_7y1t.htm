SaPortCreateBasenamedEvent

The SaPortCreateBasenamedEvent function creates a named event object that an SA miniport driver can use to communicate with applications.

NTSTATUS
  SaPortCreateBasenamedEvent(
    IN PVOID MiniPortDeviceExtension,
    IN PWSTR EventNameString,
    IN OUT PKEVENT *Event,
    IN OUT PHANDLE EventHandle
    );

Parameters

MiniPortDeviceExtension
Pointer to a driver-allocated area in which the miniport driver maintains the state of the SA device.
EventNameString
Pointer to a string of 16-bit characters that contains the name of the event.
Event
Caller-supplied pointer to a buffer that receives a pointer to a KEVENT object that SaPortCreateBasenamedEvent initializes using the event name at EventNameString.
EventHandle
Caller-supplied pointer to a buffer that receives a HANDLE object that SaPortCreateBasenamedEvent sets to the handle to the event object.

Return Value

Returns STATUS_SUCCESS on success; otherwise, it returns one of the error status values defined in ntstatus.h.
If SaPortCreateBasenamedEvent is not able to allocate a buffer to hold the event object's security descriptor, STATUS_INSUFFICIENT_RESOURCES is returned.

Headers

Declared in saport.h. Include saport.h.

Comments

An SA miniport driver calls the SaPortCreateBasenamedEvent function to create a named event. An application then waits for the miniport driver to call the KeSetEvent function to set the event to the signaled state. Once the event is set to the signaled state, the application can begin to process information from the miniport driver.

Callers of SaPortCreateBasenamedEvent must be running at IRQL < DISPATCH_LEVEL.
