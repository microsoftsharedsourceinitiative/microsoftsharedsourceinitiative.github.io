IOCTL_NVRAM_WRITE_BOOT_COUNTER

Operation

The IOCTL_NVRAM_WRITE_BOOT_COUNTER request writes a value to a boot counter of a nonvolatile random access memory (NVRAM) device. The SA port driver handles this request by calling the NVRAM miniport driver's SaWrite function.

Input

The SA port driver specifies a pointer to the SA_NVRAM_BOOT_COUNTER structure in the DataBuffer parameter of the miniport driver's SaWrite function. The Number member of this structure specifies the number of the boot counter to write to. The Value member specifies the count value to write to the boot counter. The DeviceId member specifies the Advanced RISC Computing (ARC) path to write to the boot counter. For more information, see NVRAM.

Output

None

I/O Status Block

The returned NTSTATUS value is set to STATUS_SUCCESS for a successful I/O operation; otherwise, the value is set to one of the error status values defined in ntstatus.h.

Headers

Defined in saio.h. Include aio.h.

Comments

Applications can use IOCTL_NVRAM_WRITE_BOOT_COUNTER in conjunction with the watchdog timer to force a fail over to another bootable system partition. For example, suppose an application used IOCTL_NVRAM_WRITE_BOOT_COUNTER to write the value 4 to the first boot counter. Thereafter, when the watchdog timer fires, the BIOS fails over to the second bootable system partition.
