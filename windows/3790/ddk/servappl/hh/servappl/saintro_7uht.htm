Reporting the Version of the SA Interface

Each SA miniport driver reports the version of the SA interface that it supports to ensure proper interoperability with applications and the SA port driver. An application uses the IOCTL_SA_GET_VERSION code to retrieve the version of the SA interface that the miniport driver supports. When the SA port driver receives this IOCTL code for servicing, the port driver must pass the IOCTL code's corresponding function code in a call to the miniport driver's SaDeviceIoctl function. The miniport driver then returns a pointer to a ULONG in the OutputBuffer parameter. The value in this ULONG indicates the SA interface version. The current version of the SA interface is defined as SA_INTERFACE_VERSION (0x00000004).
