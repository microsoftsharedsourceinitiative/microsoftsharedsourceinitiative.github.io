Callback Functions Supplied by the Local Display Miniport Driver

The following table lists the callback functions that a local display miniport driver must supply.

Function Description
DriverEntry Initializes the local display miniport driver and registers it with the SA port driver.
SaHwInitialize Sets up the local display device for I/O operations.
SaDeviceIoctl Uses IOCTL_SA_GET_VERSION to retrieve the version of the SA interface that the miniport driver supports and IOCTL_SA_GET_CAPABILITIES to retrieve the capabilities of the local display device.
SaWrite Transfers display information from an application to the local display device.

The following table lists callback functions that a local display miniport driver must supply if the display must generate interrupts for additional data. That is, supplying these functions depends on the capabilities of the hardware.

Function Description
SaInterruptServiceRoutine Quickly services an interrupt from the local display device.
SaIsrForDpcRoutine Performs the deferred processing of an outstanding interrupt operation.

The following table lists callback functions that a local display miniport driver supplies if the driver uses a file-system context area for the local display device. For more information, see the sample local display miniport driver that is described in the Sample Server Availability Drivers section.

Function Description
SaCreateRoutine Initializes a file-system context area that represents the local display device.
SaCloseRoutine Releases resources for a file-system context area that represents the local display device.
