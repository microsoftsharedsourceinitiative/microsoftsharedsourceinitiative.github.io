Server Availability I/O Control Codes

Server availability (SA) miniport drivers respond to the SA I/O control (IOCTL) codes that are described in this section. Each SA IOCTL code is derived from an SA function code. The SA port driver passes SA function codes to the SA miniport driver's SaDeviceIoctl callback function for processing. The SA port driver can pass input information for the I/O operation in SaDeviceIoctl's InputBuffer parameter. The SA port driver can pass input information and receive output information for the I/O operation in SaDeviceIoctl's OutputBuffer parameter. The SA port driver receives status about the I/O operation in SaDeviceIoctl's return value.

The structures mentioned in the following reference pages are described in Server Availability Miniport Driver Structures.

Each instance of IRP_MJ_DEVICE_CONTROL that is submitted to the miniport driver's SaDeviceIoctl callback function has the Parameters.DeviceIoControl.IoControlCode in the driver’s I/O stack location of the IRP set with one of the IOCTL codes in this section. For more information about the internal structure of IRPs, see Handling IRPs.
