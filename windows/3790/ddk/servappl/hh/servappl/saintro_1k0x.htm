Creating the Device Extension for the NVRAM Device

The NVRAM miniport driver must define a device extension structure that contains information about the hardware resources and the current state of the NVRAM device. This structure includes the following members:

  • A pointer to a DEVICE_OBJECT object. The NVRAM miniport driver assigns this object pointer to the NVRAM device's object pointer during initialization. The SA port driver creates the NVRAM device's object and passes a pointer to this object in a call to the miniport driver's SaHwInitialize function.
  • An internal buffer to hold system status, which is transferred between the requesting application and the NVRAM device.
  • A variable to hold the size of the internal buffer.
  • A variable to hold a flag that indicates whether the system status was received from the NVRAM device.
  • An object, such as a spin lock or a mutex, to synchronize access to members of this device extension structure.

The device extension can also contain additional members to support queries or data transfer operations that are beyond the basic functionality of an NVRAM miniport driver.

The NVRAM miniport driver's callback functions subsequently use this device extension structure to execute I/O operations with the NVRAM device.
