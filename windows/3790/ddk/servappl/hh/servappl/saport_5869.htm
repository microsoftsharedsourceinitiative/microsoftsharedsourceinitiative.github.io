SaPortReadBinaryRegistryValue

The SaPortReadBinaryRegistryValue function reads the data contained in a binary-type registry value given the registry value's name.

NTSTATUS
  SaPortReadBinaryRegistryValue(
    IN PVOID MiniPortDeviceExtension,
    IN PWSTR ValueName,
    OUT PVOID RegistryValue,
    IN OUT PULONG RegistryValueLength
    );

Parameters

MiniPortDeviceExtension
Pointer to a driver-allocated area in which the miniport driver maintains the state of the SA device.
ValueName
Pointer to a buffer that contains a null-terminated string of 16-bit Unicode characters that contains the name of the registry value from which information is read.
RegistryValue
Caller-supplied pointer to a buffer that receives the binary data that is contained in the specified value. If this buffer is too small, SaPortReadBinaryRegistryValue returns STATUS_BUFFER_TOO_SMALL and sets the value at RegistryValueLength to the required size.
RegistryValueLength
Caller-supplied pointer to a variable that receives the size, in bytes, of the data that is stored in the buffer at RegistryValue.

Return Value

Returns STATUS_SUCCESS on success; otherwise, returns one of the error status values defined in ntstatus.h.
Returns STATUS_OBJECT_TYPE_MISMATCH if the type of the value read is not REG_BINARY.

Headers

Declared in saport.h. Include saport.h.

Comments

The SaPortReadBinaryRegistryValue function reads binary (REG_BINARY) data in any form.

Callers of SaPortReadBinaryRegistryValue must be running at IRQL = PASSIVE_LEVEL.
