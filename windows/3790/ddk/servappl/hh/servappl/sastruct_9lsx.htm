SA_DISPLAY_CAPS

The SA_DISPLAY_CAPS structure contains the capabilities of a local display device. 

typedef struct _SA_DISPLAY_CAPS {
  ULONG  SizeOfStruct;
  SHORT  DisplayType;
  USHORT  CharacterSet;
  USHORT  DisplayHeight;
  USHORT  DisplayWidth;
} SA_DISPLAY_CAPS, *PSA_DISPLAY_CAPS;

Members

SizeOfStruct
Specifies the size in bytes of this structure.
DisplayType
Specifies the local display type that the hardware and miniport driver support. This type must be set to one of the constants in the following table.
Display type Meaning
SA_DISPLAY_TYPE_LED
0x0001		 The local display device consists of light emitting diodes (LED).
SA_DISPLAY_TYPE_CHARACTER_LCD
0x0002		 The local display device is a liquid crystal type that displays text characters. This type is currently not supported.
SA_DISPLAY_TYPE_BIT_MAPPED_LCD
0x0004		 The local display device is a liquid crystal type that displays bit maps.

CharacterSet
Currently not supported.
Specifies one of the following constants to indicate whether the miniport driver's device displays ASCII (8-bit) or Unicode (16-bit) characters:

SA_DISPLAY_CHAR_ASCII                0x0001

SA_DISPLAY_CHAR_UNICODE            0x0002

DisplayHeight
Specifies the display height in pixels. The maximum height that a local display can be is 128 pixels. The miniport driver uses SA_DISPLAY_MAX_HEIGHT to specify maximum height.
DisplayWidth
Specifies the display width in pixels. The maximum width that a local display can be is 128 pixels. The miniport driver uses SA_DISPLAY_MAX_WIDTH to specify maximum width.

Headers

Defined in saio.h. Include saio.h.

Comments

The SA port driver passes the IOCTL_SA_GET_CAPABILITIES code's corresponding function code in the FunctionCode parameter of a local display miniport driver's SaDeviceIoctl function to retrieve the capabilities of the local display device. The miniport driver then returns a pointer to a SA_DISPLAY_CAPS structure in the OutputBuffer parameter.
