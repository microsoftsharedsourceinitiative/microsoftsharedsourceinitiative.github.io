Introduction to Stream Pointers

A stream pointer is a reference to an AVStream data frame.

Minidrivers that use filter-centric processing should not directly use stream pointers. For minidrivers that use pin-centric processing, the stream pointer abstraction provides many different ways to access stream data.

The following rules apply to stream pointers and frames:

  • Any number of stream pointers may reference a frame.
  • A frame that is referenced by one or more stream pointers does not leave the queue in which it resides unless the frame is canceled (that is, unless the IRP that contains the frame is canceled).
  • A frame that is referenced by one or more locked stream pointers cannot be canceled. A frame that is not referenced by any locked stream pointers may be canceled.
