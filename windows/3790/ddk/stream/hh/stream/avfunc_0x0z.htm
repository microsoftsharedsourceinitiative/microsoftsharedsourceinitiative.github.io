KsFilterRegisterPowerCallbacks

The KsFilterRegisterPowerCallbacks function registers power management callbacks for Filter.

VOID
  KsFilterRegisterPowerCallbacks(
    IN PKSFILTER  Filter,
    IN PFNKSFILTERPOWER  Sleep OPTIONAL
    IN PFNKSFILTERPOWER  Wake OPTIONAL
    );

Parameters

Filter
Pointer to the KSFILTER structure for which to register power callbacks. In order to receive power notification messages, the filter must be capable of processing, so Filter must be filter-centric.
Sleep
Pointer to a function that handles sleep requests for the device. If NULL, no sleep callback is registered. For more information, see the Comments section below.
Wake
Pointer to a function that handles wake requests for the device. If NULL, no wake callback is specified. For more information, see the Comments section below.

Return Value

None

Headers

Declared in ks.h. Include ks.h.

Comments

This function is available in Windows XP and DirectX 8.0 and later.

The two callbacks should be prototyped as follows:

void Sleep/Wake (IN PKSFILTER Filter, IN DEVICE_POWER_STATE State);

The Sleep callback is made if Filter is a filter-centric filter and the device is going to sleep. The Wake callback is made if Filter is a filter-centric filter and the device is waking.

For information on device power states, see Device Power States.

Also see Initializing an AVStream Minidriver and Filter-Centric Processing.

Do not attempt to obtain the filter control mutex from within either the Sleep or Wake callback, or deadlock may occur. For more information on mutexes, read Mutexes in AVStream.

KsFilterRegisterPowerCallbacks must be called at IRQL = PASSIVE_LEVEL.
