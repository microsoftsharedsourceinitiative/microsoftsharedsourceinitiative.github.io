Vendor's GFX User Interface

A vendor who provides a GFX filter should also provide a COM user-interface object that supports the ISpecifyPropertyPages interface. For information about this interface, see the Platform SDK documentation. The CLSID of this COM object is specified in the GFX driver’s installation INF file. For more information, see the description of the UserInterface/CLSID subkey in Installing a Device-Specific GFX Filter.

In addition, the vendor should provide property-page COM objects that implement the IPropertyPage interface for each property-page CLSID that is returned by the UI object’s ISpecifyPropertyPages interface. Vendors can choose appropriate mechanisms for implementing, installing, and registering these COM objects. For information about the IPropertyPage interface, see the Platform SDK documentation.

The Multimedia control-panel passes a data object to the property pages by calling IPropertyPage::SetObjects. The vendor’s property-page object can call IDataObject::GetData with the medium type specified as TYMED_HGLOBAL. This call creates a new handle to the vendor’s GFX filter. The vendor’s property page objects can use this handle to access the GFX filter's KS properties, events, and methods. The property-page object is responsible for ensuring that the handle is eventually closed by calling CloseHandle. For more information about IDataObject::GetData and CloseHandle, see the Platform SDK documentation.
