StreamClassFilterReenumerateStreams

The minidriver calls StreamClassFilterReenumerateStreams to cause the class driver to reenumerate all streams on the filter instance. 

VOID
   StreamClassFilterReenumerateStreams(
    IN PVOID  HwDeviceExtension,
    IN ULONG StreamDescriptorSize
    );

Parameters

HwDeviceExtension
Pointer to the minidriver's device extension. The minidriver specifies the size of this buffer in the HW_INITIALIZATION_DATA structure it passes when it registers itself via StreamClassRegisterMinidriver. The class driver then passes pointers to the buffer in the HwDeviceExtension member of the HW_STREAM_REQUEST_BLOCK, HW_STREAM_OBJECT, HW_TIME_CONTEXT, and PORT_CONFIGURATION_INFORMATION structures it passes to the minidriver.
StreamDescriptorSize
Indicates the size of the buffer needed by the minidriver to hold the new HW_STREAM_DESCRIPTOR data for the filter.

Return Value

None

Headers

Declared in strmini.h. Include strmini.h.

Comments

To reenumerate all streams on the device, rather than only on the filter, the minidriver should call StreamClassReenumerateStreams.

Use this routine to increase the number of externally exposed pins so that applications can make connections on recently exposed streams. It is the responsibility of the caller to maintain the order of any connected streams.
