USB Camera with a Still Pin

A slightly more complex graph is created for a conferencing camera that exposes a “still” pin. This still pin can provide a higher resolution image when the user pushes a button on the camera.

Microsoft provides a minidriver library for USB cameras (USBCAMD) that supports cameras with still pins. For information about USBCAMD, see USB Camera Minidrivers and USB Camera Minidriver Library.

USB Video Camera Filter Graph

Here, the still pin streams only a single image when the user pushes the button on the camera. Alternatively, the still pin can be triggered by programmatic control.

The Windows Image Acquisition (WIA) technology built upon the Still Image Architecture (STI) complements the functionality provided by USBCAMD. See Windows Image Acquisition Drivers and Still Image Drivers for more information.

The WIA Video Snapshot filter is an addition to WIA that shipped with Windows XP and later operating systems. It is a filter driver that enables still frames to be captured from the video stream.

There are two methods of capturing a still from the device. The first is to insert the WIA Video Snapshot filter downstream from the capture filter and trigger a capture programmatically. The second is to enable still pin support using the USBCAMD model. The WIA Video Snapshot filter can then be triggered by pushing a button on the device.

The advantage of capturing an image from the still pin as opposed to the video stream is that the still pin can provide a higher resolution image in addition to permitting the user to capture an image by pressing a button on the device.

Driver writers can add still pin support to their minidrivers if they want to enable users to trigger captures via a hardware button on device and/or the device can capture higher resolution still. If support isn't explicitly added, the WIA Video Snapshot filter can be software triggered, but the resolution will be the same resolution as the video stream.

Some still pin implementations can only be rendered after the capture pin because they are based on the capture pin data formats.

For more information on WIA driver development see the still imaging technologies Web site.
