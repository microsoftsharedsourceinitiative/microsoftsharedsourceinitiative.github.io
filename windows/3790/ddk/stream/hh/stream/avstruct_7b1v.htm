KSSTREAM_POINTER_OFFSET

The KSSTREAM_POINTER_OFFSET structure describes the offset of the stream pointer into the data stream. It also describes the amount of data available for input/output or the mappings for scatter/gather DMA.

typedef struct {
    union {
        PUCHAR  Data;
        PKSMAPPING  Mappings;
    };
    ULONG  Count;
    ULONG  Remaining;
} KSSTREAM_POINTER_OFFSET, *PKSSTREAM_POINTER_OFFSET;

Members

Data
Pointer to a buffer containing input data or a buffer to which data is to be output. Specify Data if the client has not specified that this pin is to perform automatic scatter/gather DMA mapping generation via KSPIN_FLAG_GENERATE_MAPPINGS.
Mappings
Pointer to the first element in an array of KSMAPPING structures. Specify Mappings if the client has specified that this pin is to perform automatic scatter/gather DMA mapping generation via KSPIN_FLAG_GENERATE_MAPPINGS. In this case, Mappings describes each individual contiguous chunk of physical memory that the client can use to set up a scatter/gather DMA operation with hardware.
Count
Contains either the number of bytes of data available in the Data buffer or the number of mappings in the Mappings table depending on whether this pin has specified mapping generation or not.
Remaining
Used by the client to describe how many bytes of data have been read/written or how many scatter/gather mappings have been read from/written to depending on whether this pin has specified mapping generation or not.

Headers

Declared in ks.h. Include ks.h.

Comments

This structure is available in Windows XP and later operating systems and in DirectX 8.0 and later versions.
