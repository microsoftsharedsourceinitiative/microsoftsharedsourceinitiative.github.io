SRB_UNKNOWN_DEVICE_COMMAND

The class driver sends this request to pass IRPs that it does not know how to handle to the minidriver. pSRB->Irp points to the unhandled IRP. See HW_STREAM_REQUEST_BLOCK.

This request can be used as an IRP pass-through mechanism for IRPs that the stream class does not understand, but that the minidriver might. For example, there are several PNP IRPs that the stream class does not process, but that a 1394 camera driver does.
