CamProcessRawVideoFrame

This callback function is obsolete. New camera minidrivers should use the USBCAMD_DEVICE_DATA2 structure and provide a CamProcessRawVideoFrameEx callback function instead.

A camera minidriver's CamProcessRawVideoFrame callback function decodes a raw video frame.

NTSTATUS
  CamProcessRawVideoFrame(
    PDEVICE_OBJECT  BusDeviceObject,
    PVOID  DeviceContext,
    PVOID  FrameContext,
    PVOID  FrameBuffer,
    ULONG  FrameLength,
    PVOID  RawFrameBuffer,
    ULONG  RawFrameLength,
    ULONG  NumberOfPackets,
    PULONG  BytesReturned
    );

Parameters

BusDeviceObject
Pointer to the camera minidriver's device object created by the USB hub.
DeviceContext
Pointer to the camera minidriver's device context.
FrameContext
Pointer to the camera minidriver's frame context.
FrameBuffer
Pointer to the buffer that receives the final processed video frame.
FrameLength
Specifies the length of the frame buffer (from the original read request) in bytes.
RawFrameBuffer
Pointer to the buffer containing the received USB packets.
RawFrameLength
Specifies the length of RawFrameBuffer in bytes.
NumberOfPackets
Specifies the number of USB packets received into RawFrameBuffer.
BytesReturned
Pointer to the number of bytes read. The minidriver must set this to zero if it encounters any errors during processing.

Return Value

CamProcessRawVideoFrame returns STATUS_SUCCESS or an appropriate error code.

Headers

Declared in usbcamdi.h. Include usbcamdi.h.

Comments

Camera minidrivers that must maintain backwards compatibility with USBCAMD version 1.0 must use the USBCAMD_DEVICE_DATA structure and its associated callback functions (that is, callback functions that don't end in "Ex").

USBCAMD calls the camera minidriver's CamProcessRawVideoFrame callback function at IRQL = PASSIVE_LEVEL.

This function is optional.
