USBCAMD Callback Functions

This section describes the callback function a camera minidriver implements to use the USBCAMD library. A minidriver sets its entry points to CamXxx callback functions it defines in a USBCAMD_DEVICE_DATA2 structure that the minidriver passes to USBCAMD during minidriver initialization.

Camera minidrivers that must maintain backwards compatibility with USBCAMD version 1.0 must use the USBCAMD_DEVICE_DATA structure and its associated callback functions (that is, callback functions that don't end in "Ex").

In the header file usbcamdi.h, the following callback functions are supplied as prototypes. For example:

typedef
NTSTATUS
  (*PCAM_ALLOCATE_BW_ROUTINE)(
    PDEVICE_OBJECT  BusDeviceObject,
    PVOID  DeviceContext,
    PULONG  RawFrameLength,
    PVOID  Format
    );

A camera minidriver may name its callback functions whatever names the minidriver writer chooses. For example, (*PCAM_ALLOCATE_BW_ROUTINE) is named CamAllocateBandwidth below, but could also be named MinidriverAllocBW.

USBCAMD version 1.0 supports only the following callback functions. Required and optional implementation is documented with each topic:

CamAllocateBandwidth

CamConfigure

CamFreeBandwidth

CamInitialize

CamNewVideoFrame

CamProcessRawVideoFrame

CamProcessUSBPacket

CamRestoreState

CamSaveState

CamStartCapture

CamStopCapture

CamUnInitialize

USBCAMD version 2.0 adds the following callback functions. Required and optional implementation is documented with each topic:

CamAllocateBandwidthEx

CamConfigureEx

CamFreeBandwidthEx

CamNewVideoFrameEx

CamProcessRawVideoFrameEx

CamProcessUSBPacketEx

CamStartCaptureEx

CamStopCaptureEx

The following callback functions are passed as arguments to certain USBCAMD services:

AdapterReceivePacket

CommandCompleteFunction
