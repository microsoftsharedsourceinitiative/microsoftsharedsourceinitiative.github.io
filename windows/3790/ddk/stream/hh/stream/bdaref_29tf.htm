BdaStartChanges

The BdaStartChanges function initiates the setting of new BDA topology changes. 

STDMETHODIMP_(NTSTATUS) 
  BdaStartChanges(
    IN PIRP  Irp
    );

Parameters

Irp
Points to the IRP for the request to initiate changes.

Return Value

Returns STATUS_SUCCESS or an appropriate error code.

Headers

Declared in Bdasup.h. Include Bdasup.h.

Comments

A BDA minidriver calls the BdaStartChanges function to initiate the setting of new BDA topology changes after the minidriver receives a KSMETHOD_BDA_START_CHANGES request of the KSMETHODSETID_BdaChangeSync method set from the network provider. BDA minidrivers define dispatch and filter-automation tables so that those minidrivers either dispatch the BdaStartChanges function directly or intercept this request using an internal method, which then calls the BdaStartChanges function. For example, BDA minidrivers that intercept this request can obtain a pointer to the BDA filter from the passed IRP so that they can subsequently reset pending filter resources to the new requirements and set the change state of the filter to BDA_CHANGES_COMPLETE. See Defining Automation Tables and Changing BDA Filter Properties for more information.

Calling the BdaStartChanges function causes any previously requested topology changes that have not yet been committed to be ignored.

A BDA minidriver calls BdaStartChanges at IRQL = PASSIVE_LEVEL.

BdaStartChanges is available on Windows® XP and later. BdaStartChanges is available on the Windows 2000 platform only if DirectX 9.0 or later is installed on that platform.
