BDA Node Category GUIDs

A BDA minidriver uses the BDA node category GUIDs to specify the types of BDA nodes that are available to create. A BDA minidriver assigns one of these GUIDs in the variable to which the Type member of a KSNODE_DESCRIPTOR structure points. The Bdamedia.h header file defines these GUIDs.

The network provider uses the KSPROPERTY_BDA_NODE_TYPES property of the KSPROPSETID_BdaTopology property set to retrieve a list of nodes types available from the BDA minidriver. This list of node types is an array of KSNODE_DESCRIPTOR structures.

The following node category GUIDs are available in BDA:

KSNODE_BDA_RF_TUNER
A BDA minidriver assigns this GUID to specify that a BDA radio frequency tuner node is available.
KSNODE_BDA_QAM_DEMODULATOR
A BDA minidriver assigns this GUID to specify that a BDA QAM-type demodulator node is available.
KSNODE_BDA_QPSK_DEMODULATOR
A BDA minidriver assigns this GUID to specify that a BDA QPSK-type demodulator node is available.
KSNODE_BDA_8VSB_DEMODULATOR
A BDA minidriver assigns this GUID to specify that a BDA 8VSB-type demodulator node is available.
KSNODE_BDA_OPENCABLE_POD
A BDA minidriver assigns this GUID to specify that a BDA open-cable POD node is available.
KSNODE_BDA_PID_FILTER
A BDA minidriver assigns this GUID to specify that a BDA packet identifier (PID) filter node is available.
KSNODE_BDA_IP_SINK
A BDA minidriver assigns this GUID to specify that a BDA IP sink node is available.
