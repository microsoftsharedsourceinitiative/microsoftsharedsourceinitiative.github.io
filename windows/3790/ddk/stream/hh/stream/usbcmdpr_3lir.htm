CamNewVideoFrame

This callback function is obsolete. New camera minidrivers should use the USBCAMD_DEVICE_DATA2 structure and provide a CamNewVideoFrameEx callback function instead.

A camera minidriver's CamNewVideoFrame callback function initializes a new video frame context structure.

VOID 
  CamNewVideoFrame(
    PVOID  DeviceContext,
    PVOID  FrameContext
    );

Parameters

DeviceContext
Pointer to the camera minidriver's device context.
FrameContext
Pointer to the camera minidriver's frame context.

Return Value

CamNewVideoFrame does not return a value.

Headers

Declared in usbcamdi.h. Include usbcamdi.h.

Comments

Camera minidrivers that must maintain backwards compatibility with USBCAMD version 1.0 must use the USBCAMD_DEVICE_DATA structure and its associated callback functions (that is, callback functions that don't end in "Ex").

USBCAMD calls the camera minidriver's CamNewVideoFrame callback function at IRQL = DISPATCH_LEVEL.

This function is optional.
