KS Methods

Like COM objects, KS objects can support methods. For example, allocator objects support methods to allocate or deallocate memory.

Use methods to perform actions on KS objects, either synchronously or asynchronously. Method sets allow related actions to be grouped together. Each KSMETHOD_SET contains an array of KSMETHOD_ITEM structures that describe single methods.

Clients make method requests by calling DeviceIoControl with the IOCTL_KS_METHOD I/O control code.

If a filter or pin must support multiple distinct copies of the same method set, it must use instance identifiers.

The expected parameter passing type is determined by the definition within the filter, and must be known to the client. The filter uses the first group of flags to define the type of method. The client uses the second set of flags when initiating a method request. These flags indicate what action, if any, takes place on the second buffer of the device I/O control.

The request is passed using the KSMETHOD structure, which may contain instance information appended to the structure. A method-specific buffer may be passed.

The KsMethodHandler routine handles method requests received through IOCTL_KS_METHOD.
