BdaMethodDeletePin

The BdaMethodDeletePin function deletes a pin factory. 

STDMETHODIMP_(NTSTATUS) 
  BdaMethodDeletePin(
    IN PIRP  Irp,
    IN KSMETHOD  *pKSMethod,
    OPTIONAL PVOID  pvIgnored
    );

Parameters

Irp
Points to the IRP for the request to delete a pin factory.
pKSMethod
Points to a KSMETHOD structure that describes the method and request type of a method request.
pvIgnored
Points to a buffer that is ignored.

Return Value

Returns STATUS_SUCCESS or an appropriate error code.

Headers

Declared in Bdasup.h. Include Bdasup.h.

Comments

A BDA minidriver calls the BdaMethodDeletePin function to delete a pin factory after the minidriver receives a KSMETHOD_BDA_DELETE_PIN_FACTORY request of the KSMETHODSETID_BdaDeviceConfiguration method set from the network provider. Most BDA minidrivers can define dispatch and filter-automation tables so that those minidrivers dispatch the BdaMethodDeletePin function directly. See Defining Automation Tables and Configuring a BDA Filter for more information.

If a BDA minidriver must delete a pin without relying on the network provider, the BDA minidriver should call the BdaDeletePin function.

A BDA minidriver calls BdaMethodDeletePin at IRQL = PASSIVE_LEVEL.

BdaMethodDeletePin is available on Windows® XP and later. BdaMethodDeletePin is available on the Windows 2000 platform only if DirectX 9.0 or later is installed on that platform.
