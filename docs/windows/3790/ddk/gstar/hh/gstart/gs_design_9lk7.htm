Getting Started with Windows Drivers: Windows DDK

Using the DDK Samples

The Windows® DDK includes sample source code for a variety of drivers. The samples can be useful guides for writing your own drivers. When you install the DDK, samples are installed in subdirectories of the \src directory.

An introductory HTML file is provided with each sample. The HTML file describes what the sample does, which operating systems it has been tested on, and how to build it. The HTML file also lists the files that make up each sample.

The following table lists some of the sample subdirectories and their contents.

Sample Subdirectory Contents
src\general Various samples of kernel-mode drivers, including the comprehensive "toaster" sample.
src\ime Input Method Editor (IME) DLLs and samples.
src\input Keyboard and mouse class filter drivers.
src\kernel Additional samples of kernel-mode drivers.
src\mmedia Audio Compression Manager (ACM) software codecs.
src\network Samples for various types of network drivers.
src\print Sample drivers, plus additional DLLs and tools, for printers and plotters.
src\setup Sample INF files and installation applications.
src\smartcard Drivers for serial and PCMCIA Smartcard readers.
src\storage Sample drivers for various types of storage devices.
src\vdd Virtual device drivers for DOS compatibility.
src\video Sample display drivers and a mirroring driver.
src\wdm Sample drivers for a variety of devices that are supported by the Windows Driver Model (WDM). Device types that are supported by these samples include audio, A/V streaming, and human interface devices, among others.

Also includes a WMI filter for any WDM driver.

For more information about DDK samples, see the device type-specific documentation in this DDK, or choose the DDK viewer's Index tab and type "sample".

If you choose to base your code on one or more of the samples, keep in mind the following:

  • Some samples are not complete drivers, but instead provide guidelines for how such drivers could be written. For instance, samples might omit error-handling code in the interest of brevity and clarity.
  • Make sure to use a driver sample that was written for the type of driver that you are writing. For example, if you are writing a WDM function driver, use a sample WDM function driver. If you are writing a storage filter driver, use a sample storage filter driver.
  • Be sure to update the sample INF file for your driver. In particular, make sure that the device ID of your hardware matches the device ID that is specified in the INF file.
