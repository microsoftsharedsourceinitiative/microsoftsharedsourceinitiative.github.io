IOCTL_KS_RESET_STATE

Operation

An application can use IOCTL_KS_RESET_STATE to return a pin to the state it was in at Acquire-time. The application passes IOCTL_KS_RESET_STATE with the parameters described below to the KsSynchronousDeviceControl function.

Input

A client provides a buffer that contains the reset value (KSRESET_BEGIN, KSRESET_END) in the InBuffer parameter. InLength specifies size, in bytes, of the input buffer contents.

Output

None

Headers

Defined in ks.h. Include ks.h.

Comments

A reset request propagates along pin connections. During this process, additional reset requests are ignored. Consequently, a reset on a pin can affect the filter as a whole.

The reset state is not canceled by changing the pin state. This indicates to a pin that new data can be accepted. It also indicates that the pin can again pass data to its previous connection, even if data had been rejected in a prior attempt.
