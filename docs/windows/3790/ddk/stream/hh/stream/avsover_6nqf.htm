Locking and Unlocking Stream Pointers

Each pointer into the data stream can be in one of two states: locked or unlocked. A locked stream pointer points to valid data. An unlocked stream pointer, however, may not point at data. It is invalid for a client minidriver to attempt to access the data referenced by the stream pointer while it is unlocked. The minidriver must first call KsStreamPointerLock. To unlock a stream pointer, use KsStreamPointerUnlock.
