Allocating an IRP

The style of IRP management you use influences how to obtain an IRP for use with avc.sys. Usually IoAllocateIrp is sufficient. Never call IoInitializeIrp on an IRP you allocated by IoAllocateIrp. If you are trying to reinitialize to reuse an IRP, call IoFreeIrp on the old IRP, and then IoAllocateIrp to get a new IRP.

Note that using ExAllocatePoolWithTag and ExFreePool alone eventually fragments the nonpaged pool, which decreases system throughput.

If you plan to allocate many IRPs simultaneously, the global resource used by IoAllocateIrp could be depleted, resulting in decreased system throughput. In this situation, you should either use a zone buffer, or a lookaside list to manage IRP storage.
