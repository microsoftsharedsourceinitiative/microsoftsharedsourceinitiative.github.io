Adapter and Stream Property Sets

Property sets are classified as belonging to either the adapter or to a particular stream. Adapter property sets are global with respect to the device and affect all streams. For example, PROPSETID_VIDCAP_CAMERACONTROL, which is used for camera positioning, affects all output streams and is therefore classified as an adapter property set. Note that some property sets such as PROPSETID_VIDCAP_VIDEOCOMPRESSION (which controls the compression behavior of a single output stream) are implemented as an adapter property that includes a stream index. This is required because stream property sets are unavailable when the pin is not connected. Adapter property sets are always available.

The following table lists the primary property sets used by video capture minidrivers. It also indicates whether the property is at the adapter or stream level and whether the property set is required.

At a minimum, a minidriver must support properties to control buffer allocation and to report the number of frames dropped during capture as noted in the table above. Support for all other property sets is optional, depending on the capabilities of the device. It is strongly recommended that cameras, which offer only a limited set of capture frame rates, implement the PROPSETID_VIDCAP_VIDEOCONTROL to allow video conferencing applications to make optimal use of system bandwidth.
