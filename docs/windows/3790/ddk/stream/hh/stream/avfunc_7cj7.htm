KsPinGetLeadingEdgeStreamPointer

The KsPinGetLeadingEdgeStreamPointer function acquires the leading edge stream pointer for the queue associated with the given pin.

PKSSTREAM_POINTER
  KsPinGetLeadingEdgeStreamPointer(
    IN PKSPIN  Pin,
    IN KSSTREAM_POINTER_STATE  State
    );

Parameters

Pin
Pointer to the KSPIN owning the queue for which the leading edge stream pointer should be acquired.
State
Specifies how to acquire the leading edge stream pointer. Can be one of the following:
KSSTREAM_POINTER_STATE_UNLOCKED
Acquire the leading edge stream pointer regardless of whether it references a data frame or not.

No attempts can be made to access any data associated with the pointer until the pointer is locked. Also note that frames associated with an unlocked stream pointer can be cancelled.

KSSTREAM_POINTER_STATE_LOCKED
Acquire and lock the leading edge stream pointer. If no data frame is associated with the stream pointer, return NULL. If a non-null pointer is returned, it is a locked stream pointer and has a data frame associated with it. Frames associated with a locked stream pointer cannot be cancelled.

Return Value

KsPinGetLeadingEdgeStreamPointer returns a pointer to a KSSTREAM_POINTER or NULL. A return value of NULL may occur because there is no queue associated with the pin, indicating that the pin does not use the standard transport mechanism. Alternatively, a return value of NULL may occur because an attempt to lock the leading edge failed, indicating that there is no data frame associated with the leading edge.

Headers

Declared in ks.h. Include ks.h.

Comments

This function is available in Windows XP and DirectX 8.0 and later.

Filters that implement pin-centric processing often call KsPinGetLeadingEdgeStreamPointer.

KsPinGetLeadingEdgeStreamPointer must be called at IRQL <= DISPATCH_LEVEL.
