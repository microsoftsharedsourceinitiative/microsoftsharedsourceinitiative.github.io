KsPinGetCopyRelationships

The KsPinGetCopyRelationships function allows a pin-centric splitter to obtain copy relationship information about a specific pin.

VOID
  KsPinGetCopyRelationships(
    IN PKSPIN Pin,
    OUT PKSPIN* CopySource,
    OUT PKSPIN* DelegateBranch
    )

Parameters

Pin
Pointer to the KSPIN about which to acquire copy information.
CopySource
Pointer to a pointer to a KSPIN structure that is the copy source for Pin. If Pin is the copy source, AVStream sets this parameter to NULL.
DelegateBranch
Pointer to a pointer to a KSPIN structure that is the pin from which Pin receives delegated frames. If Pin is the delegator, AVStream sets this parameter to NULL.

Return Value

None

Headers

Declared in ks.h. Include ks.h.

Comments

This function is available in Windows XP and DirectX 8.0 and later.

KsPinGetCopyRelationships returns copy source pin and delegate branch pin information. The information returned corresponds to the CopySource and DelegateBranch fields of a KSPROCESSPIN passed into the processing dispatch of a filter-centric filter, with the sole difference that it returns PKSPIN and not PKSPROCESSPIN. For more on these fields, see AVStream Splitters.

All pins operate independently in the context of a pin-centric filter. As a result, a minidriver that calls KsPinGetCopyRelationships is responsible for ensuring that the appropriate synchronization is performed before call-time. To guarantee safety when calling KsPinGetCopyRelationships, either obtain the control mutex (do not use this mechanism in a processing dispatch) or make sure that the pin does not transition below KSSTATE_PAUSE while calling or using the information obtained. For more information on mutexes, see Mutexes in AVStream.

For more information, see Pin-Centric Processing and Filter-Centric Processing.
