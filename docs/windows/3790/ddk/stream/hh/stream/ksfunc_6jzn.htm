Method Set Functions

This section documents KS-provided macros for method sets and the helper functions for method sets.

The macros include:

KSMETHOD_SET_IRP_STORAGE

KSMETHOD_ITEM_IRP_STORAGE

KSMETHOD_TYPE_IRP_STORAGE

When a device object presents a method set to a client, you can use these helper functions to perform basic parameter validation and routing based on a method set table.

The following helper functions are provided:

KsMethodHandler

KsMethodHandlerWithAllocator

KsFastMethodHandler

Typically, a class driver, such as AVStream, calls a helper function, such as the KsMethodHandler function, to handle a method requested through IOCTL_KS_METHOD. The helper function then determines the appropriate KStrMethodHandler routine (specified in KSMETHOD_ITEM) to call from the KSMETHOD structure that accompanies the IOCTL_KS_METHOD request. This MethodHandler call receives a prepared IRP, with parameters extracted. A pointer to the KSMETHOD_SET to which the method belongs is placed in the KSMETHOD_SET_IRP_STORAGE (Irp) field and the type of method request is placed in KSMETHOD_TYPE_IRP_STORAGE (Irp). This information is used when doing asynchronous processing on the method with KsDispatchSpecificMethod.

A KSMETHOD_ITEM structure can additionally specify a KStrSupportHandler that is used to determine support for the particular method item. A pointer to the KSMETHOD_SET to which the method belongs is placed in the KSMETHOD_SET_IRP_STORAGE(Irp) field before calling the support handler.

When defining methods in a method set, a minidriver uses the MethodHandler and SupportHandler members of the KSMETHOD_ITEM structure to point to its implementations of KStrMethodHandler and KStrSupportHandler respectively.
