Cloning Stream Pointers

Minidrivers can clone stream pointers. Cloning creates a new stream pointer that is identical to the parent: The clone references the same frame and has the same lock status.

Once created, clones are independent of their parents and may be locked, unlocked, deleted or advanced to a different frame. Minidrivers clone stream pointers by calling KsStreamPointerClone.

When a clone is created using KsStreamPointerClone, the minidriver can supply an optional context size. If a non-zero context size is supplied, additional memory is allocated for the context and the stream pointer's Context field is set to point to the allocated context. Because the memory is allocated along with the stream pointer structure, no additional allocation calls are generated by using this feature.

Clone stream pointers associated with a queue can be enumerated (by way of an associated pin) using KsPinGetFirstCloneStreamPointer and KsStreamPointerGetNextClone.

Also see KsStreamPointerDelete.
