CamRestoreState

This callback function is obsolete and is no longer called in the Windows XP version of USBCAMD. If the minidriver needs to save or restore device-specific data it should do so when processing an SRB_CHANGE_POWER_STATE request.

A camera minidriver's CamRestoreState callback function restores a previously saved device context state.

NTSTATUS
  CamRestoreState(
    PDEVICE_OBJECT  BusDeviceObject,
    PVOID  DeviceContext
    );

Parameters

BusDeviceObject
Pointer to the camera minidriver's device object created by the USB hub.
DeviceContext
Pointer to the camera minidriver's device context.

Return Value

CamRestoreState must return STATUS_SUCCESS.

Headers

Declared in usbcamdi.h. Include usbcamdi.h.

Comments

USBCAMD calls the minidriver's CamRestoreState callback function in the context of an SRB_CHANGE_POWER_STATE request when the power state is going from OFF to ON (D3 to D0).

USBCAMD ignores the return value from the minidriver's CamRestoreState.

CamRestoreState is called by both versions 1.0 and 2.0 of USBCAMD.

This function is optional.
