CamFreeBandwidth

This callback function is obsolete. New camera minidrivers should use the USBCAMD_DEVICE_DATA2 structure and provide a CamFreeBandwidthEx callback function instead.

A camera minidriver's CamFreeBandwidth callback function selects an alternate setting within the USB video streaming interface that uses no bandwidth.

NTSTATUS
  CamFreeBandwidth(
    PDEVICE_OBJECT  BusDeviceObject,
    PVOID  DeviceContext
    );

Parameters

BusDeviceObject
Pointer to the camera minidriver's device object created by the USB hub.
DeviceContext
Pointer to the camera minidriver's device context.

Return Value

CamFreeBandwidth returns STATUS_SUCCESS or an appropriate error code.

Headers

Declared in usbcamdi.h. Include usbcamdi.h.

Comments

Camera minidrivers that must maintain backwards compatibility with USBCAMD version 1.0 must use the USBCAMD_DEVICE_DATA structure and its associated callback functions (that is, callback functions that don't end in "Ex").

USBCAMD calls the camera minidriver's CamFreeBandwidth callback function after the isochronous video stream has stopped.

Typically, this function calls the USBCAMD_SelectAlternateInterface service to select the correct alternate interface and prepare for streaming video.

This function is required.
