Peer Driver Enumeration

As AV/C devices are added and removed from the 1394 bus, drivers are loaded and unloaded by the Plug and Play Manager. The following diagram shows the driver stack.

Peer AV/C Device Stack

At the bottom of the stack are 1394ohci.sys and 1394bus.sys. These drivers provide the basic 1394 infrastructure support. There are instances of these drivers for each 1394 adapter in the system.

Next in the stack is 61883.sys. This protocol driver provides the IEEE 61883-1 connection management protocol (CMP), common isochronous packet (CIP), and function control protocol (FCP) support, as well as PnP enumeration of devices for each command transaction set (CTS). There is an instance of this driver for each 61883-enabled node on the 1394 bus.

Next in the stack is avc.sys. This driver provides AV/C protocol support, PnP enumeration of devices for each AV/C subunit, and plug connection management and control.

Next in the stack are the subunit drivers. This is the level where all of the unique AV/C subunit functionality is implemented. There is generally an instance of a particular subunit driver for each subunit type and ID within that type.

For more information about the mechanism for subunit enumeration and the format of the device IDs associated with an AV/C device, see Peer Subunit Enumeration.
