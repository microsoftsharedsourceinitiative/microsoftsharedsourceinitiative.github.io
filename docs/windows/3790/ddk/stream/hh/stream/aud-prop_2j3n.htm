Streaming Devices (Video and Audio): Windows DDK

DRV_QUERYDEVICEINTERFACE

The DRV_QUERYDEVICEINTERFACE message queries for the device-interface name of a waveIn, waveOut, midiIn, or midiOut device.

Parameters

dwParam1
Pointer to a caller-allocated buffer into which the function writes a null-terminated Unicode string containing the device-interface name. If the device has no device interface, the string is of length zero.
dwParam2
Specifies the buffer size in bytes. This is an input parameter to the function. The caller should specify a size that is greater than or equal to the buffer size retrieved by the DRV_QUERYDEVICEINTERFACESIZE message.

Return Value

The xxxMessage function returns MMSYSERR_NOERROR if the message is handled successfully. Otherwise, it returns an appropriate error code.

Headers

Declared in mmddk.h. Include mmddk.h.

Comments

The following two message constants are used together for the purpose of obtaining device interface names:

  • DRV_QUERYDEVICEINTERFACESIZE
  • DRV_QUERYDEVICEINTERFACE

The first message obtains the size in bytes of the buffer needed to hold the string containing the device interface name. The second message retrieves the name string in a buffer of the required size.

The DRV_QUERYDEVICEINTERFACE message is supported in Windows Me, and Windows 2000 and later.

This message is valid only for the waveInMessage, waveOutMessage, midiInMessage, and midiOutMessage functions.

The system intercepts this message and returns the appropriate value without sending the message to the device driver.

For more information, see Obtaining a Device Interface Name.
