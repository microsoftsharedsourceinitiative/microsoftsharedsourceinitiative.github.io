MEDIA_SAMPLE_CONTENT

The MEDIA_SAMPLE_CONTENT enumeration contains values that specify the type of data in a group of packets that a packet identifier (PID) filter outputs. 

typedef enum {
    MEDIA_TRANSPORT_PACKET,
    MEDIA_ELEMENTARY_STREAM,
    MEDIA_MPEG2_PSI,
    MEDIA_TRANSPORT_PAYLOAD
} MEDIA_SAMPLE_CONTENT ;

Elements

MEDIA_TRANSPORT_PACKET
Packets in the output stream are complete transport stream (TS) packets. That is, the PID filter does not demultiplex packets from the input stream but just passes them through to the output.
MEDIA_ELEMENTARY_STREAM
Packets in the output stream consist of packet elementary stream (PES) payloads. That is, the PID filter demultiplexes packets from the input stream that contain either audio or video data.
MEDIA_MPEG2_PSI
Packets in the output stream consist of MPEG2 sections. That is, the PID filter demultiplexes packets from the input stream that contain IP data or data from a Program Allocation Table (PAT), Program Map Table (PMT), or Conditional Access Table (CAT). Each data sample output will consist of exactly one MPEG2 section.
MEDIA_TRANSPORT_PAYLOAD
Packets in the output stream consist of gathered TS packet payloads. That is, the PID filter demultiplexes packets from the input stream that contain, for example, PES or electronic program guide (EPG) packets.
