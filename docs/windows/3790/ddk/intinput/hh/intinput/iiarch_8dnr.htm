Driver Stacks for a Generic HIDClass Device

The following figure illustrates a driver stack for a generic HIDClass device (which might use optional and required vendor-supplied components).

Driver Stack for a Generic HIDClass Device

If the system-supplied HID minidrivers do not support a device, a vendor-supplied HID minidriver is required for a HIDclass device. However, the system-supplied components for HIDClass devices are designed to reduce the need for upper-level kernel-mode drivers for HID collections. A vendor-supplied function or filter driver for a HID collection is optional.

A user-mode application can directly access a HID collection by using the HIDClass support routines. The application calls CreateFile to obtain a file handle to the collection and uses the HIDClass support routines to communicate with the device.

For information about device stacks for HIDClass devices and operating HIDClass devices, see HIDClass Devices.
