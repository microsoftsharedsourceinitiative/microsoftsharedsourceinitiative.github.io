Requirements on Tablet PC HID Minidrivers

This section describes the general requirements on vendor-supplied HID minidrivers for pen devices and button devices that are installed on a Table PC. A pen device is integrated with the Table PC's LCD display and is used to capture the motion of a pen stylus. A button device supplements the pen device and is used to capture button input. For more information about the Tablet PC, see the Windows XP Tablet PC Edition Web site.

For detailed information about system-supplied software that supports the Tablet PC, see the Tablet PC documentation in the Platform SDK.

Pen and button devices belong to the HIDClass device setup class. These devices are operated by the system-supplied HID class driver, which is linked to a HID minidriver. In the absence of a system-supplied HID minidriver that supports the hardware interface for a device, a vendor-supplied HID minidriver is required. The structure of the driver stack (and the corresponding device stack) for these devices complies with that described in Device Stacks for Generic HIDClass Devices. The devices are operated from user mode by using the system-supplied Tablet PC API, which is described in the Platform SDK documentation.
