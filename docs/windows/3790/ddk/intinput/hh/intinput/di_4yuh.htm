Joystick Support

There are differences from version to version in the type of joystick support that Microsoft® DirectX® offers. In Windows® 95/98/Me, DirectX supports two methods to customize joystick capabilities: through custom entries in the Windows 95/98/Me registry and by creating a virtual device driver (VxD) called a joystick minidriver. The minidrivers that are used in DirectX versions 1.0, 2.0, and 3.0 support the original minidriver interface, with minor differences in the DirectX 3.0 interface. In addition to the original minidriver model, DirectX versions 5.0, and later, include an alternative driver interface that is usually described separately.

Windows 95/98/Me joystick driver and configuration programs support analog joysticks that plug into the IBM standard game port. Joystick makers can customize joystick configuration programs by providing explicit directions to the end user on how to customize the joystick. Joysticks can signal Windows 95/98/Me about their capabilities through the registry. These capabilities can include the use of throttle, point-of-view (POV) hats, rudders, and the number of buttons the joystick has.

All non-IBM standard joysticks, such as digital joysticks, MIDI joysticks, and analog joysticks driven by joystick accelerators must provide a joystick minidriver in addition to custom registry information. A joystick OEM can write a minidriver that provides access to nonstandard joystick hardware. This provides a mechanism for digital joysticks to work with any Windows-based game that uses the joystick application programming interface (API).

The driver model is capable of dealing with up to six axes, a POV hat, and a doubleword of buttons, so that an OEM can easily create a minidriver for new hardware with a higher degree of freedom than the current game port allows. The joystick minidriver provides complete flexibility to hardware vendors while allowing game creators to use the installed base with their titles. In DirectX 5.0 and later, analog joystick support is also separated into a minidriver using a new interface so that it is loaded only when an analog game port is configured. The polling has also been extended with three extra POV hats, three more doublewords containing button data, and a method to specify that velocity, acceleration, and/or force data be returned for each axis.

The current virtual joystick driver (VJoyD) allows the configuration of up to 16 devices, any number of which can be driven by minidrivers. The configuration of minidrivers to devices can be one to one or one to many.
