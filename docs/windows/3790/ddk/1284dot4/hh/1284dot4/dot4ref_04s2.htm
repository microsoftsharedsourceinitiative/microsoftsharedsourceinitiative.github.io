Dot4 Function I/O Control Codes

This section describes the following I/O control codes that a client can use with an IRP_MJ_DEVICE_CONTROL request to operate a Dot4 function:

IOCTL_DOT4_ADD_ACTIVITY_BROADCAST

IOCTL_DOT4_CLOSE_CHANNEL

IOCTL_DOT4_CREATE_SOCKET

IOCTL_DOT4_DESTROY_SOCKET

IOCTL_DOT4_OPEN_CHANNEL

IOCTL_DOT4_READ

IOCTL_DOT4_REMOVE_ACTIVITY_BROADCAST

IOCTL_DOT4_WAIT_ACTIVITY_BROADCAST

IOCTL_DOT4_WAIT_FOR_CHANNEL

IOCTL_DOT4_WRITE

For all other I/O control codes, the Dot4 driver skips the current stack location, and sends the associated device control requests down the device stack of the Dot4 device. The device control requests are completed by the lower-level drivers without further processing by the Dot4 driver.

The Dot4 driver only processes device control requests for Dot4 functions that are opened with write access.

The Dot4 driver queues device control requests for later processing.

Generic Status Values for Dot4 IOCTLS

The Information member is set to a request-specific value.

The Status member is set to a request-specific value or to one of the following generic values:

STATUS_SUCCESS
STATUS_BUFFER_TOO_SMALL
The Parameters.DeviceIoControl.InputBufferLength member of the IRP's current I/O stack location is less than the size, in bytes, of a DOT4_DRIVER_CMD structure.
STATUS_DELETE_PENDING
Plug and Play is removing the device.
STATUS_INSUFFICIENT_RESOURCES
One of the following errors occurred:
  • System memory for a dynamic data structure is not available.
  • The channel handle or channel is not valid.
  • The specified channel is closed.
STATUS_IO_DEVICE_ERROR
An error occurred that is fatal to the Dot4 session.
STATUS_PENDING
The Dot4 driver queued the request for later processing.
STATUS_UNSUCCESSFUL
One of the following errors occurred:
  • A Dot4 session is not running.
  • The client does not have write access to the Dot4 function.
  • An error occurred that is not fatal to a Dot4 session.
