IOCTL_DOT4_ADD_ACTIVITY_BROADCAST

Operation

The IOCTL_DOT4_ADD_ACTIVITY_BROADCAST request adds Dot4 activity broadcasting on a Dot4 channel.

Input

Irp->AssociatedIrp.SystemBuffer is a pointer to a client-allocated DOT4_DRIVER_CMD structure that inputs the following information:

hChannelHandle
Specifies a channel handle that is obtained by the client by using an IOCTL_DOT4_OPEN_CHANNEL request.

Other members should be set to zero.

The Parameters.DeviceIoControl.InputBufferLength member of the IRP's current I/O stack location is set to the size, in bytes, of a DOT4_DRIVER_CMD structure.

Output

None.

I/O Status Block

The Information member is set to zero.

The Status member is set to one of the generic status values for Dot4 IOCTLS.

Headers

Defined in d4drvif.h. Include d4drvif.h.
