Dot4 Function

A Dot4 function is a generic name for a child device of a Dot4 device.

A Dot4 device's INF file adds the specific name and description of each Dot4 function under the Functions key of the Dot4 device's Plug and Play hardware device key. A client can enumerate a Dot4 device's Functions key to determine which Dot4 functions the device supports. See Dot4 Functions Registry Key and Installing a Dot4 Device for more information.

A Dot4 function is opened and closed by using an IRP_MJ_CREATE request and an IRP_MJ_CLOSE request.

Opening a Dot4 Function Without Write Access

Opening a Dot4 function without write access has the following results:

  • A Dot4 function is not associated with a Dot4 session, and opening and closing a function has no affect on a session.
  • A client can receive Plug and Play notifications (or other types of notifications) for a device. For example, a client using the services the Microsoft Windows Image Acquisition (WIA) drivers can receive Plug and Play notifications.

Opening a Dot4 Function with Write Access

Opening a Dot4 function with write access has the following results:

  • A Dot4 function is associated with a Dot4 session.
  • If the Dot4 driver detects a device communication error during a session, its error handling operation ends the session, and completes I/O requests with status STATUS_UNSUCCESSFUL until the Dot4 driver resets device communication. The Dot4 driver resets device communication after clients close all Dot4 functions that were opened with write access. After the Dot4 driver resets device communication, a client can try to start a new session.
