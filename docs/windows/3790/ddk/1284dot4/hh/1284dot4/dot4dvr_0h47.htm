Dot4 Activity Broadcasting

A client can use Dot4 activity broadcasting to be notified after certain events, or activities, occur on a Dot4 channel.

A client can add activity broadcasting when a Dot4 channel is opened or by using an IOCTL_DOT4_ADD_ACTIVITY_BROADCAST request. If activity broadcasting is active on a channel, the Dot4 driver queues activities on an activity queue in the order in which they occur. An activity is represented by a DOT4_ACTIVITY structure.

A client registers to be notified if an activity occurs by using an IOCTL_DOT4_WAIT_ACTIVITY_BROADCAST request. The Dot4 driver completes the wait request if an activity exists on the activity queue. If there are no queued activities when the Dot4 driver processes the wait request, it sets the status of the request to STATUS_PENDING, and queues the wait request.

After a client adds activity broadcasting, it should immediately use an IOCTL_DOT4_WAIT_ACTIVITY_BROADCAST request. If activities are not removed at the same rate as they are added to the queue, the activity queue will expand indefinitely.

A client removes activity broadcasting by using an IOCTL_DOT4_REMOVE_ACTIVITY_BROADCAST request. The Dot4 driver deletes all activities on the activity queue, and completes all pending wait requests.
