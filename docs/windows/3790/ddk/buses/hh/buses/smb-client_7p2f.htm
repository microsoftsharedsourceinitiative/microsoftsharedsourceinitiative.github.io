SMB Client's AddDevice Routine

Each SMB client driver must have an AddDevice routine (see Plug and Play), which initializes device-specific information.

The PnP Manager calls the SMB client's AddDevice routine once for each SMB device it enumerates. The client's AddDevice routine must create and initialize a functional device object (FDO) as a target of I/O for an SMB device. The client driver then attaches itself to the device stack for the SMB device by calling IoAttachDeviceToDeviceStack, so that subsequent I/O requests to the underlying SMB device are routed through the client.

The AddDevice routine also initializes any device-specific data structures. If a class driver is present for the device type, the SMB client might additionally need to call a class driver initialization routine or perform other class-specific initialization.

The SMB client returns STATUS_SUCCESS from AddDevice if all initialization completes successfully. If any part of the initialization fails, the client should free any allocated storage and return a meaningful error status from AddDevice.
