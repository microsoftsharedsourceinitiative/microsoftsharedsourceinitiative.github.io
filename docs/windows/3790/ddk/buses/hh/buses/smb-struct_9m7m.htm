SMB_REQUEST

SMB_REQUEST provides information required to perform a specific I/O request on an SMB device. 

typedef struct {
  UCHAR  Status;
  UCHAR  Protocol;
  UCHAR  Address;
  UCHAR   Command;
  UCHAR  BlockLength;
  UCHAR  Data[SMB_MAX_DATA_SIZE];
} SMB_REQUEST, *PSMB_REQUEST;

Members

Status
Completion status of the SMB request.
Protocol
Specifies the bus protocol that applies to the current request. The class driver supports the protocols listed in the following table:
Protocol name Description
SMB_READ_QUICK Reads from a device but returns no data.
SMB_WRITE_QUICK Writes to a device, returning no data.
SMB_SEND_BYTE Sends one byte of data to a device.
SMB_RECEIVE_BYTE Receives one byte of data from the device.
SMB_READ_BYTE Reads a byte from an SMB device.
SMB_WRITE_BYTE Writes a byte to an SMB device.
SMB_READ_WORD Reads a word from an SMB device.
SMB_WRITE_WORD Writes a word to an SMB device.
SMB_READ_BLOCK Reads up to SMB_MAX_DATA_SIZE bytes from an SMB device.
SMB_WRITE_BLOCK Writes up to SMB_MAX_DATA_SIZE bytes to an SMB device.
SMB_PROCESS_CALL Combines functions of SMB_READ_WORD and SMB_WRITE_WORD. Allows a device to return a value that depends on the value input, without a synchronization mechanism to know when the data is ready.

Address
Bus address of the device to which this request applies.
Command
Device-specific command to perform.
BlockLength
Number of bytes of data to which this request applies. This value is input for a write request and output for a read request.
Data
Array of data input or returned by this request.

Headers

Declared in smbus.h. Include smbus.h.

Comments

Each client of an SMB miniport driver should define command codes that apply to its device.
