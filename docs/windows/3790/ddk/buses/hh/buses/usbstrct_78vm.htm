USB_EXTPORT_INFORMATION_0

The USB_EXTPORT_INFORMATION_0 structure is used in conjunction with the USB_EXTHUB_INFORMATION_0 structure and the GetExtendedHubInformation routine to retrieve information about the ports on a hub device. 

typedef struct _USB_EXTPORT_INFORMATION_0 {
  ULONG  PhysicalPortNumber;
  ULONG  PortLabelNumber;
  USHORT  VidOverride;
  USHORT  PidOverride;
  ULONG  PortAttributes;
} USB_EXTPORT_INFORMATION_0, *PUSB_EXTPORT_INFORMATION;

Members

PhysicalPortNumber
Contains the port number. This member can have a value between 1 and 255.
PortLabelNumber
Contains an integer port label. This number must not match the port number.
VidOverride
Indicates, when nonzero, that the vendor ID can be configured programmatically.
PidOverride
Indicates, when nonzero, that the physical device ID can be configured programmatically.
PortAttributes
Contains one or more of the flags listed in the following table that indicate the attributes of the port.
Flag Name Meaning
USB_PORTATTR_NO_CONNECTOR The port has no connector.
USB_PORTATTR_SHARED_USB2 The port is USB 2.0-compliant.
USB_PORTATTR_MINI_CONNECTOR The port is an unused miniport.
USB_PORTATTR_OEM_CONNECTOR The port is an OEM connector with nothing on it.
USB_PORTATTR_OWNED_BY_CC The port is owned by the companion controller. In systems that have both high-speed and low-speed controllers, the low-speed controller is called the "companion controller." In some cases, the ownership of a single port is handed back and forth between these two controllers.
USB_PORTATTR_NO_OVERCURRENT_UI The port has no overcurrent protection.

Headers

Declared in hubbusif.h. Include hubbusif.h.
