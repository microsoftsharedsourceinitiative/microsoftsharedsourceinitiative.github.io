Operation of an ACPI Device Function Driver

This section describes the generic operation of a vendor-supplied function driver for an ACPI device.

A function driver for an ACPI device is a WDM driver that:

  • Complies with the minimum requirements for a WDM function driver, as described in Windows Driver Model. This includes driver entry points, dispatch routines, Plug and Play, power management, and Windows Management Instrumentation (WMI). This basic functionality provides the generic operation that Windows requires of the driver and the framework in which to implement the ACPI device-specific operations.
  • Supports the device's operation region, which is the communication interface between the function driver and the ACPI BIOS — see Supporting an Operation Region.
  • Optionally, supports a vendor-defined device interface and IOCTLs that other drivers or user-mode applications use to operate a device — see Providing a Vendor-Defined ACPI Device Interface.
