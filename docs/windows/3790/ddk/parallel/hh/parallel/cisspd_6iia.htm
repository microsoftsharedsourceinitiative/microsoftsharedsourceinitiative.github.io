PPARALLEL_SET_CHIP_MODE

The PPARALLEL_SET_CHIP_MODE typed callback routine sets the operating mode of a parallel port. The system-supplied function driver for parallel ports supplies this routine.

typedef
NTSTATUS
(*PPARALLEL_SET_CHIP_MODE)(
    IN PVOID  SetChipContext,
    IN UCHAR  ChipMode
    );

Members

SetChipContext
Pointer to the device extension of a parallel port's functional device object (FDO).
ChipMode
Specifies the operating mode of a parallel port. (For more information on operating modes, see the modes defined for the enhanced capabilities register (ECR) in parallel.h of the Windows DDK.)

Return Value

STATUS_SUCCESS
The specified operating mode was set.
STATUS_INVALID_DEVICE_STATE
The mode is not cleared.
STATUS_NO_SUCH_DEVICE
The specified operating mode is not valid.

Headers

Declared in parallel.h. Include parallel.h.

Comments

To obtain a pointer to the system-supplied PPARALLEL_CLEAR_CHIP_MODE callback, a kernel-mode driver uses an IOCTL_INTERNAL_GET_PARALLEL_PNP_INFO request, which returns a PARALLEL_PNP_INFORMATION structure. The TrySetChipMode member of the PARALLEL_PNP_INFORMATION structure is a pointer to this callback.

A caller uses the PPARALLEL_SET_CHIP_MODE callback in conjunction with the PPARALLEL_CLEAR_CHIP_MODE callback.

To set a new mode, a caller must first clear the current mode.

The PPARALLEL_SET_CHIP_MODE callback runs in the caller's thread at IRQL <= DISPATCH_LEVEL.

For more information, see Setting and Clearing the Communication Mode on a Parallel Port.
