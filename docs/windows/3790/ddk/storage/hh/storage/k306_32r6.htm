MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY_OUTPUT

The MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY_OUTPUT structure holds a unique volume ID that a mount manager client provides to the mount manager in response to an IOCTL_MOUNTDEV_QUERY_UNIQUE_ID request. 

typedef struct _MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY_OUTPUT {
  ULONG  Size;
  USHORT  OldUniqueIdOffset;
  USHORT  OldUniqueIdLength;
  USHORT  NewUniqueIdOffset;
  USHORT  NewUniqueIdLength;
} MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY_OUTPUT

Members

Size
Contains the size in bytes of the buffer that follows this header.
OldUniqueIdOffset
Contains the offset in bytes into the output buffer where the old unique ID is stored.
OldUniqueIdLength
Contains the length in bytes of the old unique ID.
NewUniqueIdOffset
Contains the offset in bytes into the output buffer where the old unique ID is stored.
NewUniqueIdLength
Contains the length in bytes of the new unique ID.

Headers

Declared in mountmgr.h. Include mountmgr.h.

Comments

The mount manager can prompt a client with an IOCTL_MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY request to give the client a chance to change its unique ID name. If the client returns a new ID to the mount manager, it must prepend the ID information in the output buffer with the header defined in the MOUNTDEV_UNIQUE_ID_CHANGE_NOTIFY_OUTPUT structure.

The format for unique volume names is "\??\Volume{GUID}\", where GUID is a globally unique identifier that identifies the volume. For a discussion of unique volume IDs and how the mount manager uses them, see Supporting Mount Manager Requests in a Storage Class Driver.
