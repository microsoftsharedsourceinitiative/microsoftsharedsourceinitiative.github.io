Event Reporting

The WIA architecture enables a still image device to notify the WIA minidriver when some action on the device occurs. For example, a scanner might have a push-button on its front panel, allowing the user to initiate a scan directly from the device. The WIA minidriver must be notified of this event so it can notify the WIA service. All running applications that have registered to receive this event are notified. Furthermore, if an application has registered to be launched as a result of this event prior to the event, the WIA service launches that application.

The WIA architecture supports interrupt events and polled events. For more information, see WIA Driver Event Support.
