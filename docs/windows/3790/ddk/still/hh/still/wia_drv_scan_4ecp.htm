WIA Scanner Item Tree Layout

A WIA scanner item tree for Windows® Me and Windows XP consists of a root item and a single child item. See How the Application Creates the WIA Device for an example on how to create an item tree. For additional information, see Initializing the WIA Minidriver, Building and Maintaining an Item Tree, and IWiaMiniDrv::drvInitializeWia.

The WIA Scanner Item Tree

The root item in a scanner item tree contains information that is present in all WIA minidrivers, as well as scanner-specific properties. The scanner-specific properties include device optics information and document feeder support.

The child item represents the data-collecting functionality of the device and is used for transferring data. The child item for a scanner should be named to reflect the operations it can perform.

Microsoft requires the following names for Windows XP and later:

Root
This is the term for the root item of the WIA item tree.
Flatbed
This is the term for a scanner that supports a flatbed scanner only, or a scanner that supports a flatbed scanner with a document feeder.
Feeder
This is the term for a scanner that supports only a document feeder.

For Windows Me and Windows XP, an application must read WIA properties on the root item and on the first child item so it can determine the functionality of the scanner device.

An application can use the WIA service to perform the following operations:

  • Query scanner capabilities.
  • Set scanner device properties.
  • Request a data transfer.

Applications typically expect flatbed scanners, including those with automatic document feeders (ADFs), to be represented by two items: a root item and a single child item. All data transfers are performed from the child item.
