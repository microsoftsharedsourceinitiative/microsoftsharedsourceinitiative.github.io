WIA_DPC_CAPTURE_DELAY

The WIA_DPC_CAPTURE_DELAY property contains a value representing the amount of time delay, in milliseconds, that should be inserted between the capture trigger and the actual initiation of the data capture. This property is not intended to be used to describe the time between frames for single-initiation, multiple captures such as burst or time lapse, which have separate interval properties WIA_DPC_BURST_INTERVAL and WIA_DPC_TIMELAPSE_INTERVAL. In those cases it still serves as an initial delay before the first image in the series is captured, independent of the time between frames. For no precapture delay, this property should be set to zero.

Property Type: VT_I4

Valid Values: WIA_PROP_LIST or WIA_PROP_RANGE

Access Rights: READ/WRITE

Headers

Declared in wiadef.h. Include wiadef.h.
