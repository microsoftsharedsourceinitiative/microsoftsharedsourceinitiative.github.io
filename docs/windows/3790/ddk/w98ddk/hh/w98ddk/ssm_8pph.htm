SSM-USER.LOG and SSM-GDI.LOG

Output will be in the following format: 

Name
Owner
Num Objects
Type
Flags
Size Near 
Size Far
Name
Module name for this element. This name can be Global (object has no owner), orphaned (Object was orphaned by its owner), or will be the name of the corresponding application that allocated this object. When the name is unknown, there was no owner identified for this object. This does not mean the object has been leaked, it means that all of the methods for finding an application name for this object failed to get a name.
Owner
Owner handle. The flag's value can be used to determine if it was a 32-bit application or a 16-bit application.
Num Objects
Number of objects allocated to this module of the specified type. This is very useful in tracking application resource leaks.
Type
The type of object that was allocated to this module. One of the types described at the end of this section.
Flags
One of the flag values described at the end of this section.
Size Near
Total size of the Near heap these objects have consumed.
Size Far
Total size of the Far heap these objects have consumed.

  Note  Both USER details and GDI details use the same API and get the same data type back. The difference is only in their names. Hence, each of the /UD and /GD options generate identically formatted data with identical column definitions.

010A    User heap structure
010B    A saved popup bitmap.
010C    User check point information
010D    Cached DC for a window
010E    Pending DeferWindowPos
010F    Window Property Information
0110    Listbox State Information
0111    Miscellaneous User object
0112    User Atom
0113    Private window state recorded while a “LockInput” is active.
0114    User Hook
0115    Debugging memory
0116    Global Hotkey list
0117    Popup menu
0118    A shared icon
0119    User subsystem information
011A    An unprocessed message in the message queue
011B    Internal information structure
011C    A SendMessage that has not returned
011D    USER process information.
011E    Records all active AttachThreadInput()
011F    System color state as recorded by SetSysColorsTemp
0120    Message Queue
0121    Information used by the Alt+Tab window.
0122    A hook notification message
0123    A queued keystate change

GDI Types: 

 1    GDI Pen
 2    Handle to a brush
 3    Font object
 4    Palette
 5    Bitmap object
 6    Handle to a region
 7    Display Context

USER and GDI Flags:
