PCIMP_GARTInit

The PCIMP_GARTInit function initializes the miniport driver.

PCIMPRET
  PCIMP_GARTInit(
    ULONG  ulInstance, 
    PVOID  pHandle, 
    PVOID  *ppRefData
    );

Parameters

ulInstance
In a miniport with more than one chip set, this parameter specifies the chip set, corresponding to a number in the .inf file.
pHandle
PCI handle used when calling PCI services. This handle is passed into the PCIMP service functions which will need it when calling PCI services.
ppRefData
Pointer to where the service writes a reference data DWORD.

Return Values

This service returns a PCIMPRET value.

Comments

Typically, the PCIMP_GARTInit function will sense and initialize the GART hardware and its own internal data structures. The driver can get registry parameters using the PCIReadConfig function supplied by the PCI class driver.

Although it is highly unusual for a GART miniport to support multiple chip sets, the instanceNum parameter is provided for consistency with other miniport definitions. However, the operating system might require a GART miniport to support multiple instances of a single chip set. In that case, the PCI class driver will call this function more than once—each time with a different ulInstance.
