INFs for Installing USB Printing on Windows Operating Systems

This section presents two sample INF files that can be modified to install USB printing on Windows 98/Windows Me and Windows 2000 and later.

Vendors should supply unique Plug and Play hardware IDs (VID/PID) for their devices and modify the following examples as needed before including them in their driver packages.

Vendors should also prepare INFs as follows before submitting their driver packages to WHQL for testing and digital signature:

  • Use the ChkINF and INFCatReady utilities to validate the INFs for digital signature.

    ChkINF is provided in the Windows DDK. For information, see:
    http://www.microsoft.com/ddk/ddkdocs/win2k/chkinf.htm.

    INFCatReady can be downloaded from the Windows Hardware Quality Labs Web site.

  • Submit the INFs with your driver package to WHQL for testing and digital signature. For more information, see the Driver Signing for Windows Web site.
