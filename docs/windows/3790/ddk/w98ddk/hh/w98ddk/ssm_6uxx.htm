Windows 98/Me: Windows DDK

SSM-HEAP.LOG

The output will be in the following format: 

Module Name
Process ID
Thread Count
Heap Moveable
Heap Free
Heap Fixed
Virtual Committed
Virtual Free
Virtual Reserved

Definitions of items:

Module Name
Name of the module of the process being examined, for example, foo.exe
Process ID
A 32-bit number used as the Process Identifier for this process. The Kernel will use this to value as the tracking mechanism for this module.
Thread Count
Total number of threads in this process. At this time, there is no information that can be gathered about the threads.
Heap Moveable
Bytes used in the process heap that can be moved into the swap file when low memory conditions arise.
Heap Free
Number of bytes in the process heap that is currently free and unallocated.
Heap Fixed
Number of bytes in the heap which cannot be moved into the swap file.

Note  Heap Moveable, Heap Free and Heap Fixed correspond to the HeapAlloc, GlobalAlloc, and LocalAlloc APIs.

Virtual Committed
Number of bytes committed with a call to VirtualAlloc with the MEM_COMMIT flag.
Virtual Free
Number of bytes of free virtual memory. Each process will always see 2 GB of virtual memory in their memory space.
Virtual Reserved
Number of bytes reserved in a call to VirtualAlloc with the MEM_RESERVED flag.

Note  Virtual Committed, Virtual Free, and Virtual Reserved all correspond to memory allocated with a call to VirtualAlloc. Also note that there is 2 GB of addressable memory for every process.

This output will appear as one single tab-delimited line in the log file.
