_IOS_UnlockDrive

Summary

This service unlocks the specified drive if it was locked for exclusive access using _IOS_LockDrive.

Entry/Exit

The service expects the entry point previously provided by the _IOS_LockDrive service as the parameter passed in (TOS+4) and will return in EAX either a zero on success or non-zero if unlocking failed.

Usage

This service function is used to unlock a drive previously (and successfully) locked with _IOS_LockDrive, and uses the passed value to search for a lock descriptor. The return value indicates success or failure. Failure occurs when the service cannot find a valid lock descriptor, which usually means there is no locked drive.

If the function succeeds it will also cancel the lock timeout timer previously set running by _IOS_LockDrive.

This service uses (writes to) EAX, ECX, EDX, and Flags.
