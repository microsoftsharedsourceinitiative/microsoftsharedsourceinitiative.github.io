Managing Expensive Statistics

Some data blocks are expensive to maintain when no data consumers are actively interested in the information. If collection of a data block impacts the performance of a driver, WMI requires that a data provider specify that the data block be told when to begin and end data collection.

When the driver registers the data block, it should set the WMIREG_FLAG_EXPENSIVE flag in the Flags field of the WMIREGGUID structure. This will cause WMI to notify the driver when a data consumer is interested in the data block. WMI will manage any reference-counting issues in the case of multiple data consumers being interested in a data block. The notification is done using WMI IRPs with minor codes IRP_MN_ENABLE_COLLECTION and IRP_MN_DISABLE_COLLECTION.
