INFs for Installing USB Printing on Windows Operating Systems

This section presents two sample INF files that can be modified to install USB printing on Windows 98/Me and Windows 2000.

Vendors should supply unique Plug and Play hardware IDs (VID/PID) for their devices and modify these examples as needed before including them in their driver packages.

Vendors should also prepare INFs as follows before submitting their driver packages to WHQL for testing and digital signature:

  • Use the ChkINF and INFCatReady utilities to validate the INFs for digital signature.
  • ChkINF is provided in the Windows 2000 DDK. For more information, see ChkINF.
  • INFCatReady can be downloaded from the Windows Hardware Quality Labs Web site.
  • Submit the INFs with your driver package to WHQL for testing and digital signature. For more information about digital signatures, see Driver Signing.
