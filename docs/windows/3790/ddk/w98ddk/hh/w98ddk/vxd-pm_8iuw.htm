CONFIG_TEST_ SET_POWERSTATE_Dx

The configuration manager sends the CONFIG_TEST_ SET_POWERSTATE_D message to a device driver when the system goes to sleep to ask whether the device can go to sleep. Device drivers should return with success if the system can go to sleep, and return with failure if it cannot. Devices should fail this query only if user data loss will occur if the system goes to sleep.

If a driver returns success, it must stop any activity on the device that would prevent the system from going to sleep. That is, any activity that would cause the driver to return failure.
