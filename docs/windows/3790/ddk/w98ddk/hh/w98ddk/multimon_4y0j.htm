Failure Codes

If a display device fails to start, usually indicated by a yellow '!' in the device manager (and usually a code 10 error code in the device manager), the VDD writes a more detailed error code in the registry. This code is contained in the hardware key of the display device ( HKLM\Enum\PCI\deviceid\instance). The following failure codes are provided:

FAIL_ANOTHER_VGA
Plug and Play subsystem allocated VGA resources to two devices.
FAIL_BAD_NUM_UNITS
GET_NUM_UNITS return value from the miniVDD was invalid (usually too large).
FAIL_BIGROM
After-POST size of the display BIOS is greater than 64K.
FAIL_BIGROMDP
VDD shows this error if the after-POST BIOS size is greater than 64K, and paging is using MS-DOS.
FAIL_CANT_OFF_VGA
After disabling the device, the VDD detected that the device was still decoding VGA resources. Multimonitor support requires that all PCI devices be capable of being disabled through PCI or the TURN_VGA_OFF function.
FAIL_CANT_READ_ROM
Common error with display devices on system motherboards. The system ROM has disabled the ROM of the device, but not the device itself. The VDD tries to POST the device, but it cannot read the ROM, usually off of the motherboard. In motherboard display systems, the VDD will write the ROM image to the registry if it can read the ROM. On motherboard display systems, the only way to read the ROM, in most cases, is to boot Windows 98 with the motherboard display as the only display system. The VDD then writes the ROM image to the registry. When it tries to POST the device and cannot read the ROM itself, the VDD gets the ROM image out of the registry.
FAIL_ISA_VGA
Some or all devices were detected as ISA devices. Multimonitor support assumes that all secondary devices are PCI devices.
FAIL_MINIVDD
MiniVDD returned an error when trying to load.
FAIL_MINIVDD_VGA
MiniVDD hooked a call that is reserved for the miniVDD of the main VGA device.
FAIL_NO_DRV2
Device has not started because there is no DRV2 entry in the software key of the registry..
FAIL_NON_PHYS_C000
Display BIOS does not exist at C000-C7FF.
FAIL_NONE
Fail code when the device is operating properly.
FAIL_POST_ASD
POST=0 is in the software key of the device, meaning that the last time the VDD tried to POST the device, it failed, hung the machine, or crashed the machine.
FAIL_TOO_LATE
VDD has started the system and it cannot start the main VGA after Windows has been started. This usually happens when the main Windows device is not a VGA and a VGA card is enabled by the user as a secondary display.
FAIL_VGA_NOT_PCI
Device was started, but it was detected as not being on a PCI bus.
