MiniVDD INF File

The INF file for the miniVDD has to include references to the INF files for the slave minidrivers so Plug and Play can load the slave minidrivers.

Adding the following lines in the INF AddReg section of the miniVDD INF file informs Plug and Play of a slave minidriver:

HKR,"ENUM\deviceN",pnpid,,"<actualpnpid>"

In this example N is a number ranging from 1 to N, and <actualpnpid> is the ID of the slave minidriver.

For example:

HKR,"ENUM\Device1",pnpid,,"testdevice1"
HKR,"ENUM\Device2",pnpid,,"testdevice2"
