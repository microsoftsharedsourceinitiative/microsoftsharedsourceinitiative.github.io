Initialization

The following steps are typically followed when initializing and using streaming minidrivers. These steps are not unique to streaming video. Also see the Streaming Minidrivers.

When initializing and using streaming minidrivers

  1. The hardware adapter supported by the minidriver is detected by the Plug and Play enumerator.
  2. Plug and Play loads the minidriver and calls the minidriver’s DriverEntry routine. A file object is created from the information in the DriverEntry routine.
  3. The minidriver calls the Stream class driver’s StreamClassRegisterAdapter function and passes the HW_INITIALIZATION_DATA structure as a parameter. The HW_INITIALIZATION_DATA structure includes the addresses of minidriver functions that handle stream request block (SRB) command codes. This allows the minidriver to respond to SRB codes sent by the class driver. The extensions to the SRB command codes that are unique to capturing video are detailed in the SRB reference at the end of this document; a complete list of SRB Command Codes supported by the stream class is included in the Stream Class SRB Reference.
