Operation

The cam minidriver passes the stream request blocks (SRB) associated with the video stream to USBCAMD. USBCAMD will then start and stop the ISO stream when necessary. USBCAMD will call the cam minidriver when it is necessary to allocate bandwidth and free bandwidth (usually the cam minidriver will select an appropriate alternate interface). This ensures that the camera will only be using USB bandwidth when it is running. The cam minidriver “sees” each packet on the ISO stream and is responsible for indicating the start of a new video frame. Some SRBs are handled by the cam minidriver. For example, SRBs that report video format or handle property requests are handled entirely by the minidriver.
