Preparing to Start

You can start the System Debugger at boot, using a debug io.sys and wdeb.ini, or as a virtual device driver, by specifying it in the SYSTEM.INI file. Before you start, however, prepare for debugging by carrying out these steps:

  1. Connect a serial terminal or other computer by serial cable to a serial port on your debugging computer. Or connect the two computers with a 1394 cable.
  2. Create symbol files for the applications, DLLs, and VxDs you intend to debug (see Creating Symbol Files).
  3. Install the system debug environment if available (see Installing Debug Environment). These are debugging versions of the system DLLs and VxDs and their symbol files that can make debugging easier. These files are available in the Windows Device Driver Development Kit. 1394 debugging requires the use of Rterm.exe.
