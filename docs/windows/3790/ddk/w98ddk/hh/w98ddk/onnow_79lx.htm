SET_MONITOR_POWER_STATE

The SET_MONITOR_POWER_STATE function places the monitor into the power state specified in the PowerState parameter. If your driver makes only a VESA call, it should simply return CR_DEFAULT and let the operating system carry out the state change.

SET_MONITOR_POWER_STATE does not need to be hooked by most display drivers. It is provided as a convenience for special cases where the miniVDD needs to do something proprietary to control the monitor's power states.

DWORD __cdecl 
  SetMonitorPowerState(
    DEVNODE  devnode,
    DWORD  PowerState
  );

Parameters

devnode
Specifies the device node for the device.
PowerState
One of the following:

CM_POWERSTATE_D0
CM_POWERSTATE_D1
CM_POWERSTATE_D2
CM_POWERSTATE_D3

Return Value

The following values are provided.

Value Description
CR_SUCCESS Power state was changed successfully. No further action needs to be taken by the operating system.
CR_DEFAULT Operating system should perform its default action for the power state transition (a VESA BIOS call).

This should also be returned if PowerState is something other than one of the four CM_POWERSTATE_DX values.
