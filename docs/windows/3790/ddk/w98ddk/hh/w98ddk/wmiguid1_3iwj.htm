SCSI Miniport WMI Events

The port driver will also support WMI event notification. The new notification type, WMIEvent, will be added to ScsiPortNotification. The third argument to ScsiPortNotification is a pointer to the WMI event and the fourth argument should be the size of that event. The fifth argument is 0xFF if the event is for the adapter, or the PathId of the event's target device. If the event is for a target device, arguments six and seven will be the TargetId and Lun of the device.

Note that the miniport driver cannot post events that have not been previously enabled. Also, if events are posted too quickly, some of them might be lost.

The miniport can indicate to WMI that the data items it supports have changed by calling ScsiPortNotification with a notification type of WMIReregister. This notification can be used if the number of instances that a miniport supports have changed. The third argument should be 0xFF if the adapter is being reregistered, or the PathId of the target device being reregistered. If the notification is for a target device, the fourth and fifth arguments should be the TargetId and Lun of that device.
