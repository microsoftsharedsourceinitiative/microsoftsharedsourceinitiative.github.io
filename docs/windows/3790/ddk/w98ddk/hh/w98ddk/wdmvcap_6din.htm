KSPROPERTY_CROSSBAR_PININFO

The KSPROPERTY_CROSSBAR_PININFO read-only property is used to retrieve the type of physical connection represented by the pin. In addition, video pins can optionally return the index of related audio pins. The property uses the following structure:

typedef struct {
  KSPROPERTY  Property;
  KSPIN_DATAFLOW  Direction; 
  ULONG  Index;
  ULONG  PinType;
  ULONG  RelatedPinIndex; 
  KSPIN_MEDIUM  Medium;
} KSPROPERTY_CROSSBAR_PININFO_S, *PKSPROPERTY_CROSSBAR_PININFO_S;

Members

Property
Specifies the property used.
Direction
Indicates either KSPIN_DATAFLOW_IN or KSPIN_DATAFLOW_OUT.
Index
Specifies the index of the pin for which information is being returned.
PinType
Specifies a value from KS_PhysicalConnectorType.
RelatedPinIndex
Specifies the index of an pin which is related to a given pin. For example, an audio pin could indicate the video pin with which it is associated. When a device supports separate composite and s-video inputs but only a single audio connector, then both video streams should set RelatedPinIndex to the index of the audio pin.
Medium
Specifies a GUID representing the hardware connection.
