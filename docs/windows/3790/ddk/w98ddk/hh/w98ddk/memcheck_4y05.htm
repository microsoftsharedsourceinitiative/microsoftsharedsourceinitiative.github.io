Memcheck Log Files

Memcheck automatically generates one or two log files. The first and most important log file is located at c:\mcxsnap.log. This log file contains nothing but snapshots and is appended to every time the display refreshes on the application window (coinciding with the time that the driver tabulates and assembles a ring zero memory report).

This file is helpful for stress testing. If the user notices that there is no more memory available in the system, but cannot immediately isolate an offending component, this file can be parsed out and a pattern can be identified by the user. A separate tool is provided with Memcheck to parse the log file and chart out memory activity on a per-driver basis. Basically, any driver showing an unacceptable increase in memory usage should be investigated further for memory leaks.

The second file contains records of ring zero memory allocation and deallocation errors. It is located at c:\mcxerr.log. For more information on these errors see Memcheck Errors. The parser, Memparse, currently shipping with Memcheck, will intake both of these files and chronologically associate each error with its corresponding driver and snapshot number.

Memparse parses a given mcxsnap.log file into a set of text files containing graphs of each drivers' memory behavior trends. By default, Memparse operates on c:\mcxsnap.log and outputs its chart files in c:\memparse\.txt. These locations can be changed with the /f: and /d: command-line parameters. The output files are designed to be viewable on a 25x80 console.
