Serenum Device Enumeration

This section describes the following topics associated with Serenum's enumeration of Plug and Play serial devices attached to serial ports.

  • Enumerating High-Capacity Multiport Adapters
  • Opening a Serial Port for Enumeration

Enumerating High-Capacity Multiport Adapters

In Microsoft® Windows® 2000, boot time can be significantly delayed by the time it takes to automatically enumerate high-capacity multiport adapters (containing 16 or more ports). To address this issue, Windows XP and later supports the following two enhancements:

  1. Compared to Windows 2000, the time Serenum requires to automatically enumerate high-capacity multiport adapters is substantially reduced.
  2. Serenum supports an optional SkipEnumerations registry entry value for each serial port installed in a system. A vendor can use this entry value to control whether or not Serenum enumerates a port (whether initiated by a system boot or by a user through the Device Manager or Hardware Wizard).

Windows does not support a single registry setting that globally controls the enumeration of all serial ports.

Windows 2000 does not support controlling a serial port's enumeration by setting a SkipEnumerations registry entry value.

For a description about how to set a serial port's SkipEnumerations entry value in Windows XP, see Serenum SkipEnumerations Entry Value.

Opening a Serial Port for Enumeration

Serenum must open a serial port to enumerate it. Devices that keep a port open indefinitely should not use Serenum.
