Plug and Play Serial Device on a Multifunction Device that Requires a 16550 UART-Compatible Interface

This section describes the configuration of hardware, drivers, and device stacks for multifunction serial devices that:

  • Support Plug and Play
  • Require a 16550 UART-compatible interface
  • Are not connected to an RS-232 port

An example is a PCMCIA card with a modem and a LAN adapter.

The following figure shows the typical configuration for an example Toaster device and an example Blender device.

Left Figure: Hardware Configuration for a Toaster device on a Multifunction PC Card

Right Figure: Configuration of Drivers and Device Stacks for a Toaster device and a Blender Device

In this configuration the Toaster device is a child device on the multifunction device, and the multifunction device is a child device on a PCMCIA bus.

The INF file and installers for the Toaster device install the Serial driver as lower-level device filter driver to provide a 16550 UART-compatible interface for the Toaster device. The Toaster driver creates and attaches an FDO to the Serial driver filter DO.
