Registry Settings for the Serial Service

This section describes the registry settings that Serial applies to all serial devices for which Serial is the function driver or a lower-level device filter driver.

Serial queries the service entry values after it is loaded. If an entry value is not present, Serial adds the service entry value. Serial sets the value of the entry value to the default value that is statically defined in serial.sys. If a service entry value is changed after Serial is loaded, the new value is used the next time Serial is loaded.

Serial uses the following service entry values that are under the ..\Services\Serial registry key:

ForceFifoEnable
The ForceFifoEnable entry value is a Boolean flag that specifies whether or not to force Serial to use FIFOs. If ForceFifofEnable is nonzero, FIFOs are used, regardless of whether or not Serial can detect the presence of FIFOs. Otherwise, FIFOs are only used if Serial can detect them. ForceFifoEnable is a REG_DWORD and the default value is nonzero. If the entry value is not present, Serial sets a ForceFifoEnable entry value to the default value. For more information on the method of detection, see the sample code in %user install directory%\src\kernel\serial in the Windows 2000 DDK.
RxFIFO
The RxFIFO entry value specifies the number of bytes in the receive FIFO that triggers a port interrupt. For valid values, see the constants defined in serial.h. RxFIFO is a REG_DWORD and the default value is eight bytes. If the entry value is not present, Serial sets a RxFIFO entry value to the default value.
TxFIFO
The TxFIFO entry value specifies the number of bytes in the transmit FIFO that triggers a port interrupt. For valid values, see the constants defined in serial.h. TxFIFO is a REG_DWORD and the default value is 14 bytes. If the entry value is not present, Serial sets a TxFIFO entry value to the default value.
PermitShare
The PermitShare entry value is a Boolean flag that specifies whether or not to permit the system to share the interrupt that a port uses. If PermitShare is nonzero, the interrupt can be shared. Otherwise, the interrupt can not be shared. PermitShare is a REG_DWORD and the default value is set to 0x00000000. If the entry value is not present, Serial sets a PermitShare entry value to the default value.
BreakOnEntry, DebugLevel, and LogFifo
These entry values are used for debugging. For more information on these entry values, see the sample code.
