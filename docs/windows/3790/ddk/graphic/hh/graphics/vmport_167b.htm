Video Port Support for Bug Check Callbacks

In Windows XP SP1 and later, a video miniport driver can implement HwVidBugcheckCallback, a callback routine that is called when a bug check of a specified type occurs.

The video miniport driver registers this callback routine by calling the VideoPortRegisterBugcheckCallback function. In the call, the miniport driver specifies the bug check code to which the HwVidBugcheckCallback function should respond. (Currently, only bug check 0xEA is supported.)

When the specified bug check occurs, the system calls HwVidBugcheckCallback, which collects data and appends it to a dump file. Display and video miniport driver developers can use this additional data to diagnose problems in their drivers.
