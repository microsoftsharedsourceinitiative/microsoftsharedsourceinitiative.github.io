Video Miniport Driver Functions

This section provides reference pages for functions that can or must be implemented in a video miniport driver.

Function names in the form of HwVidXxx and SvgaHwIoPortXxx are pseudonames only; that is, these functions can have any name chosen by the driver writer. DriverEntry is a required name. Video Miniport Header Files, Sample Code, and References lists the include files that contain declarations for these functions.

It is highly recommended that all miniport driver functions be made pageable, with a few exceptions. Miniport driver functions that must not be made pageable are:

  • HwVidInterrupt
  • An HwVidSynchronizeExecutionCallback function that is passed in calls to VideoPortSynchronizeExecution
  • HwVidResetHw
  • HwVidBankedMemoryCallback
  • HwVidTimer
  • An internal function called by any of the preceding functions

For more information about video-specific I/O requests that must be supported by the port-miniport driver pair, see also Video I/O Control Codes. For more information about the sequence in which these miniport driver functions are called, see Video Miniport Drivers.
