Installing DirectX Drivers

This topic applies only to Windows 98/Me.

Creating Installation Files for Driver-Only Installation

In this scenario, the hardware vendor provides all the driver files that make up the device driver in the distribution package, including the device's INF file.

DirectX does not provide a method for hardware vendors to distribute the runtime components. For information about distributing these components, send e-mail to directx@microsoft.com.

Using DirectX Setup

During the end-user installation process, DirectX Setup is executed to install the DirectX subsystem. DirectX Setup installs the entire DirectX subsystem, consisting of Microsoft DirectDraw®, DirectSound®, DirectPlay®, Direct3D®, and DirectInput®; there is no option for partial installation. When redistributed by ISVs and on the Web, DirectX Setup also upgrades drivers if necessary.

DirectX Setup for DirectX uses version numbering and detection of registered drivers to decide whether a driver will be updated. To determine a driver's version number, DirectX Setup examines the version numbers of all files composing the device driver and uses the highest as the version number for the entire driver.

In DirectX, the Setup application backs up any driver that is replaced during installation, and also installs dxsetup.exe as a part of the DirectX subsystem. In addition, the capability for uninstalling DirectX drivers is registered with Add/Remove Programs in Control Panel.
