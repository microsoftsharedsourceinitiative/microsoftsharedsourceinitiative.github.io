Mapping the Deinterlace DDI to DirectDraw and DirectX VA

The DirectX VA DDI is divided into two functional groups: the DirectX VA container and the DirectX VA device. The purpose of the container DDI is to determine the number of DirectX VA devices contained by the display hardware and the capabilities of each device. A DirectX VA driver can have only a single container, but it can support multiple devices.

It is not possible to map the deinterlace device query calls to any of the DDI entry points in the container group because, unlike the rest of DirectX VA, the container methods use typed parameters.

The device DDI group does not use typed parameters, so it is possible to map the deinterlace interface to the functions in that group. The remainder of this section describes how the deinterlace interface is mapped to the DirectX VA device DDI.
