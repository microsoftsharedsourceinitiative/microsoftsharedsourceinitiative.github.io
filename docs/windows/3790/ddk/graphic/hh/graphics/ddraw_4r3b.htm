Asynchronous DMA Support in DirectDraw

This topic applies only to Windows 98/Me.

Versions of DirectDraw before DirectX 5.0 did not support asynchronous direct memory access (DMA) operations. This was due to the lack of a mechanism whereby the DirectDraw runtime and the driver could maintain consistent knowledge about the status of a given DMA operation. As of DirectX 5.0, DirectDraw offers such a mechanism using the DdGetSysmemBltStatus HAL callback. The driver provides a pointer to DdGetSystemBltStatus through a call to its DdGetDriverInfo function using the GUID_MiscellaneousCallbacks GUID. The driver specifies DdGetSystemBltStatus in the GetSysmemBltStatus member of the DDHAL_DDMISCELLANEOUSCALLBACKS structure.
