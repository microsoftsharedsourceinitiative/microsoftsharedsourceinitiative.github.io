NonVGA-Compatible, x86-Based Miniports

Most miniport drivers are nonVGA-compatible, and are consequently much simpler to implement.

NonVGA-compatible video miniport drivers rely on having the system-supplied VGA miniport driver (vga.sys) or another VGA-compatible SVGA miniport driver loaded concurrently. Such a miniport driver is set up to configure itself in the registry with VgaCompatible set to zero (FALSE) and has the following features:

  • It provides no special support for full-screen MS-DOS applications in x86-based machines. Instead, it is loaded along with a system-supplied VGA (or, possibly, with a VGA-compatible SVGA) miniport driver, which provides this support for full-screen MS-DOS applications.
  • In most cases, it either is written for an adapter that has no VGA compatibility mode or for an accelerator that works independently of the VGA.
