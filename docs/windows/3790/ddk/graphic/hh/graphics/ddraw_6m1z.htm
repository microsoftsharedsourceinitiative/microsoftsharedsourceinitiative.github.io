Clip Lists

Clipped blts are never passed to the driver on Microsoft Windows 2000 and later. The IsClipped member of DD_BLTDATA is always FALSE, and the clipped list is always NULL.

Note for Windows 98/Me  On Windows 98/Me, DirectDraw supports passing an entire clip list to the driver (if the driver supports this) rather than calling the driver multiple times, once for each rectangle in the clip list. If the driver can take a list of rectangles and blt them all in a FOR loop, then the driver should set the DDCAPS_CANCLIPSTRETCHED and DDCAPS_CANCLIP flags in the DDCORECAPS structure.

In DdBlt, if a clipper is attached to the destination surface and the clipper indicates that the destination rectangle is occluded, DirectDraw breaks the blt into a list of simple unoccluded rectangles. If the driver sets the DDCAPS_CANCLIPSTRETCHED and DDCAPS_CANCLIP flags, DirectDraw calls the driver once, only expecting it to run through the list of rectangles described in the DD_BLTDATA structure. If the driver does not set these flags, then DirectDraw calls the driver once for each unoccluded rectangle, setting the DD_BLTDATA so it contains only one rectangle at a time.
