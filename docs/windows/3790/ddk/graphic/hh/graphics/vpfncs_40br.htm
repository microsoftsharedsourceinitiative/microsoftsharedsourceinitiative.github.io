AgpCommitVirtual

The AgpCommitVirtual function maps reserved virtual memory to an associated range of AGP-decodable physical addresses.

PVOID
  (*PAGP_COMMIT_VIRTUAL)(
    IN PVOID  HwDeviceExtension,
    IN PVOID  VirtualReserveContext,
    IN ULONG  Pages,
    IN ULONG  Offset
    );

Parameters

HwDeviceExtension
Pointer to the miniport driver's device extension.
VirtualReserveContext
Identifies a reserved virtual address range. The context handle was obtained from AgpReserveVirtual.
Pages
Specifies the number of pages of virtual memory to commit.
Offset
Specifies the page offset at which to commit the pages. The offset is applied to the reserved physical address range associated with PhysicalReserveContext

Return Value

AgpCommitVirtual returns the virtual address for the base of the committed pages if the mapping succeeded; otherwise returns NULL.

Headers

Declared in videoagp.h. Include video.h.

Comments

A video miniport driver must first reserve virtual addresses and associate them with AGP-decodable physical memory by calling AgpReserveVirtual before it calls this function.

When a miniport driver calls AgpCommitVirtual, Pages pages of virtual memory are locked down and mapped to the AGP-decodable physical address range.This range begins Offset pages into the physical address range associated with VirtualReserveContext. An application can access committed virtual memory as if it were contiguous.

A miniport driver must reserve and commit both physical and virtual memory before an application can use AGP-decodable physical memory.

Callers of AgpCommitVirtual must be running at IRQL = PASSIVE_LEVEL.
