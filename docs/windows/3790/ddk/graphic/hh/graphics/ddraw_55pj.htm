DirectDraw Driver Initialization Components

This topic applies only to Windows 98/Me.

Ddraw.dll, ddraw16.dll, and ddhelp.exe are all components of DirectDraw involved in DirectDraw driver initialization:

  • Ddraw.dll is the 32-bit DirectDraw component that contains the DirectDraw code.
  • Ddraw16.dll is the 16-bit DirectDraw component that handles thunking to the DirectDraw portion of the GDI driver, receiving mode information from the GDI driver, dynamic color changing, and ModeX information.
  • Ddhelp.exe is a helper process that loads the ddraw.dll and the 32-bit DirectDraw driver DLL into its process space. This ensures that ddraw.dll and the 32-bit DLL always remain loaded. Ddhelp.exe has a thread blocked on all processes that enables ddraw.dll and dsound.dll to do cleanup when a process is killed or exits.
