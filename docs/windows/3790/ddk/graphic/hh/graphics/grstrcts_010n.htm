GLYPHPOS

The GLYPHPOS structure is used by GDI to provide a graphics driver with a glyph's description and position.

typedef struct _GLYPHPOS {
  HGLYPH  hg;
  GLYPHDEF  *pgdf;
  POINTL  *ptl;
} GLYPHPOS, *PGLYPHPOS;

Members

hg
Handle to the glyph.
pgdf
Pointer to a GLYPHDEF union.
ptl
Specifies a POINTL structure that contains the coordinates of the point in device space where the character origin of the glyph should be placed. Note that if the driver had set GCAPS_HIGHRESTEXT in the flGraphicsCaps member of DEVINFO at enable time, the x and y coordinates should be interpreted as FIX point quantities; otherwise they should be interpreted as LONGs.

Headers

Declared in winddi.h. Include winddi.h.

Comments

For printer drivers:
If a printer graphics DLL is executing in user mode, and if the DLL has received the GLYPHPOS structure as input to DrvTextOut or as a result of calling STROBJ_bEnum (see STROBJ), then the pgdf member is always NULL. In such cases, the DLL can call FONTOBJ_cGetGlyphs to obtain the glyph's bitmap or outline.
