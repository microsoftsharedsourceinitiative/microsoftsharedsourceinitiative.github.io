mciDriverYield

Do not write an MCI driver to support a multimedia device; write a WDM AVStream driver instead.

The mciDriverYield function lets the operating system process tasks while the driver is in an idle loop waiting for the given multimedia device to complete a command.

UINT
  mciDriverYield(
    MCIDEVICEID wDeviceID
    );

Parameters

wDeviceID
Specifies the device identifier of the multimedia device that yields to the operating system.

Return Value

Returns zero on success; otherwise, returns a nonzero value if the driver should abort the operation because the break key was pressed.

Headers

Declared in Mmddk.h. Include Mmddk.h.

Comments

Typically, an MCI device driver returns control to the calling application immediately after processing the command message. For many devices, control returns to the application even if the hardware requires several minutes to complete the action that was initiated by the command.

If the application uses the MCI_WAIT flag, the MCI device driver must delay returning to the application while the device completes the action requested. While the MCI device driver waits for the device to finish, the driver periodically calls mciDriverYield to let the operating system process other tasks and check for the MCI break key.

For more information about MCI, see the Platform SDK documentation.
