OS_DEP_DATA (VxD)

The OS_DEP_DATA structure declaration defines the data stored in the OsData member of the SMARTCARD_EXTENSION structure. It holds smart card information that is specific to the operating system. 

typedef struct _OS_DEP_DATA {
  PSMARTCARD_EXTENSION  SmartcardExtension;
  PDIOCPARAMETERS  CurrentDiocParams;
  OVERLAPPED  *NotificationOverlappedData;
  PVMMMUTEX  Mutex;
} OS_DEP_DATA, *POS_DEP_DATA;

Members

SmartcardExtension
Pointer to the smart card device extension.
CurrentDiocParams
Contains a structure of type PDIOCPARAMETERS that has the DIOC parameters to be processed. For further information on DIOC parameters, see the Windows SDK.
NotificationOverlappedData
Pointer to the overlapped data that is used for card tracking completion. For further information on the OVERLAPPED structure, see the Windows 2000 SDK.
Mutex
Contains a mutex that reader drivers should use to synchronize access to the smart card driver library.

Headers

Declared in smcnt.h. Include smcnt.h.

Comments

To allocate this structure, drivers must call SmartcardInitialize (WDM). Do not use this structure to store driver-dependent information. When the smart card driver library calls one of your callback functions, it sets CurrentDiocParams to the requesting parameters, unless the request is a card-tracking requests. For card-tracking requests, the smart card driver library sets NotificationOverlappedData to the overlapped structure of the caller.
