VxD Device Names for Smart Card Readers

VxD device drivers use the driver name and the link name.

Driver Name: As with every other application that accesses a device managed by a VxD driver, a smart card application must provide the driver name when using the DeviceIoControl system call. The driver name must uniquely identify the driver and usually looks like DEVICE.VXD.

Symbolic Link Name: Since the same VxD driver might potentially manage several different smart card reader devices, a VxD driver must add a key value to the registry indicating to the resource manager which reader device is managed by the driver. The value name must have the format SCReader[Unit], and it must be assigned to the Devices key under the following registry hive:

System\CurrentControlSet\Services\VxD\Smclib

The VxD version of the driver library routine SmartcardCreateLink (VxD) automatically adds a value with the appropriate format to the appropriate registry key.
