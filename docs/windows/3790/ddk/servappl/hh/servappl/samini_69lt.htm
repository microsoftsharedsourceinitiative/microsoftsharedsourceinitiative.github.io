Server Availability Miniport Driver Functions

This section provides reference pages for functions that are supplied by a server availability (SA) miniport driver. All names for driver functions—except the DriverEntry function—are pseudonames to be decided upon by the driver writer.

The initial entry point of any kernel-mode driver has the explicit name DriverEntry in order to be loaded. For more information about the general requirements for and functionality of DriverEntry functions, see Standard Driver Routine Requirements.

Calling SaXxx Port Driver Functions at Appropriate IRQLs

Each function description specifies the default IRQL at which that function runs. The default IRQLs of certain miniport driver functions effectively restrict the set of SaXxx port driver functions that these miniport driver functions can call. For example, it is a fatal error for an SaInterruptServiceRoutine function, which runs at DIRQL, to call any SaXxx port driver function that requires calls from IRQL <= DISPATCH_LEVEL.

For specific information about the range of valid IRQLs for calls to each SaXxx port driver function, see Server Availability Library Functions. For more information about IRQLs, see Managing Hardware Priorities.
