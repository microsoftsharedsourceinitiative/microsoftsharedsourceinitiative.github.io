SaInterruptServiceRoutine

The SaInterruptServiceRoutine function quickly services a device interrupt and, if necessary, arranges for the SaIsrForDpcRoutine function to be called to complete the processing of received data.

BOOLEAN 
  SaInterruptServiceRoutine(
    IN struct _KINTERRUPT *Interrupt,
    IN PVOID ServiceContext
    );

Parameters

Interrupt
Pointer to an interrupt object. For more information on interrupt objects, see Introduction to Interrupt Objects.
ServiceContext
Pointer to a driver-determined context. For more information on this context, see Providing ISR Context Information.

Return Value

SaInterruptServiceRoutine returns TRUE if the interrupt came from the driver's device; otherwise, it immediately returns FALSE.

Headers

The prototype for SaInterruptServiceRoutine is declared in ntddk.h as (*PKSERVICE_ROUTINE). Include ntddk.h.

Comments

The SaInterruptServiceRoutine function is required if the miniport driver’s SA device generates interrupts.

An SA miniport driver should do as little work as possible in its SaInterruptServiceRoutine function, deferring I/O operations for each interrupt that the driver's device generates to the SaIsrForDpcRoutine function. For more information, see Writing an ISR.

SaInterruptServiceRoutine runs at DIRQL, in particular at the DIRQL that was assigned when the driver called the IoConnectInterrupt function to initialize the interrupt object.
