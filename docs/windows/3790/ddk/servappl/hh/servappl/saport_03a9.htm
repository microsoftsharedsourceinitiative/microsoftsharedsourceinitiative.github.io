SaPortInitialize

The SaPortInitialize function informs the SA port driver about an SA miniport driver's entry-point functions and the device type that the SA miniport driver controls.

NTSTATUS
  SaPortInitialize(
    IN PDRIVER_OBJECT DriverObject,
    IN PUNICODE_STRING RegistryPath,
    IN PSAPORT_INITIALIZATION_DATA SaPortInitData
    );

Parameters

DriverObject
Pointer to a DRIVER_OBJECT structure for the SA device driver.
RegistryPath
Pointer to a UNICODE_STRING structure that describes a string of 16-bit characters. This string specifies the \Registry\Machine\System\CurrentControlSet\Services\DriverName key, where the value entry of DriverName identifies the miniport driver.
SaPortInitData
Pointer to a SAPORT_INITIALIZATION_DATA structure that informs the SA port driver about the SA miniport driver.

Return Value

Returns STATUS_SUCCESS on success; otherwise, it returns one of the error status values defined in ntstatus.h.

Headers

Declared in saport.h. Include saport.h.

Comments

The SaPortInitialize function performs the following operations:

  • Finds and allocates an SA device.
  • Creates a DEVICE_OBJECT object for the SA device.
  • Initializes the DRIVER_OBJECT object corresponding to the SA miniport driver.
  • Creates a symbolic link to the name of the SA device in order to make the SA device visible to the Microsoft® Win32® subsystem.
  • Calls the SA miniport driver's SaHwInitialize function to set up the SA device for I/O operations.

Callers of SaPortInitialize must be running at IRQL = PASSIVE_LEVEL.
