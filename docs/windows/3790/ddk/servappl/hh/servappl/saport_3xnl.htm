SaPortGetRegistryValueInformation

The SaPortGetRegistryValueInformation function retrieves the registry-value type and the size of the buffer that contains the data for the registry value given the registry value's name.

NTSTATUS
  SaPortGetRegistryValueInformation(
    IN PVOID MiniPortDeviceExtension,
    IN PWSTR ValueName,
    IN OUT PULONG RegistryType,
    IN OUT PULONG RegistryDataLength
    );

Parameters

MiniPortDeviceExtension
Pointer to a driver-allocated area in which the miniport driver maintains the state of the SA device.
ValueName
Pointer to a buffer that contains a null-terminated string of 16-bit Unicode characters that contains the name of the registry value from which information is retrieved.
RegistryType
Caller-supplied pointer to a variable that receives a code that indicates the type of data contained in the specified value. The following type codes can be returned:
Type code Meaning
REG_DWORD A 4-byte numerical value
REG_BINARY Binary data in any form
REG_SZ A null-terminated Unicode string

RegistryDataLength
Caller-supplied pointer to a variable that receives the size, in bytes, of the buffer that stores the data for the specified value.

Return Value

Returns STATUS_SUCCESS on success; otherwise, it returns one of the error status values defined in ntstatus.h.

Headers

Declared in saport.h. Include saport.h.

Comments

Callers of SaPortGetRegistryValueInformation must be running at IRQL = PASSIVE_LEVEL.
