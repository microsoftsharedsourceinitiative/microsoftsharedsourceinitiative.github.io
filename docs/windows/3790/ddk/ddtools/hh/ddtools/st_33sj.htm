Using WPP Software Tracing in a Driver

To use the default form of WPP software tracing in a kernel-mode driver, do the following:

  • Define a control GUID that uniquely identifies the driver's software trace. The driver specifies this GUID in its definition of the WPP_CONTROL_GUIDS macro and in a related control file used by the Tracelog utility.
  • Add the required WPP-related C preprocessor directives and WPP macro calls to the driver's source files, as described in Adding WPP Macros to a Driver and in WPP Software Tracing Reference.
  • Build the driver, as described in WPP Preprocessor.
  • Use the WPP utilities and files to control tracing and to convert the message information to a human-readable form.
