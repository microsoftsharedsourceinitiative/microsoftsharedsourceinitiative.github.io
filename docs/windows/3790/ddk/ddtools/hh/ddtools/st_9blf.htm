Formatted Trace Message File

The Tracefmt utility creates a formatted trace message file, which contains trace log messages converted to a human-readable form. A formatted trace message file is a text file that contains zero or more lines, where each line consists of a prefix followed by the message logged by a DoTraceMessage call. The messages are added to the message file in the order in which they were originally logged.

The default prefix contains the following fields, left to right:

  • CPU number
  • Process ID, Thread ID, and Timestamp in GMT
  • Friendly name of the control GUID, as described in the comments section of Tracelog Utility

A user can change the prefix by modifying the TRACE_FORMAT_PREFIX environment variable, which is a string that specifies a message definition compatible with FormatMessage. The default value of TRACE_FORMAT_PREFIX is the following:

[%9!d!]%8!04X!.%3!04X!::%4!s! [%1!s!]

Each %n is a prefix variable identifier that indicates one of the built-in parameters described in the following table.

Prefix Variable Identifier Variable Type Description
%1 string Control GUID's friendly name, as specified in the session's control file. This control file is specified by the –guid ControlFileName option of the Tracelog utility.
%2 string Name of the source file, and line number in the file, that contains the DoTraceMessage statement that logged the message. The format of the string is a concatenation of the file name, with the period before the file name's extension replaced by an underscore, followed by line number.
%3 ULONG Thread ID
%4 string System time
%5 string Kernel time
%6 string User time
%7 LONG Sequence number
%8 ULONG Process ID
%9 ULONG CPU number
%!FUNC! string C function that contains a DoTraceMessage macro

For an example of content in a formatted trace message file, see Formatting Trace Messages from the tracedrv Sample.
