Organization of Boot Options in NVRAM

Like the boot options on x86-based systems, the boot options in NVRAM on Itanium-based systems are of two types:

  • Globally-defined variables that determine which systems boot and in what order, such as BootOrder, an ordered array of NVRAM boot entry IDs, and Timeout, the boot menu timeout.

    These variables apply to all bootable devices and bootable programs on the computer, including all operating system configurations.

  • Boot option variables that apply only to a particular load configuration of a bootable device or program, such as an operating system.

    These system-specific variables comprise a boot entry for each configuration of a bootable device or bootable program on the computer.

Unlike the boot.ini file, NVRAM storage is finite, and it is used to store other vital system variables, including options that that determine the boot order of devices, and EFI driver entries, which are settings that direct EFI to load particular EFI driver extensions for devices. As a result, you need to manage the NVRAM resource and delete unused boot options periodically.
