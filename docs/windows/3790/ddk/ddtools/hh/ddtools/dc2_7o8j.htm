Windows Socket Test

Command-line parameter: /w

During the Windows Socket (WinSock) Test, Device Path Exerciser establishes a pair of sockets and tries to transmit the opened file or device, between the sockets using the fast transfer mechanism. Device Path Exerciser performs this test on devices opened during the Basic Open Operations and other open operations.

The Windows Socket Test is divided into two parts:

  • Part 1: Device Path Exerciser builds the socket and creates a listen thread.
  • Part 2: Device Path Exerciser attempts to transmit the file.
