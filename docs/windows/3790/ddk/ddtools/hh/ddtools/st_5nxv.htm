Trace Message Format File

The Tracepdb utility creates a trace message format file from the message definitions that the WPP preprocessor adds to a driver's PDB file. The Tracefmt utility uses the information in a trace message format file to convert logged messages to a human-readable form. A trace message format file is a text file that contains the following information:

  • A software trace's control GUID and the control GUID's friendly name.
  • For each DoTraceMessage statement in the driver's source code, a message line that specifies the message type, source code file name, line number, message number, message definition string, trace flag name, and the name of the C function that contains the macro call. Messages are formatted using the formatting capability of the FormatMessage function (described in the Platform SDK documentation).
  • List of variables and their associated internal type names, which are referenced by the %n notation in a message definition string.

The trace message format file is primarily for internal use and its format is subject to change.
