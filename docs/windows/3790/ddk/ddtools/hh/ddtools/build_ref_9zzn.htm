LINKER_NOREF

Use the LINKER_NOREF macro to turn off switches to the linker.

By default, the Build utility turns some switches on. One of them is the OPTICAL_AND_REF switch, which instructs the Build utility to dispose of everything that is not referenced in the current module. This default helps to keep your modules small, but can have the disadvantage of causing your to lose, for example, debug routines that you might want to call from the debugger.

To avoid such problems, set the LINKER_NOREF value in your environment, rebuild your product, load it up in the debugger, and then run those functions that are only used in the debug scenario.
