The Checked Build and the Debugger

The typical setup for debugging kernel-mode drivers on an NT-based operating system consists of two systems that are connected by a null modem cable (or optionally, for Windows XP and later, by a 1394 connection). For details, see Hardware Requirements for Debugging.

The host computer is the system on which the debugger runs. It should be a stable system and should always run the free build of the operating system.

The target computer is the system on which the driver you are testing runs. It can run either the free build or the checked build. For more accurate debugging, the checked build is preferred.

The debugger running on the host computer controls the target computer through the attached null modem cable or 1394 debugging cable.
