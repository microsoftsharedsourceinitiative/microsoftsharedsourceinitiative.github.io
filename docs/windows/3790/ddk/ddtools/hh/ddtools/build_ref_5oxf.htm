COFFBASE_TXT_FILE

The COFFBASE_TXT_FILE macro specifies the name of the file that contains the base addresses for the images you build.

By default, this is %DDKROOT%\bin\coffbase.txt. You can specify a different file by setting the COFFBASE_TXT_FILE macro equal to the full path and file name of the alternate file.

It is recommended that you set COFFBASE_TXT_FILE to a value other than the default. If you edit the coffbase.txt file in its default location and then reinstall the Windows DDK, the file will be overwritten.

The linker will open this file and search for the name of the executable you are building. See COFFBASE for details of what name is used in this lookup.

The file should have three columns. The first is the name of the executable. The second is the base address of the image; this is the starting address of where the executable image will reside in memory. The last is the maximum size of the image. Numbers can be prefixed with 0x (hexadecimal) or not prefixed at all (decimal).

There should also be an entry with the name "usermode" that is the default value.

Here is an example:

usermode      0x60000000    0x20000000
MyDll         0x70000000    0x00010000
AnotherDll2   0x70010000    0x00010000

In this example, if you have set COFFBASE to MyDll, the linker will use a starting address of 0x70000000 and a maximum size of 0x00010000.

Note  Any specified base address or size is just an educated guess. Different architectures and build environments are going to generate different size binaries — thus, for example, the coffbase.txt file you use in an x86 free build environment may be inappropriate for an Itanium checked build environment. It is probably best to make your best guess when writing your coffbase.txt file, and then running the rebase.exe tool after the build is complete to pack all your binaries together in an optimal manner.
