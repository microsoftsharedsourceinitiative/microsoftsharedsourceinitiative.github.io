Query and Set Security Test

Command-line parameter: /sd

During the Query and Set Security Test, Device Path Exerciser issues calls to retrieve the security descriptor and change the security state of devices opened by the Basic Open Operations and other open operations, including the Relative Open Test.

Device Path Exerciser issues each query or set call at least 1024 times with a valid buffer and a variety of buffer lengths and security information types (OWNER_SECURITY_INFORMATION, GROUP_SECURITY_INFORMATION, DACL_SECURITY_INFORMATION, SACL_SECURITY_INFORMATION, and no information type). One request of each type is also sent with an invalid buffer pointer and a zero buffer length.

This test is optional. To add the Query and Set Security Test to a Device Path Exerciser test session, use the /sd parameter.

If you use the /pr (protection) parameter to set the protection option, Device Path Exerciser varies the security setting on the buffer in each query and set call.

This test uses the ZwQuerySecurityObject, and ZwSetSecurityObject functions, which are documented in the Installable File Systems Kit (IFS Kit).
