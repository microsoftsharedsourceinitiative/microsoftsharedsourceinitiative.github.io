Statistics for ATM Miniport Drivers

The following chart summarizes the OIDs used to get or set statistics for ATM miniport drivers.

Length Q S Name
4 O   OID_ATM_CELLS_HEC_ERROR

Cells received with header error control errors
4 O   OID_ATM_RCV_CELLS_DROPPED

Cells received and dropped for any reason
4 O   OID_ATM_RCV_CELLS_OK

Cells received without errors
4 O   OID_ATM_RCV_INVALID_VPI_VCI

Cells received with an unrecognized VPI and/or VCI
4 O   OID_ATM_RCV_REASSEMBLY_ERROR

Received packets that failed reassembly
4 O   OID_ATM_XMIT_CELLS_OK

Cells transmitted without error

The following describes each of the OID_ATM_XXX listed in the preceding chart more fully.

OID_ATM_CELLS_HEC_ERROR

The number of cells that were received with a header error control (HEC) error. A HEC error is a checksum error for a cell header.

OID_ATM_RCV_CELLS_DROPPED

The number of received cells that were dropped for any reason.

An ATM miniport driver must support this OID on an per-adapter basis and can optionally support this OID on a per-VC basis.

OID_ATM_RCV_CELLS_OK

The number of cells that were received without errors.

An ATM miniport driver must support this OID on an per-adapter basis and can optionally support this OID on a per-VC basis.

OID_ATM_RCV_REASSEMBLY_ERROR

The number of packet reassembly errors that have occurred. Such an error occurs when a packet reassembled from received cells fails a checksum test or if the reassembly times out (that is, if the NIC receives all but the last cell for a packet and times out waiting for that cell).

OID_ATM_RCV_INVALID_VPI_VCI

The number of cells that were received with unrecognized VPI and/or VCI values.

OID_ATM_XMIT_CELLS_OK

The number of cells that were transmitted without error.

An ATM miniport driver must support this OID on an per-adapter basis and can optionally support this OID on a per-VC basis.
