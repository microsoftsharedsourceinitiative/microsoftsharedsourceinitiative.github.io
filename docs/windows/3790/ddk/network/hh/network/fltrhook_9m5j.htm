Creating a Filter-Hook Driver

A filter-hook driver is a kernel-mode driver that implements a callback function called a filter hook and registers that callback function with the system-supplied IP filter driver. The IP filter driver then uses the filter hook to determine how to process incoming and outgoing packets. For more information on developing a kernel-mode driver, see Kernel-Mode Driver Components in the Microsoft® Windows DDK.

To supplement the Packet Filtering application programming interface (API), a filter-hook driver can be created to filter network packets. User-mode applications use this API to create and manage, in the system-supplied IP filter driver, input and output filters that filter packets with specific IP addresses or TCP/UDP port numbers. Because the Packet Filtering API optimizes the system-supplied IP filter driver to process packets without the overhead that is associated with a filter-hook driver, you should consider using this API if possible. For more information about the Packet Filtering API, see the Microsoft Windows Platform SDK documentation.
