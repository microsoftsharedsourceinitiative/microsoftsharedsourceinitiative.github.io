Setting Information for a Connection-Oriented Miniport Driver

To set an OID maintained by a connection-oriented miniport driver, a bound protocol calls NdisCoRequest and passes a structure of type NDIS_REQUEST that specifies the object (OID) being queried and that points to a buffer that contains the value to which the object should be set. The call to NdisCoRequest causes NDIS to call the miniport driver's MiniportCoRequest function, which sets the object with the supplied value.

The call to NdisCoRequest can complete synchronously or asynchronously. To complete the call asynchronously, a miniport driver calls NdisCoRequestComplete.
