NdisMGetDmaAlignment

NdisMGetDmaAlignment returns the alignment requirements of the DMA system for a NIC.

ULONG
  NdisMGetDmaAlignment(
    IN NDIS_HANDLE  MiniportAdapterHandle
    );

Parameters

MiniportAdapterHandle
NDIS handle that identifies the miniport driver's NIC. This handle was originally passed to MiniportInitialize.

Return Value

Returns a value that specifies the alignment requirements of the DMA system.

Headers

Declared in Ndis.h. Include Ndis.h.

Comments

A miniport driver can call the NdisMGetDmaAlignment function to determine alignment requirements for DMA buffers it allocates. A miniport driver might require increasing the returned value because of additional hardware device restrictions. A miniport driver should call the NdisMAllocateMapRegisters or NdisMInitializeScatterGatherDma function before calling NdisMGetDmaAlignment.

System support for NdisMGetDmaAlignment is available in Windows XP and later OS versions. Earlier NT-based OS versions and Windows 98/Me support the NdisGetCacheFillSize function for retrieving alignment requirements.

Callers of NdisMGetDmaAlignment must be running at IRQL = PASSIVE_LEVEL.
