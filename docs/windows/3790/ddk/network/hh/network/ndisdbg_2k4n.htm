Setting NDIS Debugging Options in the Registry

You can enable different levels of debug tracing in various NDIS components by setting values for the following entries under the HKLM\System\CurrentControlSet\Services\VxD\NDIS registry key:

(DWORD) DebugBreak (Boolean) 
(DWORD) TraceImpt (Boolean) 
(DWORD) DebugSystems 
(DWORD) DebugLevel

The following indicates acceptable values to set for DebugBreak, TraceImpt, DebugLevel and DebugSystems:

DebugBreak
Setting to 1 (enabled) causes NDIS to break into the debugger through an INT 3 when NDIS loads (that is, debug breaks in NDIS's DriverEntry function).
TraceImpt
The trace-important flag. Setting to 1 (enabled) causes trace output to be emitted to the debugger. While debugging, this flag should always be enabled.
DebugLevel
Selects the level or amount of trace information that is emitted to the debugger for the NDIS components that are selected with the DebugSystems value. The following values specify levels that you can select:
Level Description Value
INFO All available debug information. This is the highest level of trace. 0x00000000
LOG Log information. 0x00000800
WARN Warnings. 0x00001000
ERR Errors. 0x00002000
FATAL Fatal errors, which can cause the operating system to crash. This is the lowest level of trace. 0x00003000

INFO produces the most trace information. If TraceImpt is enabled, DebugLevel's default value is INFO; otherwise, it is FATAL.

DebugSystems
Selects various NDIS components to emit debug trace information. The following values specify the NDIS components that you can select:
Component Description Value
INIT Handles adapter initialization. 0x00000001
CONFIG Handles adapter configuration. 0x00000002
SEND Handles sending data over the network. 0x00000004
RECV Handles receiving data from the network. 0x00000008
PROTOCOL Handles protocol operations. 0x00000010
BIND Handles binding operations. 0x00000020
BUSINFO Handles bus queries. 0x00000040
REG Handles registry operations. 0x00000080
MEMORY Handles memory management. 0x00000100
FILTER Handles filter operations. 0x00000200
REQUEST Handles requests. 0x00000400
WORK_ITEM Handles work-item operations. 0x00000800
PNP Handles Plug and Play operations. 0x00001000
PM Handles Power Management operations. 0x00002000
OPENREF Handles operations that open reference objects. 0x00004000
LOCKS Handles locking operations. 0x00008000
RESET Handles resetting operations. 0x00010000
WMI Handles Windows Management Instrumentation operations. 0x00020000
CO Handles Connection-Oriented NDIS. 0x00040000
REF Handles reference operations. 0x00080000
ALL Handles all NDIS components. 0xFFFFFFFF

You can select more than one NDIS component. If more than one component is selected, OR the data values together. By default, if TraceImpt is enabled, the INIT, CONFIG, PNP, PM, PROTOCOL, BIND, BUSINFO, and REG components are selected. That is, the DebugSystems value appears as follows with these values selected: 

"DebugSystems"=dword:000030F3
