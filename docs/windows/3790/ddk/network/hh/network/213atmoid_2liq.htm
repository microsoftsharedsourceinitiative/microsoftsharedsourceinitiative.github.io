OID_ATM_ASSIGNED_VPI

The OID_ATM_ASSIGNED_VPI OID requests an ATM miniport driver to return an assigned VPI that the miniport driver has acquired (for example, from the registry). In an environment that supports more than one host per ATM switch port, a host may be required to transmit and receive using an assigned VPI. A connection-oriented client or call manager can request the assigned VPI so that it can appropriately specify VCs for outgoing and incoming calls.

If the miniport driver does not have an assigned VPI to return to the call manager, it returns NDIS_STATUS_NOT_SUPPORTED.
