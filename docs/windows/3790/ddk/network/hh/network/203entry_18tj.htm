Choosing a Receive Function for Connectionless Miniport Drivers

The way in which a connectionless miniport driver passes incoming packets to upper layers determines whether it must register a MiniportReturnPacket function or a MiniportTransferData function:

  • If the miniport driver indicates packets to upper layers by calling NdisMIndicateReceivePacket, it must register a MiniportReturnPacket function.
  • If the miniport driver is a legacy driver that indicates packets to upper layers by calling a filter-specific NdisMXxxIndicateReceive function, it must register a MiniportTransportData function.

For a description of miniport driver receive operations, see Receiving Data.
