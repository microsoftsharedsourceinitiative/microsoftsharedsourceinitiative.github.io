Handling a System Query-Power IRP in a Bus Driver

When a system query-power request reaches a bus driver (that is not the power policy owner for a device), the driver ensures that it can support a device state that corresponds to the queried system state and, if wake-up is enabled, that the queried system state would not prevent its device from waking the system.

The bus driver calls PoStartNextPowerIrp, and it sets Irp->IoStatus.Status to STATUS_SUCCESS if it can change to the specified power state or sets a failure status if it cannot. It then calls IoCompleteRequest, specifying IO_NO_INCREMENT.

After the bus driver completes the IRP, the Power Manager calls IoCompletion routines set by other drivers as they passed the IRP down the stack.
