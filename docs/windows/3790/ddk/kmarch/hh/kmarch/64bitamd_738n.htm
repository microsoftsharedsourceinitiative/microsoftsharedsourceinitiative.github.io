Ambiguous Conversion On Cast of Constant 0

In the following code, casting 0 to an int is ambiguous and produces errors; however, recasting to long and void* is valid:

void foo(long);
void foo(void*);
void g() {
    foo((int)0);
    foo((long)0);
    foo((void*)0);
}

In the general case, the workaround is to cast 0 to the exact type of the function parameter it is being used for, so that no conversions are required.
