C and C++ Language for AMD64

The following AMD64 compiler features are related to the C and C++ languages:

  • This compiler conforms significantly to the ANSI/ISO standard. If the compiler generates build errors or warnings not seen in earlier versions, the errors might be a result of the increased conformance to the standard. Refer to the Code Compatibility for AMD64 section for more information on the changes made.
  • If the same virtual function is declared in two or more interfaces and if a class is derived from these interfaces, you can explicitly override each virtual function.
  • Default packing (/Zp) is 16.
  • The intrinsic functions memset, memcpy, and memcmp might call library functions. These intrinsic functions might not be expanded inline even if you compile with /Oi.
  • You might need to change any code that does not specify the size of time_t and size_t. Both size_t and time_t are now __int64.
  • If you get an error outputting a size_t to an I/O stream:

    Support for the 6.0 version of iostream is being phased out. Functionality for iostream is now provided through the standard C++ library. Details of the iostream implementation have changed. Therefore, you might need to rewrite parts of your code that use iostream if you want to link with the standard C++ library.

    You will have to remove any old iostream headers (fstream.h, iomanip.h, ios.h, iostream.h, istream.h, ostream.h, streamb.h, and strstream.h) you have included in your code and add one or more of the new standard C++ iostream headers (fstream, iomanip, ios, iosfwd, iostream, istream, ostream, sstream, streambuf, and strstream, all without the .h extension).
