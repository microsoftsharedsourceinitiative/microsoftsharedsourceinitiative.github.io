KeGetCurrentProcessorNumber

The KeGetCurrentProcessorNumber routine returns the system-assigned number of the current processor on which the caller is running.

ULONG 
  KeGetCurrentProcessorNumber(
    );

Parameters

None

Return Value

KeGetCurrentProcessorNumber returns a ULONG value that represents the processor on which the caller is currently running.

Headers

Declared in ntddk.h. Include ntddk.h.

Comments

KeGetCurrentProcessorNumber can be called by NT-based drivers to debug spin lock usage on SMP machines during driver development. An NT-based driver also might call KeGetCurrentProcessorNumber if it maintained some per-processor data and attempted to reduce cache-line contention.

The number of processors in an SMP machine is a zero-based value.

If the call to KeGetCurrentProcessorNumber occurs at IRQL < DISPATCH_LEVEL, a processor switch can occur between instructions. Consequently, callers of KeGetCurrentProcessorNumber usually run at IRQL >= DISPATCH_LEVEL.
