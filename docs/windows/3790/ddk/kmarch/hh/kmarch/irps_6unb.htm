Canceling IRPs

Drivers in which IRPs might remain queued for an indefinite interval (so a user could cancel a previously submitted I/O request) must have one or more Cancel routines to complete user-canceled I/O requests. Examples of drivers that should have Cancel routines are keyboard, mouse, parallel, serial, and sound device drivers (or drivers layered over them), and file system drivers.

Drivers for Windows XP and later operating systems can use Cancel-Safe IRP Queues rather than implement their own Cancel routines.

To "cancel an IRP" means to complete the IRP as quickly as possible while still maintaining system integrity. For a general discussion of IRP completion, see Completing IRPs.

The cancellation process begins when either the system or a driver calls IoCancelIrp. The system cancels unprocessed IRPs if the thread that initiated the I/O request exits. Drivers can only cancel IRPs that they have created (see Creating IRPs for Lower-Level Drivers.)
