Making Drivers Pageable

The system-space virtual memory shown in the Virtual Memory Spaces and Physical Memory figure consists of a limited amount of paged pool and an even more limited amount of nonpaged pool.

Paged code runs at IRQL < DISPATCH_LEVEL; it can be paged in when referenced without bringing down the system or adversely affecting a driver's operation. Nonpaged code is always memory-resident; it can run at IRQL >= DISPATCH_LEVEL and never takes a page fault.
