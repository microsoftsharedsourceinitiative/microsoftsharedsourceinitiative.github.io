Kernel-Mode Driver Architecture: Windows DDK

Memory

This section describes the kernel-mode support routines and macros that drivers can call to:

  • Allocate and free temporary buffers in Buffer Management
  • Allocate long-term internal driver buffers in Long-Term Internal Driver Buffers
  • Manage buffered data or to initialize driver-allocated buffers in Buffered Data and Buffer Initialization
  • Get mapped addresses and to allocate or manage MDLs (memory descriptor lists) in Address Mappings and MDLs
  • Manipulate buffers and MDLs in Buffer and MDL Management
  • Communicate with their respective devices in Device Memory Access
  • Lock and unlock their pageable code or data sections, or that they can call to make their entire driver pageable in Pageable Drivers
  • Set up mapped sections and views of memory in Sections and Views
  • Access parts of structures in Access to Structures
