Documentation Conventions

This documentation uses the following typographic conventions.

Convention Indicates
bold System-supplied or system-defined functions and support routines, structure members, enumerators, and registry key names. For example, EngCreatePath is a system-supplied function provided to support graphics drivers. These items appear in the system header files provided by this DDK exactly as shown in this documentation.
italic Placeholder function names, formal parameters, or any other text that is meant to be replaced by driver writers.

For example:

AddDevice is a placeholder name for a required function whose actual name is defined by the driver writer.

DeviceObject is a formal parameter to a number of system-supplied support routines called by kernel-mode drivers.

Italic items in registry paths are placeholders that the driver writer should replace with driver- or device-specific text.
UPPERCASE Constant identifiers, structure and array types, bitwise operators, Microsoft MS-DOS® device names, and system-supplied macros. For example, FONTOBJ and IO_STACK_LOCATION are system-defined structures. Uppercase identifiers must be typed exactly as shown. 
monospace
 Code examples, such as: 
    hwInitData.DeviceIdLength=4;

Note  Some of the interfaces described in the Graphics Driver Design Guide and Graphics Driver Reference follow the convention known as Hungarian notation for formal parameters and variable names. That is, these are prefixed with lowercase letters indicating their data type. For example, lpszNewDocname is a long pointer to a zero-terminated string. For more information about Hungarian notation, see Programming Windows by Charles Petzold, published by Microsoft Press®.
