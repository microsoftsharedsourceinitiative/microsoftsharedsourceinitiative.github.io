Supporting Blacklisting and Delayed

Blacklisting and Delayed are response characteristics. Blacklisting occurs when the modem does not dial because there have been too many unsuccessful attempts. A delayed state occurs when the modem does not dial, until some time has elapsed, because there have been too many recent unsuccessful attempts.

Two codes in the bResponseState field of a Responses entry correspond to the Delayed and Blacklisting characteristics. For more information on the bResponseState settings in the Responses entries, see Data Modem INF File Entries. For more information about creating Responses entries, see Creating Data Modem INF File Entries with ProCalc.
