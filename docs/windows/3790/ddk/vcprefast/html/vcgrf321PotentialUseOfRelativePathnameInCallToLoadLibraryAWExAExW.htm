321 - Potential use of relative pathname in call to LoadLibrary(A/W/ExA/ExW).

This message indicates that you are passing a relative pathname to LoadLibraryA, LoadLibraryW, LoadLibraryExA, or LoadLibraryExW.

Passing a relative pathname can result in inadvertently picking up the wrong DLL, depending on what the current directory is, thus resulting in a security hole.

Note   Programmers developing on Windows Server 2003 can ignore this warning by filtering it out. For the Windows Server 2003 family, LoadLibrary includes the system directories in its search path, which eliminates the possiblity of picking up the wrong DLL. Warning 321 can still be generated; however, it can be safely ignored.

Example

Defective Source

LoadLibrary("file.ext");

Corrected Source

LoadLibrary("C:\\file.ext");
