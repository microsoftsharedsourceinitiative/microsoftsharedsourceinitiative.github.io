Suppressing Individual Warning Messages

PREfast includes support for pragma prefast, which enables source-level suppression of specific PREfast warnings.

You can use pragma prefast to suppress individual warnings in situations where PREfast is reporting an incorrect or irrelevant message.

Because pragma prefast is an extension to the PREfast AST compiler, it will not be recognized by other compilers, including the C++ compiler included with Visual C++ 6.0 and later. This will result in a 4068 warning. This warning can cause builds to fail under -W4 -Wx flags.

Using pragma prefast

To use pragma prefast, disable warning 4068 in your main build. This is the recommended method for using pragma prefast and can be done by adding the following to a header that is used in all your source files:

#ifndef _PREFAST_
#pragma warning(disable:4068)
#endif
Note   If you are using Visual C++ .NET 2002 or later, you can also add -wd4068 to the cl command line to suppress this warning.
